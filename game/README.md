# game — 나라카 밸리 게임 본체

스타듀밸리형 본편 게임(Godot 4.6 / GL Compatibility). **학습용인 [`../godot-sandbox/`](../godot-sandbox/)·[`../godot-tutorial/`](../godot-tutorial/)와 달리 이쪽이 진짜 게임이다.** Phase 1 Sprint 1에서 시작한 본체 프로젝트.

> ⚠️ Phase 1은 **회색 도형(그레이박스)만** 쓴다. 도트·초상화·사운드는 재미 게이트 통과 후 Phase 2에서만 입힌다(ADR-0001).

## 여는 법 / 실행

```bash
godot --path game -e   # 에디터로 열기
godot --path game      # 바로 실행
```

또는 Godot 실행 → **Import** → `game/project.godot` 선택 → **F5**.

## T3.4 — 여우불 성장 촉진 (현재 산출물)

**미호 호감도가 높을수록 작물이 더 빨리/넓게 자라는 차이가 체감된다**(ADR-0001 그레이박스 — 회색 밭·하트·HUD만). CONTEXT '여우불 성장 촉진'의 **방식 A(관계 보상형)** — 미호와 친해질수록 여우불 도움이 강해진다. T3.3 호감도(하트) 위에 그 이중 보상의 한 축(농사 효율↑)을 얹어, **관계와 핵심 농사 루프를 한 몸으로** 묶는다. 속죄 테마(ADR-0004): 미호의 불은 태우는 불 → 키우는 불(양육의 불).

| 요소 | 구현 | 메모 |
|---|---|---|
| 여우불 규칙 | `foxfire.gd`(`Foxfire`) — 호감도 하트 → 여우불 세기 **매핑 한 곳** | `crops.gd`처럼 static class_name 규칙(세이브 상태 아님). 세기는 하트에서 매번 파생 → 세이브할 게 없다(`SaveManager`·main 세이브 불변) |
| 가속('빨리') | `accel(hearts)` = 하트 2칸당 +1 (0/0/1/1/2/2). 물 준 칸이 하룻밤에 `+1+accel` 자람 | 같은 작물이 더 빨리 다 자란다. 하트0이면 0 |
| 범위('넓게') | `reach(hearts)` = 하트당 1칸 (0..5). 물 못 준 심긴 칸을 reach개까지 여우불이 대신 `+1` 돌봄 | 깜빡한 칸도 진척 → 밭 전체가 넓게 자람. (y,x) 정렬 순으로 골라 결정적 |
| 관계 게이팅 | 하트0이면 가속·범위 둘 다 0 = **여우불 잠듦**(순수 스타듀 성장) | 보상이 관계에 게이팅(방식 A). 입력 하트는 [0,MAX]로 자름(방어) |
| 밭 적용 | `field.gd advance_day(accel:=0, reach:=0)` — 인자화(기본 0 = 기존 T2.3 동작) | 물 준 칸은 범위 후보에서 빠져 가속·범위 **이중적용 방지**. `_grow`가 작물 성장일수까지만 잼(과성장 방지). `field`는 `Foxfire`/`Affinity`를 모름(디커플링) |
| 연결 | `main._on_day_advanced`가 `affinity.hearts()`로 `Foxfire.accel/reach` 파생해 `farm.advance_day`에 넘김 | 취침 시 호감도만큼 여우불이 돕는다 |
| UI | 좌상단 `여우불: N칸 돌봄 · 자람 +M` 한 줄(하트0이면 `잠듦 — 미호와 친해지면 깨어난다`) | `Foxfire.summary()`. 관계→농사 보상을 눈에 보이게(체감). 회색 라벨만 |

### 완료기준 직접 확인하기

`godot --path game`로 띄운 뒤:

1. **하트 0**(시작)일 땐 좌상단에 `여우불: 잠듦 …`이 뜬다. 밭에 작물을 심고 **물 준 칸만** 다음 날 자라며(물 안 준 칸은 그대로), 물 준 칸은 하룻밤에 1단계만 오른다 — 순수 스타듀 성장.
2. 미호와 **일일 대화·선물**로 호감도를 올려 하트를 채우면 좌상단이 `여우불: N칸 돌봄 · 자람 +M`으로 깨어난다.
3. 하트가 오른 뒤 작물을 심고 취침하면 **차이가 체감된다**: 물 준 칸이 하룻밤에 **여러 단계** 자라 더 빨리 수확가능해지고(가속), **물을 깜빡한 칸도** 여우불이 돌봐 자란다(범위 — 하트가 높을수록 더 많은 칸).

위가 되면 **T3.4 완료.** (여우불 로직은 헤드리스 단위검증으로도 통과: accel/reach 하트별 매핑·음수/초과 clamp·is_awake·summary 텍스트 / 가속0=기존 스타듀·물 안 준 칸 성장 없음 / 가속2 단번 성숙 / 가속 과성장 방지(성장일수까지만) / 범위 (y,x) 정렬 순 선택·한도 밖 제외 / 물 준 칸 범위 이중적용 없음 / 성숙 칸 범위 제외. 통합 임포트·구동 에러·경고 0.)

## T3.3 — 호감도(하트): 일일 대화 + 선물

**미호와 일일 대화(하루 1회 소폭)·선물(영혼 호박 선호)로 호감도가 오르고, 하트 단계가 좌상단 UI에 반영된다**(ADR-0001 그레이박스 — 회색 하트 막대만, 초상화는 Phase 2). T3.2 첫 사람 접점 위에 관계 루프를 얹는다. CONTEXT '호감도'의 이중 보상(친해질수록 농사 효율↑ = 여우불 강화 T3.4 / 캐릭터 이야기↑ = 미호 대사 변화)이 이 하트 단계 위에 얹힐 토대다.

| 요소 | 구현 | 메모 |
|---|---|---|
| 호감도 상태 | `affinity.gd`(`Affinity` 노드) — "미호 호감도 수치" **단일 책임**+`changed` 시그널 | `energy.gd`·`wallet.gd`와 같은 결. 점수·마지막 대화날·마지막 선물날 정수 셋이라 그대로 직렬화 |
| 두 채널 | ㉠ 일일 대화 `daily_talk`(+5 소폭, 느린 채널) / ㉡ 선물 `gift`(일반 +15·선호 영혼 호박 +40, 빠른 채널) | 둘 다 게임 날짜(`GameClock.day`)로 **"하루 1회" 게이팅**(같은 날 중복 보상 방지) |
| 하트 단계 | 점수 / `POINTS_PER_HEART`(50) 내림, `MAX_HEARTS`(5)에서 멈춤 | `heart_bar()`가 `♥♥♡♡♡` 막대 생성. 그레이박스 기준값, 밸런싱은 후속 |
| 일일 대화 | 미호 바라보고 `E` → 오늘 **첫 대화에만** 소폭 지급, 그 결과·하트로 대사 묶음 선택 | 대사는 미호가 들고 옴(ADR-0005). 하트0 온보딩 / 1~2 속죄 한 토막('방화→양육') / 3+ 더 깊은 토막+여우불 떡밥 / 오늘 두번째면 짧은 인사 |
| 선물 | 미호 바라보고 `G` → 선택 작물(`Q`로 고름) 수확물 1개 소모(`take_harvest`)·호감도↑ | 영혼 호박이면 큰 폭(선호). 수확물 없음·오늘 이미 선물 시 안내만(방어). 하루 1회 |
| 세이브 | `main._save_game`에 `affinity` 한 조각 추가 | 설계대로 `SaveManager`(IO 전담) 불변. 손상 점수는 [0,MAX] clamp, 게이팅 날짜 보존 |
| UI | 좌상단 `미호 ♥♥♡♡♡ N/5` 하트 막대 + 프롬프트 `[E] 대화  [G] OO 선물` | 회색 라벨만 |

### 완료기준 직접 확인하기

`godot --path game`로 띄운 뒤:

1. 밭 입구의 **미호**를 바라보면 `[E] 대화  [G] OO 선물` 프롬프트가 뜬다. `E`로 대화하면 좌상단 `미호 ♥…`의 점수가 **그날 처음엔 소폭** 오른다(같은 날 또 말 걸면 짧은 인사만, 점수 변화 없음).
2. 밭에서 작물을 키워 **수확**한 뒤(`Q`로 줄 작물 선택), 미호를 보고 `G`로 **선물**하면 호감도가 오른다. **영혼 호박**을 선물하면 더 크게 오른다(`(선호!)` 안내).
3. 점수가 쌓여 50을 넘을 때마다 좌상단 하트 막대가 `♥` 한 칸씩 채워진다(`N/5`) → **하트 단계가 UI에 반영** OK. 하트가 오르면 `E` 대화 대사도 바뀐다(속죄 서사 한 토막 해금).
4. 집에서 **취침해 날이 바뀌면** 일일 대화·선물을 또 할 수 있다(하루 1회 리셋). `F5` 저장 → 종료 → 재실행해도 하트가 그대로 복원된다.

위가 되면 **T3.3 완료.** (호감도 로직은 헤드리스 단위검증으로도 통과: 시작0·하트0·빈 막대 / 선호 판정 / 일일 대화 소폭·같은날 중복 금지·이튿날 재지급 / 선물 일반15·선호40·같은날 중복 금지 / 55점→1하트·막대 / MAX clamp / 세이브 라운드트립·게이팅 유지·손상·음수 방어 / `take_harvest` 소모·0키 삭제·없음 실패 / 미호 대사 하트단계·첫대화 분기. 통합 빌드 임포트·구동 에러·경고 0.)

## T3.2 — 미호 NPC 배치 + 대화 텍스트박스

**미호에게 말 걸면 텍스트박스가 뜨고, `E`로 끝까지 넘기면 닫힌다**(ADR-0001 그레이박스 — 회색 박스+텍스트만, 초상화 일러스트는 Phase 2). 간판 캐릭터이자 농사 멘토 미호(CONTEXT '미호')가 밭에 등장해, 다음 스프린트의 호감도(T3.3)·여우불(T3.4)이 붙을 첫 사람 접점을 만든다.

| 요소 | 구현 | 메모 |
|---|---|---|
| 미호 NPC | `miho.gd`(`Miho` 노드, Node2D) — `player.gd`처럼 16×32 회색 자리를 `_draw` | 플레이어보다 따뜻한 톤+머리 위 작은 뿔(여우귀 암시)로 한눈에 구분. 회색 기조 유지 |
| 대사 출처 | 미호가 `LINES`를 들고 옴(`lines()`) | ADR-0005: 서사 텍스트는 캐릭터에만. 밝은 멘토+"파괴의 불→양육의 불" 속죄(ADR-0004), 메인 플롯 비의존 |
| 대사 진행 | `dialogue.gd`(`DialogueBox` 노드) — 화자·줄·열림만 들고 `start`/`advance`/`is_last`/`progress` + `changed`/`finished` | 누구 대사든 모르는 **순수 진행기**(멜·바나 재사용). 대화는 일시 상태라 세이브 안 함(호감도는 T3.3) |
| 배치 | 밭 남쪽 입구 칸(`MIHO_TILE 20,14`) | 길에서 위를 보면 바로 미호를 향함(멘토가 문 앞에서 맞이). 그 칸은 `_is_farmable`에서 제외 → 밭 동작과 충돌 방지 |
| 말걸기/넘기기 | 미호 칸을 바라보고 `E`(밭 동작보다 우선) → 대화 중엔 player 물리 끄고 이동 잠금, `E`로 한 줄씩 | 끝까지 넘기면 패널 숨김+이동 잠금 해제 |
| UI | 화면 하단 회색 대화 패널(화자·본문·`N/M`·`[E] 다음/닫기`) | `changed` 시그널로 갱신(시그널 디커플링). 하단 프롬프트에 `[E] 미호와 대화` |

### 완료기준 직접 확인하기

`godot --path game`로 띄운 뒤:

1. 도착 지점에서 길을 따라 **위로** 올라가 밭 입구에 서면 `[E] 미호와 대화` 프롬프트가 뜬다.
2. `E`를 누르면 하단에 텍스트박스가 뜨고(`1/5`), `E`로 한 줄씩 넘긴다. 대화 중엔 캐릭터가 움직이지 않는다.
3. 마지막 줄(`5/5`, `[E] 닫기`)에서 `E`를 한 번 더 누르면 텍스트박스가 닫히고 이동이 다시 된다.

위가 되면 **T3.2 완료.** (대사 진행 로직은 헤드리스 단위검증으로도 통과: 빈 대사 무시 / 시작 시 첫 줄·화자 / 열린 중 재시작 무시 / 줄 넘김 / 마지막 판정 / 끝까지 넘기면 닫힘+`finished` 1회 / `changed`=줄 수 / 닫힌 뒤 advance 무시 / 미호 대사·이름. 통합 빌드 임포트·구동 에러·경고 0.)

## T3.1 — 카페 출하대: 수확물 → 골드 → 씨앗 재구매

**수확물을 팔아 골드를 얻고, 그 골드로 씨앗을 사서 다시 심는 한 바퀴가 돈다**(작은 경제 순환의 폐쇄, ADR-0001 그레이박스). T2.1~T2.3 밭 루프가 이제 닫힌 고리가 되어 "왜 농사를 짓는가"의 보상 동기가 생긴다. (CONTEXT상 카페 경제·회계는 멜의 속죄 영역(예정)이지만, 지금은 NPC 없이 화폐·출하대만 그레이박스로 둔다.)

| 요소 | 구현 | 메모 |
|---|---|---|
| 골드 | `wallet.gd`(`Wallet` 노드) — "골드 잔액" **단일 책임**+`changed` 시그널 | `energy.gd`와 같은 결. `spend`는 잔액 부족 시 false(음수 방지). `START_GOLD` 0 |
| 재고 | `inventory.gd`(`Inventory` 노드) — 수확물·씨앗 두 재고+`changed` | `field.gd`처럼 순수 Dictionary(crop_id→개수)라 그대로 직렬화. 카탈로그에 없는 id·음수 거부 |
| 종잣돈 | 골드가 아니라 **시작 씨앗**(`START_SEEDS` 혼령초 3) | 골드 0에서 첫 수확→첫 판매로 골드가 처음 생기는 자연스러운 순환. 새 게임에만 지급(로드가 덮어씀) |
| 심기/수확 연동 | 심기 시 `take_seed`(씨앗 1 소모), 수확 시 `add_harvest`(`crop_of`로 거둘 id 미리 확보) | 씨앗 0이면 심기 막힘 → "카페에서 구매" 안내 |
| 출하대 | **카페 구역 안에서 `E`로 패널 토글**(집 취침과 같은 '구역 안에서만' 패턴) | `S`=수확물 전량 판매(`sell_price` 합산), `B`=선택 작물 씨앗 구매(`seed_cost`). 구역 벗어나면 자동 닫힘 |
| 가격 데이터 | `CropCatalog.seed_cost` / `sell_price` 헬퍼 | 가격은 데이터(crops.gd)에 두고 조회만(데이터/로직 분리). 혼령초 +10 / 피안화 +35 / 영혼 호박 +110 순익 |
| 세이브 | `main._save_game`에 `wallet`·`inventory` 두 조각 추가 | 설계대로 `SaveManager`(IO 전담)는 불변. 손상 세이브는 0골드·빈 재고로 방어 |
| UI | 좌상단 `골드: N`·`씨앗N`, 출하대는 화면 중앙 회색 패널 | 회색 라벨·박스만 |

### 완료기준 직접 확인하기

`godot --path game`로 띄운 뒤:

1. **밭(중앙)** 에서 시작 씨앗(혼령초 3)으로 심고 → 물주고 → 며칠 취침해 키운 뒤 **수확**한다(수확물이 인벤토리에 쌓임). 씨앗을 다 쓰면 심기 프롬프트가 `씨앗 없음 — 카페에서 구매`로 바뀜.
2. **카페(우측)** 로 들어가 `E`로 출하대 패널을 연다 → `S`로 **전량 판매**(좌상단 골드 증가) → `Q`로 작물을 고르고 `B`로 **씨앗 구매**(골드 감소·씨앗 증가).
3. 산 씨앗을 들고 밭으로 돌아가 **다시 심는다** → 순환이 닫힘.

위가 되면 **T3.1 완료.** (경제 로직은 헤드리스 단위검증으로도 통과: 골드 earn/spend·음수 방지·clamp / 재고 누적·소모·0키 삭제·카탈로그 검증·손상 방어 / 시작 씨앗 / "심기→수확→판매→재구매" 순환 폐쇄 시나리오. 통합 빌드 임포트·구동 에러·경고 0.)

## T2.5 — 세이브/로드 최소 직렬화

저장 후 게임을 **껐다 켜도 날짜·밭·작물·혼력이 그대로 복원된다**(ADR-0001 그레이박스). 단일 슬롯 — 다중 슬롯·암호화·클라우드는 후속(범위 폭주 방지). T2.1~T2.4로 돈 농사·자원 루프가 한 세션을 넘어 이어진다.

| 요소 | 구현 | 메모 |
|---|---|---|
| 파일 IO | `save.gd`(`SaveManager` 노드) — "세이브 파일 입출력"이라는 **단일 책임**만 | `user://save.dat` 단일 슬롯. 어떤 상태를 저장할지 고르는 '조율'은 `main`이 맡음(이미 모든 노드 참조 보유) |
| 직렬화 | `var_to_str` / `str_to_var`(JSON 아님) | 밭(`_tiles`)이 **Vector2i를 키로** 쓰는 Dictionary라, JSON으론 키를 풀어야 하지만 `var_to_str`는 키 타입까지 그대로 라운드트립 |
| 자기 직렬화 | 각 시스템 노드의 `to_save()` / `load_save()` | `GameClock`(날짜·분) · `SoulEnergy`(혼력) · `FarmField`(밭 칸 Dictionary). 단일 책임대로 자기 상태만 직렬화 |
| 시각 동기화 | `FarmField.load_save()`가 칸마다 `tile_changed` 발화 | `main`이 듣고 오버레이를 다시 칠함(로드도 시그널 디커플링 유지). 로드 전 옛 오버레이 제거는 `main` 책임 |
| 저장 트리거 | **취침 시 자동 저장**(스타듀식) + **F5 수동 저장** | 자동 저장으로 "껐다 켜도"가 자연히 보장됨 |
| 복원 트리거 | **시작 시 세이브 있으면 자동 로드** + **F9 수동 불러오기** | `_ready`에서 `has_save()`면 복원 → "껐다 켜도 그대로"가 성립 |
| 방어 | 파일 없음·손상·버전 불일치 → `{}`(새 게임), 혼력 손상값 clamp | 그레이박스라 가볍게. 버전(`VERSION`) 래핑으로 포맷 변경 대비 |
| UI | 좌상단 `[F5] 저장 · [F9] 불러오기`, 저장/불러오기 시 2초간 `저장됨`/`불러옴` | 회색 라벨만 |

> **골드·인벤토리는 T3.1에서 함께 직렬화된다.** 설계대로 `main._save_game()`에 `wallet`·`inventory` 두 조각을 더했고, `SaveManager`(IO 전담)는 손대지 않았다(위 T3.1 참조).

### 완료기준 직접 확인하기

`godot --path game`로 띄운 뒤:

1. 밭에서 몇 칸 경작·심기·물주기를 하고 혼력을 좀 쓴 뒤 **`F5`로 저장**(좌상단에 `저장됨`) → 게임을 **완전히 종료**.
2. 다시 `godot --path game`로 **새로 실행**하면, 시작하자마자 **날짜·밭 칸 상태·심은 작물·혼력이 저장 시점 그대로** 복원됨 → "껐다 켜도 복원" OK.
3. 플레이 중 상태를 바꾼 뒤 **`F9`를 누르면** 마지막 저장 시점으로 되돌아감(밭 오버레이도 함께 갱신) → 불러오기 OK. (집에서 **취침해도 자동 저장**된다.)

위가 되면 **T2.5 완료.** (세이브/로드 로직은 헤드리스 단위검증으로도 통과: 각 노드 `to_save`/`load_save` 라운드트립 / 파일 경유 Vector2i 키·작물·성장일수·날짜 보존 / 복원 시 칸 수만큼 `tile_changed` 발화 / 손상·버전 방어 / 혼력 clamp. 통합 빌드 임포트 에러·경고 0.)

## T2.4 — 혼력(에너지) 행동당 소모 + 취침 회복

밭 행동(괭이질·심기·물주기·수확)을 할 때마다 **혼력이 줄고, 바닥나면 행동이 막히며, 취침하면 가득 찬다**(ADR-0001 그레이박스, CONTEXT '혼력'). "해 지기 전에 / 혼력 떨어지기 전에 끝내야 한다"는 스타듀식 자원 압박을 T2.1~T2.3 밭 루프 위에 얹는다.

| 요소 | 구현 | 메모 |
|---|---|---|
| 혼력 상태 | `energy.gd`(`SoulEnergy` 노드) — "혼력"이라는 **단일 책임**만 | `clock.gd`·`field.gd`와 같은 결. 상태는 정수 `current` 하나라 T2.5 세이브에 그대로 직렬화 |
| 시그널 | `changed(current, maximum)` / `depleted()` | `main`이 HUD 갱신. `depleted`는 향후 "지쳐 쓰러짐" 연출이 붙을 훅 |
| 소모 규칙 | 행동 한 번당 `COST_PER_ACTION`(10) 고정 소모 | CONTEXT '혼력'대로 **단순 고정 규칙으로 시작**, 행동별 차등·밸런싱은 후속(T4.3) |
| 막힘 | `can_act()` false면 `main`이 `interact`를 막음 | `MAX`(100)가 `COST`의 배수라 **딱 10번 행동 후 정확히 0**이 되어 막힘 = 완료기준 "0이면 막힘" |
| 회복 트리거 | `GameClock.day_advanced` → `SoulEnergy.refill()` | 취침 시 가득. 작물 성장(`advance_day`)과 **같은 훅에 나란히** 연결(시그널 디커플링) |
| UI | 좌상단 `혼력: NN/100`, 바닥나면 `지쳤다(취침 필요)` + 밭 안내가 `혼력 부족 — 집에서 취침`으로 | 회색 라벨만 |

### 완료기준 직접 확인하기

`godot --path game`로 띄운 뒤 **밭(중앙)** 으로 가서:

1. 밭 칸에 `E`로 괭이질·심기·물주기·수확을 할 때마다 좌상단 **`혼력: NN/100`이 10씩 줄어듦** → 행동당 소모 OK.
2. 계속 행동해 **혼력이 0이 되면** HUD에 `지쳤다(취침 필요)`가 뜨고, 밭 칸을 봐도 `[E] …` 대신 `혼력 부족 — 집에서 취침` 안내가 뜨며 **`E`를 눌러도 동작이 안 일어남** → 0이면 막힘 OK.
3. 집으로 가 **취침**하면 다음 날 아침 **혼력이 100으로 가득** 차고 다시 행동 가능 → 취침 회복 OK.

위 3개가 되면 **T2.4 완료.** (혼력 로직은 헤드리스 단위검증으로도 통과: 초기 가득 / 행동 10회=가득→0 / 0에서 행동 막힘·음수 방지 / `depleted` 1회 / 취침 회복 / `changed` 발화 수. 통합 빌드 임포트 에러·경고 0.)

## T2.3 — 작물 성장(일수 경과)

심은 작물이 **날이 지나며 단계가 오르고, 다 자라면 수확 가능 표시가 뜬다.** 성장 규칙은 스타듀식 — **물 준 칸만 다음 날 자라고, 매일 아침 흙이 마른다.** T2.2 작물 카탈로그를 T2.1 밭 위에 올려 "심고-물주고-키워서 수확"하는 한 사이클을 완성한다.

| 요소 | 구현 | 메모 |
|---|---|---|
| 성장 상태 | `field.gd` 칸에 `crop`(작물 id) / `grown_days`(누적 성장일) 추가 | "어디에 뭐가 심겼고 며칠 자랐나"라는 세이브 상태만. 성장일수 등은 `CropCatalog`에서 조회(데이터/상태 분리) |
| 성장 트리거 | `GameClock.day_advanced` → `FarmField.advance_day()` | 시그널 디커플링(시계는 코드 수정 없이 밭을 구동). 취침 시 발화 |
| 성장 규칙 | 물 준(`watered`) 칸만 `grown_days +1`, 그 뒤 모든 칸은 아침에 마름 | 스타듀 규칙. 물 안 준 칸은 안 자람 |
| 수확 | `harvest()` — 다 자란 칸을 거두고 빈 경작 칸으로 리셋, 거둔 **작물 id 반환** | 골드 지급은 T3.1 경제가 이 반환값으로 처리(범위 분리) |
| 단일 키 흐름 | `next_action`에 **수확** 추가: 괭이질→심기→물주기→(자람)→수확 | `E` 한 키가 칸 상태에 맞는 다음 동작 수행 |
| 작물 선택 | `Q`로 카탈로그(빠른 성장 순) 순환, 선택 작물을 심음 | 그레이박스라 씨앗 인벤토리 UI 없이 한 변수로. 상단 HUD에 `심을 작물: 이름(N일)` 표시 |
| 시각화 | 오버레이 8타일(외형 4 × 젖음 2): 빈 고랑 / 씨앗 / 새싹 / **수확가능(황금)** | 외형단계는 `growth_stage`(씨앗0/새싹1/수확가능2). 다 자라면 가운데 새싹이 **황금색** = 수확 가능 표시 |

### 완료기준 직접 확인하기

`godot --path game`로 띄운 뒤 **밭(중앙)** 으로 가서:

1. `Q`로 작물을 고르고(상단 HUD에 이름·성장일수 표시), 한 칸에서 **괭이질→심기→물주기**까지 진행.
2. 집으로 가 **취침**을 반복하면, 그 칸에 **물을 준 다음 날에만** 새싹이 커진다(물 안 주면 그대로) → 성장 규칙 OK.
3. 작물 성장일수(혼령초 3 < 피안화 5 < 영혼 호박 8)만큼 키우면 새싹이 **황금색**으로 바뀌고 `[E] 수확` 안내가 뜸 → **수확 가능 표시** OK.
4. `E`로 수확하면 칸이 다시 빈 경작 칸으로 돌아가 또 심을 수 있음 → 한 사이클 OK.

위가 되면 **T2.3 완료.** (성장 로직은 헤드리스 단위검증으로도 통과: 물 준 칸만 성장 / 아침에 마름 / 성장일수 도달 시 수확 가능 / 작물별 속도 차이 / 수확 후 리셋 / 잘못된 id 거부. 통합 빌드 임포트 에러·경고 0.)

## T2.2 — 저승 작물 3종 데이터 정의

저승 작물 3종이 **데이터로** 정의되고, **성장일수가 빠름 < 중간 < 느림**으로 구분되는지 보장한다(CONTEXT '저승 작물' 용어 그대로). 이건 순수 데이터 작업이다 — 작물을 밭에 심고 키우는 **연결은 T2.3(작물 성장)**에서 이 카탈로그 위에 붙는다(범위 분리).

| 작물 | id | 성장일수 | 단계 | 씨앗가 → 판매가 (순익) |
|---|---|---|---|---|
| **혼령초** | `honryeongcho` | **3일** (빠름) | 2 | 10G → 20G (**+10G**, 저수익) |
| **피안화** | `pianhwa` | **5일** (중간) | 3 | 25G → 60G (+35G, 중간) |
| **영혼 호박** | `yeonghon_hobak` | **8일** (느림) | 4 | 50G → 160G (**+110G**, 고수익) |

| 요소 | 구현 | 메모 |
|---|---|---|
| 카탈로그 | `crops.gd`(`CropCatalog`) — "작물 정의"라는 **단일 책임**만 | `clock.gd`·`field.gd`와 같은 결. 단 **정적 참조 데이터**라 씬 노드가 아니라 `static const` + `class_name`(오토로드 불필요) |
| 식별자 분리 | 영문 `id`(코드·세이브용) ↔ `name_ko`(표시용) | T2.5 세이브엔 가벼운 영문 id만 저장 |

| 수익 차등 | `seed_cost` / `sell_price` | ROADMAP의 '저수익/고수익'을 데이터로 박음. 그레이박스 기준값, 밸런싱은 후속 |
| 조회 API | `ids()` / `get_crop()` / `name_of()` / `growth_days()` / `has_crop()` | 없는 id는 빈 Dictionary·`""`·`-1`로 안전 처리 |

### 완료기준 직접 확인하기

작물 데이터는 화면 요소가 아니라 **데이터**라, 헤드리스 단위검증으로 확인한다(`crops.gd`는 아직 어느 씬에도 붙지 않는다 — T2.3에서 밭에 연결). `game/`에서 임시 검증 스크립트를 띄우면:

1. **3종이 모두 정의**되고(`ids().size() == 3`),
2. **성장일수가 `혼령초(3) < 피안화(5) < 영혼 호박(8)`**으로 정렬되며,
3. 순익이 **저수익(+10) < 고수익(+110)**으로 벌어진다.

위 3개가 되면 **T2.2 완료.** (전역 `class_name CropCatalog` 등록 + 전체 빌드 임포트 에러·경고 0 확인.)

## T2.1 — 밭 칸 상호작용: 괭이질 → 심기 → 물주기

한 칸에서 **괭이질→심기→물주기가 순서대로** 되고, 칸 상태(미경작/경작/심김/젖음)가 회색 도형만으로 시각적으로 바뀌는지 검증한다(ADR-0001). 작물 종류·성장은 아직 없다 — T2.2(작물 데이터)·T2.3(성장)이 이 위에 붙는다.

| 요소 | 구현 | 메모 |
|---|---|---|
| 밭 상태 | `field.gd`(`FarmField` 노드) — "밭 칸 상태"라는 **단일 책임**만 | `clock.gd`와 같은 결. 상태는 순수 `Dictionary`(경작된 칸만)라 T2.5 세이브에 그대로 직렬화됨 |
| 시그널 | `tile_changed(tile)` | `main`이 듣고 오버레이 타일을 갱신(디커플링). T2.3 성장이 붙을 훅 |
| 순서 강제 | `next_action()` — 경작 전 심기·심기 전 물주기 불가 | 완료기준 "순서대로"를 로직이 보장 |
| 입력 | **앞 칸을 바라보고 `E`** 한 키 → 칸 상태에 맞는 다음 동작 수행 | `interact` 액션은 코드로 런타임 등록(키 E). 도구 전환 UI는 후속 |
| 대상 칸 | 플레이어 발 타일 + 바라보는 방향 한 칸(대각선은 4방향 스냅) | 밭 흙(SOIL) 칸만 상호작용. **흰 테두리 커서**로 강조 |
| 시각화 | `Field` 오버레이 `TileMapLayer`(코드 TileSet 4종) | 경작(마른 고랑)/경작+젖음(어둡게)/심김(새싹)/심김+젖음. 미경작은 오버레이 없음(맨 흙) |
| UI | 밭 칸을 볼 때 하단 `[E] 괭이질/심기/물주기` 안내 | 다 키운(물준) 칸이면 숨김 |

### 완료기준 직접 확인하기

`godot --path game`로 띄운 뒤 길을 따라 **밭(중앙)** 으로 가서:

1. 밭 흙 칸을 바라보면 **흰 커서**와 하단 `[E] 괭이질` 안내가 뜸 → 대상 판정 OK.
2. `E`를 누를 때마다 **괭이질(고랑)→심기(새싹)→물주기(어두운 젖은 흙)** 로 칸이 바뀌고, 안내 문구도 다음 동작으로 갱신 → 순서·시각 변화 OK.
3. 길·집·카페·벽 칸을 바라보면 커서·안내가 **안 뜸**(밭만 경작) → 범위 한정 OK.

위 3개가 되면 **T2.1 완료.** (`FarmField` 로직은 헤드리스 단위검증으로도 통과: 순서 강제 / 각 동작 / `tile_changed` 발화.)

## T1.5 — 하루 사이클 + 취침 진행

회색 UI만으로 **시간이 흐르고, 취침하면 날짜가 +1 되며 시간이 아침으로 리셋되는지** 검증한다(ADR-0001). 이 시계는 ROADMAP 메모대로 이후 **혼력 회복(T2.4)·작물 성장(T2.3)의 트리거**가 되며, 그 시스템들이 코드 수정 없이 붙을 수 있도록 시그널을 미리 노출한다.

| 요소 | 구현 | 메모 |
|---|---|---|
| 시계 | `clock.gd`(`GameClock` 노드) — 시간이라는 **단일 책임**만 | 06:00 시작 → 24:00 도달 시 "쓰러짐"(강제 취침). 실제 90초에 하루(18시간)를 흘림 |
| 시그널 | `minute_ticked` / `day_advanced` / `collapsed` | 미래 시스템(혼력·작물)이 연결할 **훅**. 지금은 `collapsed`만 main이 사용 |
| 취침 | **집 구역에서 Enter/Space** → 페이드→날짜+1→06:00 리셋→페이드 | 그레이박스라 침대 오브젝트 없이 "집에 있으면 잔다"로 단순화(가구는 Phase 2) |
| 강제 취침 | 24:00 도달 시 위치 무관 자동 취침 | 스타듀식 쓰러짐 |
| UI | 우상단 `Day N  HH:MM  아침/낮/저녁/밤` + 집에서 `[Enter] 취침` 안내 | 회색 라벨만 |
| 연출 | `Fade` ColorRect(`CanvasLayer`)를 Tween으로 페이드 + 연출 중 이동 잠금 | 카메라와 무관하게 전체 화면 덮음 |

### 완료기준 직접 확인하기

`godot --path game`로 띄운 뒤:

1. 가만히 있어도 우상단 **시간이 흐르고** 단계(아침→낮→저녁→밤)가 바뀜 → 시간 흐름 OK.
2. **집** 구역에 들어가면 하단 `[Enter] 취침` 안내가 뜨고, 누르면 화면이 어두워졌다 밝아지며 **Day가 +1, 시간이 06:00 아침으로 리셋** → 취침 진행 OK.
3. 자지 않고 버티면 **24:00에 자동으로 쓰러져** 다음 날 아침이 됨 → 강제 취침 OK.

위 3개가 되면 **T1.5 완료.** (시계 로직은 헤드리스 단위검증으로도 통과: 06:00 시작 / 시간 경과 / 24:00 collapsed·정지 / 취침 후 Day+1·06:00·재시작.)

## T1.2 — 320×180 정수배 스케일 뷰포트

이후 **모든 화면 작업의 기준**이 되는 픽셀아트 뷰포트 설정(ADR-0003).

| 설정 (`project.godot`) | 값 | 이유 |
|---|---|---|
| `display/window/size/viewport_*` | 320×180 | 내부(게임) 해상도. 16:9 |
| `window_*_override` | 1280×720 | 시작 창 = ×4 |
| `window/stretch/mode` | `viewport` | 320×180을 통째로 렌더 후 확대 → 순수 픽셀룩 |
| `window/stretch/aspect` | `keep` | 비율 유지(남는 공간 레터박스) |
| **`window/stretch/scale_mode`** | **`integer`** | **★ 정수배(×2/×3/…)로만 확대 — 1.5배 같은 분수배 차단** |
| `textures/.../default_texture_filter` | `0`(Nearest) | 보간 끔 → 픽셀이 흐려지지 않음 |
| `2d/snap/snap_2d_*` | `true` | 좌표를 픽셀 격자에 스냅 |

### 완료기준 직접 확인하기

`godot --path game`로 띄운 뒤 **창을 자유롭게 늘려보라.**

1. **체커보드 16×16 칸**이 늘 정확한 정사각형 → 정수배 스케일 OK.
2. **대각선**이 흐릿한 번짐 없이 또렷한 "계단"으로 → Nearest 보간 OK.
3. **남는 공간**이 검은 레터박스로 처리되고, 좌상단 readout의 **"배율 x__"가 정수로만** 바뀜 → `scale_mode=integer` 작동 중.

분수배 + 선형보간이었다면 대각선과 1px 외곽선부터 뭉개진다. 위 3개가 모두 깨끗하면 **T1.2 완료.**

> 위 체커보드 검증 씬은 **T1.3에서 캐릭터 이동 씬으로 대체**되었다(체커보드 코드는 T1.2 커밋의 git 히스토리에 보존). 정수배 스케일 동작은 이동 씬의 16px 격자로도 동일하게 확인된다.

## T1.4 — 16×16 타일맵 더미 맵 1개

회색 캐릭터가 **진짜 타일맵**(`TileMapLayer`) 위를 돌아다니며 **밭·집·카페** 세 구역을 시각적으로 구분할 수 있는지 검증한다. 회색 도형만(ADR-0001). 온보딩 동선(CONTEXT '온보딩': 도착→집→밭→카페)을 염두에 둔 배치.

| 요소 | 구현 | 메모 |
|---|---|---|
| 타일맵 | `TileMapLayer` + **코드로 조립한 TileSet** | 에셋 없이(ADR-0001) 단색 16×16 블록 아틀라스를 코드 생성. "도트화 툴"이 아니라 그레이박스 플레이스홀더 |
| 타일 6종 | GROUND/PATH/SOIL/HOUSE/CAFE/WALL | 밝기·미세 색조로만 구분(회색 기조). 각 타일 위/왼쪽 1px 격자선으로 타일감 표시 |
| 맵 크기 | 40×24 타일 = 640×384px | 화면(320×180)보다 커서 **카메라가 캐릭터를 따라가며** 맵 경계에서 멈춤 → "돌아다니며" |
| 구역 | 집·카페 = 벽으로 둘러싼 방(문 1칸) / 밭 = 벽 없는 열린 흙 / 길 = 동선 허브 | 구역 위에 떠 있는 라벨(집/밭/카페/도착)로 식별 |
| 충돌 | WALL 타일에만 충돌 폴리곤(TileSet 물리 레이어) | 맵 4변 경계벽 + 건물 외벽. 문으로만 출입 |
| readout | 현재 구역·위치·FPS | `_zone_at()`이 발 위치 타일로 집/밭/카페/바깥 판정 |

### 완료기준 직접 확인하기

`godot --path game`로 띄운 뒤 **방향키로 돌아다녀 보라.**

1. 도착 지점에서 길을 따라 올라가면 **집(좌)·밭(중앙)·카페(우)** 세 구역이 색·라벨로 **분명히 구분** → 시각 구분 OK.
2. 집·카페는 **벽으로 막혀 문으로만** 드나들고, 맵 밖으로는 못 나감(경계벽) → 타일 충돌 OK.
3. 각 구역에 들어가면 좌상단 readout의 **"구역: ___"**이 바뀜 → 구역 판정 OK.

위 3개가 모두 되면 **T1.4 완료.**

## T1.3 — 캐릭터 그레이박스 이동 + 충돌

회색 캐릭터(16×32 자리)가 탑다운 4방향으로 **부드럽게** 움직이고 **벽을 통과하지 못하는지** 검증한다. 회색 도형만(ADR-0001).

| 요소 | 구현 | 메모 |
|---|---|---|
| 캐릭터 | `CharacterBody2D` + `move_and_slide()` | origin은 **발치 중앙**(탑다운 깊이 정렬·충돌 기준에 유리) |
| 이동 입력 | `Input.get_vector(ui_left/right/up/down)` | **대각선 자동 정규화** → 대각선이 더 빨라지지 않음. 현재 **방향키**(WASD는 후속) |
| 속도 | `SPEED = 80 px/s` (약 5타일/초) | 그레이박스 기준값. 밸런싱은 후속 |
| 충돌체 | 캐릭터 = 발치 14×10 박스 / 벽 = `StaticBody2D`(코드 생성) | 벽 데이터는 `main.gd`의 `_walls: Array[Rect2]` |
| 배경 | 16px 격자(이동량 가늠) + 회색 벽 + 위치/FPS readout | 임시 도형. 본격 타일맵 더미 맵은 **T1.4** |

### 완료기준 직접 확인하기

`godot --path game`로 띄운 뒤 **방향키로 움직여보라.**

1. 캐릭터가 4방향으로 **끊김 없이 부드럽게** 이동 → 이동 OK.
2. **대각선**으로 가도 상하/좌우보다 빨라지지 않음 → 정규화 OK.
3. 화면 4변 경계벽·내부 장애물에 닿으면 **멈추거나 벽을 따라 미끄러짐**(통과 불가) → 충돌 OK.

위 3개가 모두 되면 **T1.3 완료.**

## 파일

- `project.godot` — 뷰포트/렌더 설정(위 표).
- `main.tscn` / `main.gd` — 더미 맵 월드: 코드로 `TileMapLayer`/TileSet 조립, 구역 배치, 추적 카메라, readout + **하루 사이클 연동(취침 입력·페이드 연출·시계 UI)** + **밭 상호작용(오버레이 타일·커서·`E` 입력)** + **혼력 연동(행동당 소모·취침 회복·HUD)** + **세이브/로드 조율(상태 수집·분배·`F5`/`F9`·자동 저장/로드)**. 회색 도형만.
- `clock.gd` — `GameClock` 노드: 하루 시계(날짜·시간 상태 + 시그널). 화면 연출은 `main.gd`가 맡고 여기는 시간만.
- `field.gd` — `FarmField` 노드: 밭 칸 상태(미경작/경작/심김/젖음)와 `tile_changed` 시그널. 화면 표시·입력은 `main.gd`가 맡고 여기는 상태만.
- `energy.gd` — `SoulEnergy` 노드: 혼력 상태(행동당 소모·취침 회복)와 `changed`/`depleted` 시그널. 소모 트리거(밭 상호작용)·회복 트리거(취침)·HUD는 `main.gd`가 맡고 여기는 상태만.
- `crops.gd` — `CropCatalog`: 저승 작물 3종(혼령초/피안화/영혼 호박) 정적 데이터 정의(성장일수·수익). 씬 노드가 아니라 `static const` 카탈로그. T2.3 성장이 이 위에 붙는다.
- `save.gd` — `SaveManager` 노드: 세이브 파일 입출력(`user://save.dat` 단일 슬롯, `var_to_str` 직렬화·버전 래핑·손상 방어). 무엇을 저장할지 고르는 조율은 `main.gd`, 각 노드의 `to_save()`/`load_save()`가 자기 상태를 직렬화. T3.1 골드는 여기 손 안 대고 `main`에 한 줄만 추가하면 붙는다.
- `affinity.gd` — `Affinity` 노드: 미호 호감도(점수·하트 단계)와 `changed` 시그널. 두 채널(일일 대화·선물)은 게임 날짜로 하루 1회 게이팅, 트리거·HUD·여우불 연동(T3.4)은 `main`이 맡고 여기는 상태만.
- `player.tscn` / `player.gd` — 플레이어(`CharacterBody2D`, 16×32 회색, 4방향 이동).
