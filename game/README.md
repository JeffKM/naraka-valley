# game — 나라카 밸리 게임 본체

스타듀밸리형 본편 게임(Godot 4.6 / GL Compatibility). **학습용인 [`../godot-sandbox/`](../godot-sandbox/)·[`../godot-tutorial/`](../godot-tutorial/)와 달리 이쪽이 진짜 게임이다.** Phase 1 Sprint 1에서 시작한 본체 프로젝트.

> ⚠️ Phase 1은 **회색 도형(그레이박스)만** 쓴다. 도트·초상화·사운드는 재미 게이트 통과 후 Phase 2에서만 입힌다(ADR-0001).

## 여는 법 / 실행

```bash
godot --path game -e   # 에디터로 열기
godot --path game      # 바로 실행
```

또는 Godot 실행 → **Import** → `game/project.godot` 선택 → **F5**.

## T2.2 — 저승 작물 3종 데이터 정의 (현재 산출물)

저승 작물 3종이 **데이터로** 정의되고, **성장일수가 빠름 < 중간 < 느림**으로 구분되는지 보장한다(CONTEXT '저승 작물' 용어 그대로). 이건 순수 데이터 작업이다 — 작물을 밭에 심고 키우는 **연결은 T2.3(작물 성장)**에서 이 카탈로그 위에 붙는다(범위 분리).

| 작물 | id | 성장일수 | 단계 | 씨앗가 → 판매가 (순익) |
|---|---|---|---|---|
| **혼령초** | `honryeongcho` | **3일** (빠름) | 2 | 10G → 20G (**+10G**, 저수익) |
| **피안화** | `pianhwa` | **5일** (중간) | 3 | 25G → 60G (+35G, 중간) |
| **영혼 호박** | `yeonghon_hobak` | **8일** (느림) | 4 | 50G → 160G (**+110G**, 고수익) |

| 요소 | 구현 | 메모 |
|---|---|---|
| 카탈로그 | `crops.gd`(`CropCatalog`) — "작물 정의"라는 **단일 책임**만 | `clock.gd`·`field.gd`와 같은 결. 단 **정적 참조 데이터**라 씬 노드가 아니라 `static const` + `class_name`(오토로드 불필요) |
| 식별자 분리 | 영문 `id`(코드·세이브용) ↔ `name_ko`(표시용) | T2.5 세이브엔 가벼운 영문 id만 저장 |
| 성장일수 | `growth_days` 빠름(3) < 중간(5) < 느림(8) | **완료기준.** T2.3가 `GameClock.day_advanced` 카운트와 비교해 단계를 올릴 값 |
| 수익 차등 | `seed_cost` / `sell_price` | ROADMAP의 '저수익/고수익'을 데이터로 박음. 그레이박스 기준값, 밸런싱은 후속 |
| 조회 API | `ids()` / `get_crop()` / `name_of()` / `growth_days()` / `has_crop()` | 없는 id는 빈 Dictionary·`""`·`-1`로 안전 처리 |

### 완료기준 직접 확인하기

작물 데이터는 화면 요소가 아니라 **데이터**라, 헤드리스 단위검증으로 확인한다(`crops.gd`는 아직 어느 씬에도 붙지 않는다 — T2.3에서 밭에 연결). `game/`에서 임시 검증 스크립트를 띄우면:

1. **3종이 모두 정의**되고(`ids().size() == 3`),
2. **성장일수가 `혼령초(3) < 피안화(5) < 영혼 호박(8)`**으로 정렬되며,
3. 순익이 **저수익(+10) < 고수익(+110)**으로 벌어진다.

위 3개가 되면 **T2.2 완료.** (전역 `class_name CropCatalog` 등록 + 전체 빌드 임포트 에러·경고 0 확인.)

## T2.1 — 밭 칸 상호작용: 괭이질 → 심기 → 물주기

한 칸에서 **괭이질→심기→물주기가 순서대로** 되고, 칸 상태(미경작/경작/심김/젖음)가 회색 도형만으로 시각적으로 바뀌는지 검증한다(ADR-0001). 작물 종류·성장은 아직 없다 — T2.2(작물 데이터)·T2.3(성장)이 이 위에 붙는다.

| 요소 | 구현 | 메모 |
|---|---|---|
| 밭 상태 | `field.gd`(`FarmField` 노드) — "밭 칸 상태"라는 **단일 책임**만 | `clock.gd`와 같은 결. 상태는 순수 `Dictionary`(경작된 칸만)라 T2.5 세이브에 그대로 직렬화됨 |
| 시그널 | `tile_changed(tile)` | `main`이 듣고 오버레이 타일을 갱신(디커플링). T2.3 성장이 붙을 훅 |
| 순서 강제 | `next_action()` — 경작 전 심기·심기 전 물주기 불가 | 완료기준 "순서대로"를 로직이 보장 |
| 입력 | **앞 칸을 바라보고 `E`** 한 키 → 칸 상태에 맞는 다음 동작 수행 | `interact` 액션은 코드로 런타임 등록(키 E). 도구 전환 UI는 후속 |
| 대상 칸 | 플레이어 발 타일 + 바라보는 방향 한 칸(대각선은 4방향 스냅) | 밭 흙(SOIL) 칸만 상호작용. **흰 테두리 커서**로 강조 |
| 시각화 | `Field` 오버레이 `TileMapLayer`(코드 TileSet 4종) | 경작(마른 고랑)/경작+젖음(어둡게)/심김(새싹)/심김+젖음. 미경작은 오버레이 없음(맨 흙) |
| UI | 밭 칸을 볼 때 하단 `[E] 괭이질/심기/물주기` 안내 | 다 키운(물준) 칸이면 숨김 |

### 완료기준 직접 확인하기

`godot --path game`로 띄운 뒤 길을 따라 **밭(중앙)** 으로 가서:

1. 밭 흙 칸을 바라보면 **흰 커서**와 하단 `[E] 괭이질` 안내가 뜸 → 대상 판정 OK.
2. `E`를 누를 때마다 **괭이질(고랑)→심기(새싹)→물주기(어두운 젖은 흙)** 로 칸이 바뀌고, 안내 문구도 다음 동작으로 갱신 → 순서·시각 변화 OK.
3. 길·집·카페·벽 칸을 바라보면 커서·안내가 **안 뜸**(밭만 경작) → 범위 한정 OK.

위 3개가 되면 **T2.1 완료.** (`FarmField` 로직은 헤드리스 단위검증으로도 통과: 순서 강제 / 각 동작 / `tile_changed` 발화.)

## T1.5 — 하루 사이클 + 취침 진행

회색 UI만으로 **시간이 흐르고, 취침하면 날짜가 +1 되며 시간이 아침으로 리셋되는지** 검증한다(ADR-0001). 이 시계는 ROADMAP 메모대로 이후 **혼력 회복(T2.4)·작물 성장(T2.3)의 트리거**가 되며, 그 시스템들이 코드 수정 없이 붙을 수 있도록 시그널을 미리 노출한다.

| 요소 | 구현 | 메모 |
|---|---|---|
| 시계 | `clock.gd`(`GameClock` 노드) — 시간이라는 **단일 책임**만 | 06:00 시작 → 24:00 도달 시 "쓰러짐"(강제 취침). 실제 90초에 하루(18시간)를 흘림 |
| 시그널 | `minute_ticked` / `day_advanced` / `collapsed` | 미래 시스템(혼력·작물)이 연결할 **훅**. 지금은 `collapsed`만 main이 사용 |
| 취침 | **집 구역에서 Enter/Space** → 페이드→날짜+1→06:00 리셋→페이드 | 그레이박스라 침대 오브젝트 없이 "집에 있으면 잔다"로 단순화(가구는 Phase 2) |
| 강제 취침 | 24:00 도달 시 위치 무관 자동 취침 | 스타듀식 쓰러짐 |
| UI | 우상단 `Day N  HH:MM  아침/낮/저녁/밤` + 집에서 `[Enter] 취침` 안내 | 회색 라벨만 |
| 연출 | `Fade` ColorRect(`CanvasLayer`)를 Tween으로 페이드 + 연출 중 이동 잠금 | 카메라와 무관하게 전체 화면 덮음 |

### 완료기준 직접 확인하기

`godot --path game`로 띄운 뒤:

1. 가만히 있어도 우상단 **시간이 흐르고** 단계(아침→낮→저녁→밤)가 바뀜 → 시간 흐름 OK.
2. **집** 구역에 들어가면 하단 `[Enter] 취침` 안내가 뜨고, 누르면 화면이 어두워졌다 밝아지며 **Day가 +1, 시간이 06:00 아침으로 리셋** → 취침 진행 OK.
3. 자지 않고 버티면 **24:00에 자동으로 쓰러져** 다음 날 아침이 됨 → 강제 취침 OK.

위 3개가 되면 **T1.5 완료.** (시계 로직은 헤드리스 단위검증으로도 통과: 06:00 시작 / 시간 경과 / 24:00 collapsed·정지 / 취침 후 Day+1·06:00·재시작.)

## T1.2 — 320×180 정수배 스케일 뷰포트

이후 **모든 화면 작업의 기준**이 되는 픽셀아트 뷰포트 설정(ADR-0003).

| 설정 (`project.godot`) | 값 | 이유 |
|---|---|---|
| `display/window/size/viewport_*` | 320×180 | 내부(게임) 해상도. 16:9 |
| `window_*_override` | 1280×720 | 시작 창 = ×4 |
| `window/stretch/mode` | `viewport` | 320×180을 통째로 렌더 후 확대 → 순수 픽셀룩 |
| `window/stretch/aspect` | `keep` | 비율 유지(남는 공간 레터박스) |
| **`window/stretch/scale_mode`** | **`integer`** | **★ 정수배(×2/×3/…)로만 확대 — 1.5배 같은 분수배 차단** |
| `textures/.../default_texture_filter` | `0`(Nearest) | 보간 끔 → 픽셀이 흐려지지 않음 |
| `2d/snap/snap_2d_*` | `true` | 좌표를 픽셀 격자에 스냅 |

### 완료기준 직접 확인하기

`godot --path game`로 띄운 뒤 **창을 자유롭게 늘려보라.**

1. **체커보드 16×16 칸**이 늘 정확한 정사각형 → 정수배 스케일 OK.
2. **대각선**이 흐릿한 번짐 없이 또렷한 "계단"으로 → Nearest 보간 OK.
3. **남는 공간**이 검은 레터박스로 처리되고, 좌상단 readout의 **"배율 x__"가 정수로만** 바뀜 → `scale_mode=integer` 작동 중.

분수배 + 선형보간이었다면 대각선과 1px 외곽선부터 뭉개진다. 위 3개가 모두 깨끗하면 **T1.2 완료.**

> 위 체커보드 검증 씬은 **T1.3에서 캐릭터 이동 씬으로 대체**되었다(체커보드 코드는 T1.2 커밋의 git 히스토리에 보존). 정수배 스케일 동작은 이동 씬의 16px 격자로도 동일하게 확인된다.

## T1.4 — 16×16 타일맵 더미 맵 1개

회색 캐릭터가 **진짜 타일맵**(`TileMapLayer`) 위를 돌아다니며 **밭·집·카페** 세 구역을 시각적으로 구분할 수 있는지 검증한다. 회색 도형만(ADR-0001). 온보딩 동선(CONTEXT '온보딩': 도착→집→밭→카페)을 염두에 둔 배치.

| 요소 | 구현 | 메모 |
|---|---|---|
| 타일맵 | `TileMapLayer` + **코드로 조립한 TileSet** | 에셋 없이(ADR-0001) 단색 16×16 블록 아틀라스를 코드 생성. "도트화 툴"이 아니라 그레이박스 플레이스홀더 |
| 타일 6종 | GROUND/PATH/SOIL/HOUSE/CAFE/WALL | 밝기·미세 색조로만 구분(회색 기조). 각 타일 위/왼쪽 1px 격자선으로 타일감 표시 |
| 맵 크기 | 40×24 타일 = 640×384px | 화면(320×180)보다 커서 **카메라가 캐릭터를 따라가며** 맵 경계에서 멈춤 → "돌아다니며" |
| 구역 | 집·카페 = 벽으로 둘러싼 방(문 1칸) / 밭 = 벽 없는 열린 흙 / 길 = 동선 허브 | 구역 위에 떠 있는 라벨(집/밭/카페/도착)로 식별 |
| 충돌 | WALL 타일에만 충돌 폴리곤(TileSet 물리 레이어) | 맵 4변 경계벽 + 건물 외벽. 문으로만 출입 |
| readout | 현재 구역·위치·FPS | `_zone_at()`이 발 위치 타일로 집/밭/카페/바깥 판정 |

### 완료기준 직접 확인하기

`godot --path game`로 띄운 뒤 **방향키로 돌아다녀 보라.**

1. 도착 지점에서 길을 따라 올라가면 **집(좌)·밭(중앙)·카페(우)** 세 구역이 색·라벨로 **분명히 구분** → 시각 구분 OK.
2. 집·카페는 **벽으로 막혀 문으로만** 드나들고, 맵 밖으로는 못 나감(경계벽) → 타일 충돌 OK.
3. 각 구역에 들어가면 좌상단 readout의 **"구역: ___"**이 바뀜 → 구역 판정 OK.

위 3개가 모두 되면 **T1.4 완료.**

## T1.3 — 캐릭터 그레이박스 이동 + 충돌

회색 캐릭터(16×32 자리)가 탑다운 4방향으로 **부드럽게** 움직이고 **벽을 통과하지 못하는지** 검증한다. 회색 도형만(ADR-0001).

| 요소 | 구현 | 메모 |
|---|---|---|
| 캐릭터 | `CharacterBody2D` + `move_and_slide()` | origin은 **발치 중앙**(탑다운 깊이 정렬·충돌 기준에 유리) |
| 이동 입력 | `Input.get_vector(ui_left/right/up/down)` | **대각선 자동 정규화** → 대각선이 더 빨라지지 않음. 현재 **방향키**(WASD는 후속) |
| 속도 | `SPEED = 80 px/s` (약 5타일/초) | 그레이박스 기준값. 밸런싱은 후속 |
| 충돌체 | 캐릭터 = 발치 14×10 박스 / 벽 = `StaticBody2D`(코드 생성) | 벽 데이터는 `main.gd`의 `_walls: Array[Rect2]` |
| 배경 | 16px 격자(이동량 가늠) + 회색 벽 + 위치/FPS readout | 임시 도형. 본격 타일맵 더미 맵은 **T1.4** |

### 완료기준 직접 확인하기

`godot --path game`로 띄운 뒤 **방향키로 움직여보라.**

1. 캐릭터가 4방향으로 **끊김 없이 부드럽게** 이동 → 이동 OK.
2. **대각선**으로 가도 상하/좌우보다 빨라지지 않음 → 정규화 OK.
3. 화면 4변 경계벽·내부 장애물에 닿으면 **멈추거나 벽을 따라 미끄러짐**(통과 불가) → 충돌 OK.

위 3개가 모두 되면 **T1.3 완료.**

## 파일

- `project.godot` — 뷰포트/렌더 설정(위 표).
- `main.tscn` / `main.gd` — 더미 맵 월드: 코드로 `TileMapLayer`/TileSet 조립, 구역 배치, 추적 카메라, readout + **하루 사이클 연동(취침 입력·페이드 연출·시계 UI)** + **밭 상호작용(오버레이 타일·커서·`E` 입력)**. 회색 도형만.
- `clock.gd` — `GameClock` 노드: 하루 시계(날짜·시간 상태 + 시그널). 화면 연출은 `main.gd`가 맡고 여기는 시간만.
- `field.gd` — `FarmField` 노드: 밭 칸 상태(미경작/경작/심김/젖음)와 `tile_changed` 시그널. 화면 표시·입력은 `main.gd`가 맡고 여기는 상태만.
- `crops.gd` — `CropCatalog`: 저승 작물 3종(혼령초/피안화/영혼 호박) 정적 데이터 정의(성장일수·수익). 씬 노드가 아니라 `static const` 카탈로그. T2.3 성장이 이 위에 붙는다.
