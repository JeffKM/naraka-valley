# 광원 일관성 감사 — Slice 0 Phase C ⑥ (2026-07-01)

> [asset-ruleset.md §1](./asset-ruleset.md) NW 광원 / SE 그림자 규칙에 대한 **기존 대형 에셋 전수 감사**.
> 방법 = §1.2 **2단계 필터(완주 우선, [ADR-0028])** — 1단계는 대형 에셋만 즉시 검수·교정, 2단계(소형 ≤1타일)는 패스하고 신규 생성분부터 통제.

## 방법(정량 + 육안)

- **정량:** 불투명 픽셀만 대상으로 **좌열 vs 우열 평균 명도**(진짜 수평 광원 방향 — NW면 좌(L) > 우(R))를 측정. 보조로 TL vs BR 사분면·코너 알파.
  - ⚠️ **사분면(TL/BR) 단독은 오탐** — 건물은 *지붕(위·어두운 재질) vs 벽(아래·밝은 재질)* 재질 구간차를 광원 위반으로 잘못 잡는다. **좌-우 비교가 광원 방향의 참 지표.**
- **육안:** 대형 에셋 콘택트 시트(`/tmp/audit_sheet.png` — 재현: `tools/` 글루)로 하이라이트/그림자 방향 확인.

# z-stack 레이어 순서 감사 — Phase C ⑤(§13) (2026-07-01)

> [asset-ruleset §13](./asset-ruleset.md): **그림자 < 지형 < 밭 < 프롭(플레이어 Y-Sort §6) < 지붕 오버행 < 발광(§8.3) < UI.**

## 실측 렌더 순서(아래→위)

| 레이어 | 근거 |
|---|---|
| 지형(Ground)·밭(Field) | `z_index = -1` (Field가 트리 후순위라 Ground 위) |
| 지면 디테일 → facade → **뒤 프롭(+인라인 접지 그림자)** → 작물 | `main._draw()` z=0(부모 즉시모드, 이 순서) |
| **플레이어** | `$Player` z=0 자식(부모 _draw 위) |
| **앞 프롭(+그림자)** | `_front_props` z=1 (플레이어 위 — §6 Y-split) |
| 카메라 라벨 | z=10 |
| HUD·대화·패널 | `$CanvasLayer/*` (월드 위) |

## 판정

- **지형 < 밭 < 프롭 < 플레이어(Y-Sort) < UI = ✅ 정합.** 뒤 프롭 z0(플레이어 아래)·앞 프롭 z1(위) 샌드위치가 §6 Y-split을 실현.
- **그림자:** 각 프롭 바로 앞(밑)에 인라인 → *지형 위·객체 아래*. §11 "별도 오버레이·객체 아래" 실동작 정확. ⚠️ §13 문구 "그림자 < 지형"은 문자 그대로면 불투명 지형에 가려 **안 보임** → **실무 해석 = 지형 위/객체 아래**가 규칙의 참 의도(문서 표현이 압축적일 뿐). 코드가 옳음.
- **지붕 오버행·야간 발광 = 미구현(향후).** 현재 facade는 z0(프롭·플레이어 아래 층) — 남향 정면 진입이라 플레이어가 늘 건물 앞이 정상(§2). *별도 지붕 오버행 레이어*(건물 북쪽으로 플레이어가 지날 때 처마가 가림)와 *야간 발광 마스크*(§8.3 `*_emit`)는 **Phase 3+/S1-11 도입 예약** — 회귀 아님.

**결론: 구현된 전 레이어 순서 §13 정합, 재정렬 0.** ⑤ 추가분(front_props z1·인라인 접지 그림자)이 순서에 정확히 삽입됨.
