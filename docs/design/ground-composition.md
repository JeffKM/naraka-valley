## 0. 문제

바닥 전환 타일을 warm 리컬러했지만, 확대하면 **단일 베이스 잔디 타일이 격자로 반복**돼 평면적이다.
스타듀 바닥은 색·패턴이 아니라 **여러 메커니즘의 합**으로 살아 보인다.

## 1. 스타듀의 "바닥 생동감" = 4겹 (위키 전수 조사 결론)

| 겹 | 메커니즘 | 시각 비중 | 비고 |
|---|---|---|---|
| **A. 동적 잔디 텍스처** | 타일당 **4 tuft** 밀도가 매일 차오르며 번짐(65%/일 성장, 인접 25% 확산) | **가장 큼**(면적 지배) | 낫으로 베어 Hay |
| **B. Debris 3대장** | **잡초·잔가지·잔돌**을 잔디 위에 흩뿌림. 벨 수 있고 자원 드롭. 방치 시 파괴적 확산 | 중간(점점이) | 개간 게임루프의 핵심 긴장 |
| **C. Forage 스폰** | 맵당 **주 6개** 계절 채집물이 녹색 타일에 희소 출현 | 작음(희소·고가치) | 탐험 보상 |
| **D. 절기 교체** | 계절1일 forage 전멸 + 잡초 리스폰 + 겨울 잔디 휴면 | 전면적(주기) | "시간이 흐른다" 감각 |

> **시사점:** 비어 보이지 않는 땅의 80%는 **A(자라는 잔디) + B(흩뿌린 debris)**가 만든다.
> 바닥 텍스처 1종만 깔면 평면. "베이스 잔디 + 스캐터 + 희소 채집 + 절기 교체" 4겹이 정답.

## 1.5 스타듀 타일 시스템 *실제 구현* (조사 2026-06-30 — 디컴파일·모딩 출처)

> owner 지적("단순 합성하면 과장되게 울퉁불퉁·지저분, 스타듀는 경계 타일이 풀/길을 함께 머금고
> 코너는 둥글고 같은 경계라도 풀이 더/덜 나와 자연스러움")을 검증하려 실제 구현을 파봄.

**① 베이스 지형(풀/흙 바닥) = 런타임 오토타일이 아니라 *아티스트가 손으로 그린 타일을 손배치*.**
- 맵은 xTile/Tiled(.tbin)로 **수작업 저작**. 풀↔흙 경계·둥근 코너·"풀이 더/덜 나온" 변주는 **타일시트에
  변종 엣지/코너 타일을 잔뜩 그려두고** 작가가 칸마다 골라 깐 것(알고리즘 1-combo-1-tile 아님).
- 16×16 타일, 레이어 Back/Buildings/Front/AlwaysFront/Paths(+계절별 타일시트 4벌 교체).
- → **"자연스러움"의 정체 = 풍부한 변종 타일 + 변주 배치.** 단일 전환 타일 반복이 아님.

**② 잔디(자라는 풀) = 런타임 *오버레이*가 맞다 — 단 작고 부드러움.** (`Grass.cs` 확인)
- 타일당 **tuft 1~4개**, 각 **15×20px** 작은 스프라이트. 위치는 **타일좌표 시드 난수**로 칸마다 다르게
  (2×2 + 중앙 배치, ±2~8px 지터). 바람 흔들림, 계절은 소스 Y오프셋(봄0/여름20/가을40)으로 틴트.
- 매일 +1~3 tuft 번짐(최대 4). → 잔디 풍성함은 **오버레이 시스템이 맞다**(owner "오버레이 금지"는
  부분만 옳음). 단 **작고 저대비·부드러워** 베이스에 녹아든다(우리의 크고 청키한 클럼프와 정반대).

**③ 플레이어 바닥/길 = 비트마스크 오토타일**(이웃 같은 바닥 16조합을 작가가 다 그려둠).

**④ debris(잡초·잔가지·돌)·forage = 지형과 별개 Object 스폰**(§1 B·C).

### 우리 결과가 지저분했던 진단
1. 베이스 = PixelLab Wang **단일 전환 타일(combo당 1개)** → 체커 반복(변종 없음).
2. 리컬러를 **청키·고대비**로 굳혀 "과장되게 울퉁불퉁"(ADR-0035 청키 락이 역효과).
3. 디테일을 **크고 진한 클럼프**로 오버레이 → 스타듀의 작고 부드러운 15×20 tuft와 정반대 + 반복 베이스와 이중으로 번잡.

### 증분 진행 (2026-06-30, [ADR-0042])
- **증분 1(완):** 부드러운 warm 베이스(`tools/recolor_ground_soft.py`) — 청키÷2·하드 스냅 폐기.
- **증분 3(완, [ADR-0043] — owner 인게임 재지적):** 증분2의 *오버레이 + 기존타일 후처리*가 틀린 수단이라 교정.
  ⑴ **오버레이 모델 폐기** — 풀 생동감·연결은 칸 위 tuft가 아니라 *풀로 꽉 찬 lush 베이스 타일*로(`_build_ground_details` 비활성).
  ⑵ **타일셋 재생성**(PixelLab `mode=pro`·medium detail·`raggedness 0.4~0.45`) — grass·길·밭·물 체인을 lush full-grass + 유기적(들쭉날쭉) 경계로 새로. terrain id 길0·풀1·밭2·물3 보존.
  ⑶ **풀 톤 정규화 런타임 일원화**(`_harmonize_grass_variants` 패스 A: 모든 풀 픽셀 hue/채도 warm-moss 수렴 → water_grass 노랑·연못 링·candy 정리 / 패스 B: base 변종 명도 정합 → 필드 체커 제거). 둘 다 lush 결 보존. 회귀 15종 통과. *남은 선택:* per-cell 경계 변종(시드 2벌 → terrain alternative).
- **증분 2(완, 이후 ⑶에서 대체):** ⑴ **잔디 변종 조화** — *진단:* 체커/"퍼즐 그물망"의 정체는 합쳐진 Wang 4세트가
  *서로 다른 PixelLab 런*의 all-grass 타일을 셋(좌표 1,3 / 1,11 / 1,15) 만들고 터레인이 이를 무작위로
  까는데, soil_grass의 1,11이 밝고 잎사귀가 잔뜩인 **고대비 타일**(휘도 std 18.8 vs 나머지 3.1)이라
  도드라진 것. *교정:* `main.gd::_harmonize_grass_variants()`가 빌드 때 세 타일을 **공통 베이스 톤으로
  리톤 + 내부대비 평탄화(std 기반)** 하고, 경계에서 0이 되는 **중심 가중 명도 변주**(sin·sin)만 남겨
  하드 엣지 없는 부드러운 3변종으로 만든다 → 퍼즐 소멸·체커 소멸(런타임 글루, tres·소스 PNG 불변).
  ⑵ **작은 부드러운 tuft 재도입** — 청키 덤불 `GD_GRASS3` 폐기, `GD_GRASS2`는 드문 warm 액센트(weight↓).
  `_gd_soft_image()`가 풀포기를 평균색으로 끌어 저대비·반투명하게 녹이고 **결정적 좌우반전 변종**으로
  도장 반복을 깬다. 맨 잔디 비중↑(차분한 여백). `_build_ground_details()` 재가동.

### 교정 방향(핵심 레버)
- **(a) 더 부드럽고 저대비·작은 아트** — "과장 울퉁불퉁"의 근원은 청키·고대비. 스타듀는 저대비·미세.
- **(b) 한 지형 config당 *변종 타일 여러 개*** — Godot 터레인 **alternative tile**로 오토타일이 랜덤 변주
  → "같은 경계라도 풀이 더/덜, 코너 둥글게"를 알고리즘으로 재현. (스타듀의 손배치 변종을 자동화)
- **(c) 베이스 잔디도 변종 2~3종**(저대비)으로 체커 반복 제거.
- **(d) 잔디 tuft 오버레이는 유지 가능하되 작고 부드럽게**(스타듀식 15×20급). 큰 청키 클럼프 폐기.

> ⚠️ **ADR 긴장:** (a)는 ADR-0035 "거친 청키 low detail"와 충돌. 스타듀 레퍼런스(4.34.17)는 우리 청키 락보다
> 부드럽고 디테일이 높다. 청키 강도를 낮출지(=ADR-0035 일부 개정)가 이 방향의 핵심 결정.

## 3. 확률 계층형 지면 구성 — **지형 종류별 독립 시스템** (owner 모델)

> **★핵심(owner 2026-06-30):** 지면 디테일은 *지형마다 자기 확률 시스템*을 가진다 — 잔디 칸엔
> 잔디 시스템, 길 칸엔 길 시스템, 밭·모래·돌엔 각자 시스템. 한 테이블을 전 지형에 난사하지 않는다.
>
> 공통 규칙: **각 칸이 자기 지형의 테이블로 결정적 해시 *상호배타* 가중확률 1롤** → 한 종을 뽑는다.
> 결정적(셀 좌표 해시)이라 프레임·세이브·재방문에 **고정**(깜빡임 없음). 피벗 bottom-center +
> 타일 내 미세 지터(±8/±6px, §asset-ruleset 3) + SE 반투명 타원 그림자(§11, 평면·초소형 생략).

**구현 = 지형→테이블 맵.** `TERRAIN_DETAILS = { GRASS:[…], PATH:[…], SOIL:[…], … }`.
각 칸은 자기 타일 종류로 테이블을 고르고 그 안에서 가중 1롤. 신규 지형(모래·자갈·숲바닥·암반)은 표만 추가.

### 3.4 기타 지형 — 표만 추가하면 확장

4. **검증** ✅ — `home_full_dump`(실제 main 베이크 합성)로 시각 확인 + 회귀 테스트 5종 통과(prop_layout·place_mode·building·world·home_expansion).
5. **남은 것 → Track 2** — 베이스 잔디 PixelLab 재생성(organic blade로 격자 반복 제거, 별도 spec-gate). 신규 지형(모래·숲바닥 등)은 `_GD_TABLES`에 표만 추가.
