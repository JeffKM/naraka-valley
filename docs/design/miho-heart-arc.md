# 미호 하트 서사 — Phase 1 슬라이스 골격 (그레이박스)

> **상태:** 게이트(T4.4) 전 **가변 설계 메모**다. 확정 사양이 아니다.
> 모든 서사 텍스트는 `[P]`(플레이스홀더) — 구조 검증용이지 완성 대사가 아니다. 문장 다듬기는 재미 게이트 통과 후 Phase 2.
> 관련: [CONTEXT.md](../../CONTEXT.md) '호감도'·'여우불 성장 촉진'·'메인 스토리'·'미호', [ADR-0004](../adr/0004-atonement-theme.md), [ADR-0005](../adr/0005-main-story-emergent.md), [ROADMAP](../../ROADMAP.md) T3.2~T3.4.

## 목적

- **검증:** 14일 안에 "친해질수록 농사 잘 됨 + 캐릭터 진짜 이야기"의 **이중 보상**(CONTEXT '호감도')이 체감되나.
- **부차 목적:** *하트 한 토막의 적정 크기/리듬*을 몸으로 학습 → 멜·바나 하트에 같은 틀 재사용.

## 진행 규칙

- **올리는 법(T3.3):** 일일 대화(하루 1회 소폭) + 선물(영혼 호박 선호 = 큰 폭). 하트 단계 UI 반영.
- **14일 도달 곡선:** ♡1 ≈ D5 / ♡2 ≈ D10 / ♡3 ≈ D14.
- **T4.3에서 이 곡선에 맞춰 확정한 수치:** 일일 대화 `8`점 · 하트 임계값 `40`점/칸(`affinity.gd`). 헤드리스 시뮬로 검증 — **대화만으로도 ♡1=D5·♡2=D10**(선물 없이 여우불 '번짐'까지 보장), **♡3은 선물이 밀어 올린다**(영혼 호박 1개 = 40점 = 정확히 1하트).
- ※ 영혼 호박은 *느림·고수익*이라 늦게야 선물 가능 → **♡3이 자연히 슬라이스 후반에 걸린다.** 의도된 시너지.

## 하트 3단계 골격

| | 도달 | ㉠ 여우불 보상(T3.4) | ㉡ 일일 대화 톤 | ㉢ 미호 속죄 비트 `[P]` |
|---|---|---|---|---|
| **♡ 1** | ~D5 | **점화** — 하루 1회, 지정 1칸 성장 +1단계 | 명랑·들뜸, 멘토링 | `[P]` "불은 태우기만 하는 게 아니야 — 봐, 이렇게 *키울* 수도 있어!" (능력=속죄의 밝은 표면. 죄는 아직 안 비침) |
| **♡ 2** | ~D10 | **번짐** — 주변 3×3 범위 가속(또는 하루 2회) | 농담 많아짐, 가끔 *불 얘기에 잠깐 굳음* | `[P]` "…나, 예전엔 불을 잘 못 다뤘거든." 하고 말 돌림 (사고 암시, 디테일 0) |
| **♡ 3** | ~D14 | **만개** — 범위↑(밭 일부) + 가속폭↑ | 신뢰, 진솔 | `[P]` **첫 고백 한 토막** — 생전 여우불로 (호박밭을) 태운 사고를 처음 털어놓음. *악의가 아니라 못 다뤄서.* 그래서 영혼 호박을 키우는 게 자기한텐 특별하다 + **㉣ 옥자 감정 씨앗(아래)** |

### ㉣ 옥자 감정 씨앗 — ♡3에 한 톨만 (옅게)

> `[P]` "옥자? …그 친구, 보기보다 속은— …아냐, 됐다."

- 미호가 옥자의 숨긴 속정을 안다는 암시 한 톨. ADR-0005 '미호=감정' 조각의 *씨앗*.
- 미호와 옥자는 **친구 사이**(존칭 없이 '옥자'로 부름, '언니' 아님 — CONTEXT '미호').
- ④(플레이어의 죄)가 어떤 죄로 확정돼도 회수되는 **범용 복선.** Phase 1에선 **여기서 더 안 나간다.**

## 검증 체크리스트 (재미 게이트 T4.4에서 물을 것)

- [ ] 여우불 1→2→3 티어의 **농사 체감차**가 분명한가? (안 분명하면 T3.4 보상 재설계 신호)
- [ ] 하트가 오를 때 ㉡대사 변화 + ㉢속죄 비트가 **"더 알고 싶다"를 만드나?** (A 모델의 핵심 동력)
- [ ] 일일 대화 vs 선물의 **비중이 적절한가?** (대화만으로도 오르되, 호박 선물이 "특별 가속"으로 느껴지는지)
- [ ] ♡3 고백이 **14일 엔딩 직전**에 떨어져 "다음 시즌 더 하고 싶다"로 이어지나?

## ROADMAP 연결

이 골격이 곧 **T3.2**(대화 텍스트박스)·**T3.3**(호감도 단계+선물)·**T3.4**(여우불 티어)의 *콘텐츠 명세*다. `[P]` 텍스트는 T3.2에서 회색 박스에 그대로 넣어 구조만 돌리고, **실제 문장 다듬기는 Phase 2.** (T3.5 수확 사연 한 줄은 별개 장치.)

## 학습 메모 (멜·바나 재사용용)

> 미호를 만들며 측정할 것: **하트 1단계 = 비트 몇 줄이 적정한가**, **여우불 같은 "능력 보상"이 하트마다 *느껴지는 증분*을 주려면 티어가 몇 단 필요한가.** 이 숫자가 나오면 멜(경제)·바나(밤경비) 하트도 같은 틀로 빠르게 찍는다.
