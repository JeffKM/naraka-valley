## 3. 이행 계획 — 두 갈래로 분리

작업 성격이 다르므로 **(A) 작명 정리(rename)** 와 **(B) 구조 빌드아웃**을 갈라 진행한다.

### (A) 작명 코드 정리 — 저위험 rename (선행)
현재 존재하는 것의 *이름만* 바꾼다. **표시명(player-facing)만 교체, 내부 id/const/세이브 키는 보존**(세이브·회귀 안전, 표시명≠식별자 원칙).

| 대상 | 현재(표시) | 신규(표시) | 코드 위치(표시명·주석) | 내부 id(보존) |
|---|---|---|---|---|
| 짐승 소형 | 혼백 닭 | **노을닭** | `animal_catalog.name_ko` | `HONBAEK_DAK`/`"honbaek_dak"` 유지 |
| 짐승 대형 | 혼백 소 | **안개소** | `animal_catalog.name_ko` | `HONBAEK_SO` 유지 |
| 산물(알) | 혼백란 | **노을알** | `product_name`·ItemCatalog 표시 | `honbaek_ran` 유지 |
| 산물(젖) | 혼백유 | **안개젖** | `product_name`·ItemCatalog 표시 | `honbaek_yu` 유지 |
| 창고(단일) | 창고 | **갈무리방** | 진입 프롬프트·주석 표시 | `STOREHOUSE_*` const 유지 |

**★불가침(rename 금지 — 이름만 비슷할 뿐 무관):** `혼백도`(魂魄桃, 혼의 나무 과일)·`혼백관`(박물관). grep 시 반드시 제외.

**동물 건물 명칭(축사)은 (A)에서 확정 rename하지 않는다** — (B) 2분리 빌드아웃에서 현 축사가 **넋우릿간(대형)**으로 재정의되고 **넋둥우리(소형)**가 신설되므로, 지금 단일 축사를 한 이름으로 바꾸면 곧 재작업된다. (A)에선 짐승·산물·창고만.

### (B) 구조 빌드아웃 — 시스템+맵+아트 (후행, 슬라이스 배치)
로스터의 신규 건물을 실제로 짓는다. rename이 아니라 새 시스템/맵/아트.

1. **동물 건물 2분리** — 현 축사(넋우릿간, 대형/안개소) + 신규 넋둥우리(소형/노을닭). 각 진입 실내(SDV처럼)·돌봄 UI 실내 이동·맵 배치. → 목축 심화 슬라이스(Slice 1 후반 or 별도).
2. **여물광(사일로)** — 목축 건초 저장·낫 사일로 채집(S1-7 하류).
3. **혼우물(우물)** — 물뿌리개 리필.
4. **늘봄방(온실)** — 계절무관 재배.
5. **혼 연못(물고기 연못)** — 낚시(Slice 3) 연계.

각 항목은 ROADMAP에 슬라이스로 배치. 진입 실내가 필요한 건물(넋둥우리·넋우릿간·갈무리방)은 SDV 대응상 enterable.
