# 대화 초상화 스펙 카드 (S0-6, ADR-0025 생성 전 승인 게이트)

> **워크플로우:** owner가 **제미나이로 원화 직접 생성** → 이 카드의 규격에 맞춰 → 내가 **변환기(PIL 글루)로 에셋화**. (ADR-0001 허용: 변환 엔진 제작이 아니라 배경 제거·크롭·리사이즈 정리 단계.)
> **근거:** [ADR-0003] 표정=별도 일러스트 초상화 · [ADR-0026] #3 "얼굴·매력은 초상화가 전담" · 인게임 슬롯 실측(`main.tscn`).

## 1. 프레이밍 = "헤드 & 체스트 버스트" (owner 질문: 몸 어디까지?)

정사각 캔버스, **머리 위 여백 ~ 가슴 상단(명치 위)** 까지. 얼굴이 주인공.

```
┌─────────────────┐  ← 정사각 캔버스 (투명)
│   (투명 여백)    │  상단 ~8%: 정수리·모자·귀·여우불 안 잘리게 헤드룸
│    ██ 머리 ██    │
│   ◕   얼굴   ◕   │  얼굴 = 캔버스 상단~중앙 세로 ~55–60% (크고 또렷)
│  ██████████████ │
│ 어깨 ─ 쇄골/목걸이 │  하단: 어깨→쇄골/넥라인. 가슴 정체성 요소 포함
└─────────────────┘  ← 하단 경계 = 가슴 상단(명치 위). 팔·손·허리 아래 X
```

- **각도:** 정면 또는 살짝 3/4, **시선은 정면**(플레이어를 봄).
- **포함:** 얼굴 전체 + 헤어/모자/귀 + 어깨 + **쇄골/넥라인의 정체성 소품** — 멜 빨간 염주(mala)·바나 초커/프릴·옥자 드레스 넥라인·미호 상의 깃.
- **제외:** 팔·손·허리·다리(버스트라 하단서 자연 컷). 전신 원화는 이번 스코프 밖(필요 시 별도).
- **왜 버스트인가:** 슬롯이 정사각 186px라 얼굴을 키워야 표정이 읽힘(ADR-0026). 가슴 상단까지 넣어야 강시 염주·로리타 초커 같은 캐릭터 시그니처가 살아 정면 컷과 구분됨.

## 2. 원화(제미나이 입력) 규격

| 항목 | 값 | 이유 |
|---|---|---|
| 캔버스 | **정사각, ≥ 1024×1024** | 320으로 줄일 때 선명(다운스케일만, 업스케일 X) |
| 프레이밍 | §1 헤드&체스트 버스트로 **이미 잡아서 생성** | 옛 파이프라인의 "전신→상단46% 크롭" 추정 제거 → 크롭 실패 없음·얼굴 픽셀 최대 |
| 배경 | **투명 PNG 1순위** | 변환기가 바로 씀 |
| 배경 대안 | 투명 안 되면 **단색 마젠타 `#FF00FF`** | 캐스트에 마젠타 없음 → 안전. **초록/청록 금지**(멜 teal·미호와 충돌해 디스필 오작동) |
| 배경 또다른 대안 | 제미나이가 투명을 **회색 체커보드**로 렌더하면 그대로 제출 | 변환기가 모서리 flood-fill로 제거(흰 레이스·소품 보존) |
| 스타일 | 기존 초상화 톤 유지(부드러운 셀/픽셀 일러스트, 스타듀-코지), **새 스프라이트 디자인과 일치** | ADR-0026 룩 통일 |

## 3. 표정 세트 (캐릭터당 5장)

파일: `<stem>.png`(기본/중립) + `<stem>_<expr>.png`. 폴백 = 지정표정→`talk`→기본→슬롯off.

| 파일 | 표정 | 비고 |
|---|---|---|
| `<stem>.png` | **중립(기본)** | 태그 없을 때 기본값 |
| `<stem>_talk.png` | 말하는 중(입 열림) | 폴백 허브 — **필수** |
| `<stem>_smile.png` | 미소 | |
| `<stem>_shy.png` | 수줍음/홍조 | |
| `<stem>_sad.png` | 슬픔/시무룩 | |

- **표정만 바뀌고 정체성·의상·프레이밍·각도는 동일**해야 자연스럽게 스위칭됨(같은 버스트에서 얼굴만 교체).
- stem(게임 화자키) = `miho` · `okja` · `mel` · `bana`. **플레이어 초상화 없음**(스타듀식 의도적 제외, §9.4).
- 총 **4캐릭터 × 5장 = 20장**.

## 4. 출력 에셋 규격 (변환기 산출)

| 항목 | 값 |
|---|---|
| 파일 | `game/assets/portraits/<stem>[_<expr>].png` |
| 크기 | **320×320 정사각, 투명 PNG** |
| 표시 | 슬롯 124×124 논리(×1.5 = 186px 실제), 좌하단, `KEEP_ASPECT_CENTERED`, `filter=linear` |

## 5. 디자인 일치 체크리스트 (새 최종 스프라이트 기준 — 옛 초상화와 달라진 점)

- **옥자:** 검은 마녀모자(작은 버건디 깃털 1개)·버건디 웨이브 머리·단색 버건디 드레스·큰 또렷한 눈·냉정. **★ 안경 제거**(스프라이트 §10.8에서 뺌 — 옛 초상화는 안경 有).
- **바나:** 보라/검정 프릴 로리타·금발+검정 앞머리 브릿지·빨간 눈·**흡혈 송곳니 유지**·초커. **★ 안경 제거**(새 스프라이트엔 없음 — 옛 초상화 전용 요소였음).
- **멜:** 청록(teal) 강시 로브(파란 꽃무늬)·teal 강시 모자(옆 빨간 구슬 술 1개)·검은 일자 단발·청회색 눈·**빨간 염주 목걸이**·빨간 입술·볼홍조·만다린 카라+frog버튼. (옛 초상화와 거의 일치 — 유지.)
- **미호:** 흰 여우귀·다크브라운 웨이브 롱헤어·큰 눈·회보라빛 흰 상의(+노란 스커트 살짝)·**여우불(옆 파란 불꽃) 옵션**·따뜻. (머리색: 옛 초상화 검정 → 새 스프라이트 다크브라운으로 맞추면 통일.)

## 6. 변환기 (owner 생성 후 내가 실행)

`game/tools/make_portrait.py` / `make_expressions.py`를 **버스트-입력 모드**로 조정:
배경 3분기 자동 판정(투명/마젠타 or 그린/체커) → 알파 크롭 → **320² 정사각 contain**(46% 재크롭 없음, 이미 버스트라). 산출 후 `godot --headless --import` 강제.
