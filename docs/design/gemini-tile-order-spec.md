# Gemini 타일 발주 스펙 (owner ↔ Claude 분담 규격)

> **목적:** owner가 **Gemini(nano-banana / AI Studio)로 지형 타일을 직접 뽑아올 때**, 뽑은 결과가
> 게임에 **바로 적용되도록** 스타일·규격을 한 장에 못박는다. owner는 *스타일·형태*만 통제하고,
> *정확한 픽셀 규격·팔레트·이음매 정합*은 Claude가 후처리한다.
>
> **근거 문서(진실의 원천):** [asset-ruleset.md](./asset-ruleset.md)(§0 그리드·§1 광원·§9 팔레트·§16 램프) ·
> [tileset-ruleset.md](./tileset-ruleset.md) · [ADR-0050](../adr/0050-environment-32-native-revert-ai-source-supersede-0049.md)(32-native) ·
> [ADR-0057](../adr/0057-terrain-tiles-pixellab-lowcolor-crisp-regen-supersede-0043.md)(지형 타일 저색 crisp) ·
> [ADR-0025](../adr/0025-in-game-prop-placement-mode-data-externalization.md)(스펙 카드 게이트).
>
> ⚠️ 값이 근거 문서와 어긋나면 **근거 문서가 이긴다.** 이 문서는 발주 편의용 요약+실무 가이드다.

---

## 0. 역할 분담 (핵심)

| | owner (Gemini로 뽑음) | Claude (적용) |
|---|---|---|
| 담당 | **스타일·형태·톤·구도** | **32px 다운샘플·팔레트 스냅·2px 그레인·이음매 정합·배선** |
| 산출 | "스타듀 톤에 형태 맞는 **큰** 픽셀아트" | 게임에 들어가는 최종 32×32 타일 + 코드 배선 |

→ owner는 픽셀 하나하나 맞추려 애쓸 필요 없다. Gemini는 어차피 정확한 32px 그리드·팔레트를
못 지키므로, **형태와 스타일만 맞으면** 나머지는 Claude가 후처리한다.

---

## 1. ⚠️ 확산모델(Gemini) 적합성 — 먼저 읽을 것 (ADR-0057)

프로젝트는 **지형 seamless 타일에 확산모델을 원칙적으로 배제**한다(고색 노이즈·타일링 불보장·팔레트 이탈,
실측 2573색 노이즈 카펫). 정석은 PixelLab(픽셀 네이티브). **단, owner가 스타일을 직접 통제해 뽑고
Claude가 후처리하는 이 워크플로에서는 아래 우선순위로 감수한다:**

| Gemini 적합도 | 타일 성격 | 이유 |
|---|---|---|
| ✅ **가장 안전** | **코너·특수 마감**(맵에 1~2개만 놓임) | 옆칸과 이어질 필요 적음 → 이음매 부담 최소 |
| 🔶 **가능(매수 적게)** | 물가·경계 등 **반복 배치** 타일 | Claude가 이음매 수동 정합 필요 → 장수 늘면 부담 급증 |
| ❌ **피함** | 넓게 깔리는 **베이스 지형**(풀·흙 바닥) | seamless 타일링 필수 → PixelLab/코드가 정답 |

**규칙:** 반복 타일은 되도록 **적은 변주**로, 코너/특수 타일 위주로 Gemini에 발주한다.

## 2. 해상도 · 그리드 (ADR-0050 = 32-native 확정)

| 항목 | 값 |
|---|---|
| **최종 파일** | **32×32px PNG, 투명 배경** (1칸 타일 = 32×32 네이티브) |
| 월드 타일 | `TILE=32`, **1:1 드로우** (더는 ×2 업스케일 안 함) |
| 온스크린 | ×2 스트레치 = 64px |
| **★캐논 그레인** | **2px 블록** ("1 논리px = 2 화면px"). 2×2 동일색 블록 **≥95%** — 전 에셋 예외 없음(asset-ruleset §0.1) |
| owner 발주 크기 | **크게 뽑을 것**(예: 1024²). 작게 뽑으면 다운샘플에서 뭉갬. 32px 스냅은 Claude가 |
| 후처리 방침 | 과한 양자화·하드 외곽선 **금지** → 가벼운 정리(**≈24색 median + 알파 하드에지**)로 부드러움 보존(ADR-0050) |

## 3. 팔레트 (asset-ruleset §9·§16, ADR-0057) — 재질별 램프, 램프당 3~5색·총 ~30~40색

| 분류 | 규칙 |
|---|---|
| **베이스 지형**(걸어다니는 땅) | warm 스타듀 팔레트(살짝 가라앉힘), **무외곽선·저대비·소프트**, 저색(~45색) |
| **객체**(건물·나무·바위·프롭·**절벽**·debris) | muted 저채도 저승 팔레트 + **단색 외곽선 `#401818`** |
| **물**(연못·강·바다) | 영혼빛 파랑 `#60d8f0`→`#2068e8` |
| **그림자** | 스프라이트에 굽지 말 것 → 반투명 블랙(#000, α~30%) SE 오버레이(Claude가 처리) |

**연못 절벽 타일 실측 hex(기존 `pond_cliff/`에서 추출 — 코너/경계 발주 시 그대로 지정):**

| 재질 | hex |
|---|---|
| 흙벽 상판(tan) | `#C9995C` · `#A97E4C` · `#BF9157` |
| 흙벽 수직면 | `#7A5A36` · `#82613B` · `#967043` |
| 흙벽 음영 | `#24221A` · `#0B0D0A` |
| 물잠김 물결(밝은 teal) | `#2F5557` · `#345D5A` · `#628E91` |
| 물잠김 물속(어두운 teal) | `#22464A` · `#1A383F` · `#1B3033` |

**기타 지형 마스터 hex:** 흙길 `#684840` / 밭흙 `#503030` / 풀(이끼녹) `#306033` / 전경 외곽선 `#401818`.

## 4. 광원 · 스타일 프롬프트 (asset-ruleset §1) — 전 오브젝트 **NW 빛 / SE 그림자**

Gemini 프롬프트에 **그대로 복붙**할 고정 스타일 문장(단순 "top-left light"는 그라데이션→쯔꾸르 회귀):

```
Flat 2D pixel art tile, Stardew Valley farm style with muted underworld tone,
light source from top-left (NW), distinct directional step-shading,
1px highlight on top and left edges, crisp dark shadows cast to bottom-right (SE),
limited palette, chunky hard pixels, clearly visible pixel blocks,
NO anti-aliasing, NO smooth gradients, no text, transparent background.
```

- **룩:** 스타듀 톤 + 저승 저채도 + 거친 레트로 도트. lineless/매끄러운 RPG메이커 그라데이션 **금지**.
- **★참조 첨부가 최강:** 기존 타일 이미지를 첨부하고 `"match this exact pixel style and palette"` 추가.
  - 연못 절벽 참조 시트: `game/assets/tiles/pond_cliff/`(원본) 또는 Claude가 만든 확대 참조본.

## 5. 타일별 콘텐츠 레이아웃 (32px 안에서 뭐가 어디에)

> 이음매가 맞으려면 콘텐츠 위치가 타일 종류마다 다르다. Gemini 출력은 크게 나오므로,
> **아래 "세로 위치"는 Claude가 32px로 스냅할 때의 목표**다. owner는 형태만 맞추면 된다.

### 5.A 연못 북벽 절벽 (코너·정면) — `pond_cliff/` 계열
- **콘텐츠 = 세로 중앙 y7~24(17px)만**, 위·아래 여백. 기존 `wall_dirt`/`wall_sub`와 세로 정합.
- **2단 구조:** 위 = 마른 흙 상판→수직 흙벽면(어두운 갈색, 뿌리·잔돌), 아래(물잠김 타일) = teal 물속 벽+물결선.
- **코너(신규):** 정면 흙벽이 **좌끝은 왼쪽·아래로, 우끝은 오른쪽·아래로 감기며** 측면 물가로 전환.
  좌(북서)·우(북동) 각 1장. `_draw_north_pond_cliff`가 좌우 끝 셀에 blit.

### 5.B 물↔흙 경계 (물가 shore) — 이미 확정 시스템
- **32px 꽉 채움.** 확정 방식 = **손그림 형태 마스크**(0=물 / 1=흙 / 2=테두리) — water-dirt-shore
  손그림 4_0 형태 마스크에 맞춰야 함(기존 결정 준수, 재설계 금지).
- owner가 새로 뽑을 경우: "흙(바깥)과 물(안)이 손그림 불규칙 테두리로 나뉜 top-down shore tile" 형태로.

### 5.C 물가 평면 (남/동/서 가장자리)
- **32px 꽉 채움**, 위=흙/아래=물의 수평 경계(현재 `4_0` 평면 타일 역할). 반복 배치라 매수 최소.

### 5.D 일반 베이스 지형(풀·흙 등)
- ❌ Gemini 비권장(§1). 필요 시 PixelLab/코드. 부득이하면 32px 꽉 채움·seamless 필수(Claude 정합 부담 큼).

## 6. 발주 카드 템플릿 (owner가 채워서 Claude에게 주면 정밀 스펙으로 확정)

```
타일 이름: <예: 연못 북서 코너>
성격: 코너 / 특수마감 / 반복경계 / 물가   (§1 적합도 확인)
붙는 위치: <예: 연못 북벽 좌끝 x26,y34 — 왼쪽은 서쪽 평면 물가와 만남>
콘텐츠 레이아웃: <§5 참조 — 세로위치·2단여부·감김방향>
팔레트: <§3에서 해당 hex>
참조 첨부: <기존 어떤 타일에 맞출지>
매수: <좌/우 등 몇 장>
```

Claude가 이 카드를 받으면 → 이음매·인접 정합 규칙을 확정해 **최종 발주 문장**을 만들어 주고,
owner가 Gemini로 뽑아오면 → **32px 스냅·팔레트 양자화·2px 그레인·이음매 정합·코드 배선·회귀·커밋**까지 잇는다.

## 7. 후처리 파이프라인 (Claude 담당 — 참고용)

1. **다운샘플** — Gemini 대형 출력 → 32×32 (NEAREST/BOX, 콘텐츠 크롭·정렬)
2. **팔레트 스냅** — `tools/quantize_to_palette.py`류로 §3 램프에 미디엄 양자화(형태 보존)
3. **2px 그레인** — `tools/enforce_chunk.py`로 2px 블록 강제(§0.1, ≥95%)
4. **알파 하드에지** — 투명 배경 경계 정리
5. **이음매 정합** — 인접 타일(위 흙지면·아래 물·옆 벽)과 경계 픽셀 수동 정합
6. **배선·검증** — main.gd blit/오토타일 연결 → `game/run_tests.sh <name>` 선별 회귀 → 라이브 덤프 → 커밋
