# 게임 루프 지도 — 넓이의 청사진 (전체 그림)

> **상태:** 농사 단일 게이트(T4.4)는 "판정 보류 — 범위 확장"으로 종결됐고(ADR-0006), 게이트는 **Phase 1.5 통합 슬라이스**로 옮겨졌다. 이 문서는 그 통합 게이트 전 **가변 설계 메모**로 계속 유효하다. 확정 사양이 아니다. 세부 수치·서사 텍스트는 `[P]`(플레이스홀더) 또는 **서랍(미정)** 으로 비워 둔다 — 여기서 정하는 건 *각 루프가 어느 방향으로 깊어지는가*라는 구조뿐이다.
>
> **이 문서가 푸는 문제:** "다른 루프(낚시·멜·바나·채광)도 MVP로 다 코드에 깔아두고 하나씩 업그레이드할까, 농사부터 깊게 끝낼까"의 고민에 대한 답 — **넓이는 코드가 아니라 이 문서로 그린다.** 전체 형태를 글로 먼저 보고, 실제 코드는 ROADMAP 순서대로 한 루프씩 *끝까지* 쌓는다(개발 원칙 "한 시스템 100% 끝내고 다음", 번아웃 1순위 리스크).
>
> 관련: [CONTEXT.md](../../CONTEXT.md), [ROADMAP](../../ROADMAP.md) Phase 3, [ADR-0004](../adr/0004-atonement-theme.md)(속죄), [ADR-0005](../adr/0005-main-story-emergent.md)(창발 서사·활동은 서사 비의존), [miho-heart-arc](./miho-heart-arc.md)(농사↔미호 관계 루프의 콘텐츠 명세).

## 0. 왜 이 문서가 필요한가 — 스타듀밸리 3층 모델

스타듀밸리 농사가 "끝없이 하게 되는" 이유를 뜯어보면 한 루프가 **세 층위**로 쌓여 있다:

| 층 | 이름 | 하는 일 | 스타듀 예 |
|---|---|---|---|
| **1층** | 일일 루프 | 매일 반복하는 핵심 동작 | 물주기→수확→심기→취침 |
| **2층** | 선택·전략 루프 | 무엇을·언제 할지의 선택지 | 계절 작물 선택, 재생장 작물, 품질 |
| **3층** | 성장 곡선 | 플레이어를 우상향시키는 장기 엔진 | 자동화(스프링클러)·숙련(스킬)·가공(와인·치즈) |

**핵심 통찰:** 재미의 "또 하고 싶다"는 주로 **3층(우상향 곡선)** 에서 나온다. 1층만 있으면 루프가 평평하다.

우리(나라카밸리) 농사는 현재 **1층은 스타듀와 거의 동급, 2층은 얇고, 3층은 거의 비어 있다**(자동화·숙련·가공 없음 — 모두 ROADMAP Phase 3 서랍). 대신 우리에겐 스타듀가 *다르게 푼* **고유의 4번째 축**이 있다:

> **㉠ 관계 보상축 (우리 고유):** 캐릭터 호감도가 그 캐릭터의 활동 효율에 직결된다. 미호 호감도 → 여우불 농사 가속(`foxfire.gd`). 스타듀는 NPC 호감도와 농사가 분리돼 있지만, 우리는 **관계가 곧 그 루프의 성장 곡선**이다. (CONTEXT '여우불 성장 촉진' = 관계 보상형 A.)
>
> **㉡ 속죄 사연축 (우리 고유):** 활동 산출물에 속죄 테마가 얹힌다 — 수확 시 영혼의 사연 한 줄(`flavor.gd`). 저비용 깊이 장치(ADR-0004).

**설계 규칙:** 각 활동 루프는 *기계적으로* 3층 모델을 따르되, **메인 서사 텍스트는 짊어지지 않는다**(CONTEXT 운영규칙 — 서사는 캐릭터에만, 활동은 게이트 열쇠로만 참여). 서사 곡선은 **관계 루프**가 따로 진다.

## 2. 루프별 3층 스케치 (MVP → 업그레이드)

각 루프를 "MVP(최소로 돌아가는 1층) → 2층 → 3층" 순으로 그린다. **MVP만 먼저 만들고 위층은 나중에 그 루프를 *재방문*하며 한 겹씩 올린다** — 동시에 다 만들지 않는다.

### 2.1 농사 (미호) — 유일하게 구현된 루프, 3층 로드맵의 기준점

- **1층 (구현됨):** 괭이→심기→물→수확, 혼력 소모, 골드→씨앗 재구매. 작물 3종 속도 3단(혼령초/피안화/영혼호박). `field.gd`·`crops.gd`.
- **2층 (얇음 → 채울 후보):**
  - **재생장 작물** `[서랍]` — 한 번 심어 여러 번 수확(스타듀 블루베리류). 현재 `harvest()`는 칸을 비우므로 *없는 층*. 노동/수익 다양성의 핵심.
  - **작물 품질** `[서랍]` — 같은 작물도 잘 키우면 비싸게. 현재 `sell_price` 단일값.
  - **계절·달력** `[서랍]` — 작물 계절 제약. ROADMAP "계절·달력 확장".
- **3층 (성장 곡선):**
  - **㉠ 관계 자동화 = 여우불(구현됨, 미세조정 필요):** 호감도↑ → 가속(`accel`)·범위(`reach`)↑. **이게 우리의 스프링클러다** — 관계로 물주기 노동을 던다. ⚠️ 현재 14일 안에 거의 안 열림(T4.3 밸런싱 과제, [miho-heart-arc](./miho-heart-arc.md) ♡1≈D5/♡2≈D10/♡3≈D14 목표 곡선 미달).
  - **농사 숙련** `[서랍]` — 농사를 할수록 수확량/품질↑(스타듀 스킬트리).
  - **가공** → 2.7 가공·서빙으로 연결(작물 → 카페 메뉴).

### 2.2 낚시 (순수 활동) — 삼도천 낚시

- **1층 MVP:** 물가에서 캐스팅 → 타이밍/미니게임 → 물고기(망자의 혼·도깨비 물고기) 획득 → 판매. 혼력 소모, 시계 압박.
- **2층:** 물고기 종 다양성, 장소·시간대(밤/낮)·날씨별 분포 `[서랍]`.
- **3층:** 낚시 숙련(잡는 종↑·확률↑) `[서랍]`, 잡은 물고기 → 가공/카페 메뉴(2.7).
- **서사 역할:** 메인 플롯 글쓰기 비의존. **게이트 열쇠로만** — 특정 물고기가 옥자 게이트 장면의 *기계적 열쇠*가 될 수 있다(CONTEXT 인연 조각 출처 규칙).

### 2.3 카페 운영 (멜) — 속죄: 돈으로 망침 → 카페를 굴림

> **갱신([ADR-0007](../adr/0007-mel-cafe-operation-loop.md)):** 원안은 멜을 *카페 경제·회계*(판매가/이자/장부 같은 **돈 관리** 시스템)로 그렸으나, Sprint 5 grill에서 "돈을 직접 굴리는 추상 금융(예금·투기·대출)은 재미가 안 난다 — 게임에서 돈은 활동이 아니라 활동의 *결과*다"를 확인하고, 멜 루프를 **플레이어가 직접 하는 카페 운영 활동**으로 재정의했다. 아래 1층을 그에 맞춰 고쳐 쓴다. **정체성 격상이 아니라 멀티루프에 루프 하나 추가**(농사 강등 없음, ADR-0002 불변).

- **1층 MVP:** 카페 운영 활동 — 손님(나라카 주민 **단골**)을 응대·서빙(다른 활동 산출물로 만든 재료)해 매출을 낸다. **출하대**(`_process_shop`)는 멜이 운영하는 산출물 입고·환전 카운터가 되고, 입고분이 카페 운영 재료로 흐른다. + 일일 정산("오늘 매출 +N"). **그레이박스 최소부터**(회색 손님 → 서빙 → 골드 + 멜 관계축).
- **2층:** 체키 `[서랍]`, 손님 종류·요구 다양성 `[서랍]`, 메뉴 가공(2.7) 연결, 멜 호감도 보상(㉠ 운영 이득 — 내용은 원자 분해) `[P]`, 시세 변동·수요 `[서랍]`.
- **3층:** 카페 수익 곡선 — 가공품(2.7) 단가가 멜 운영 레이어 위에서 복리.
- **서사 역할:** 멜 = **옥자의 기록(사실)** 조각 — 장부·계약을 다루니 봉인된 죄목의 *존재*를 안다(CONTEXT). 호감도 단계로 해금(틀은 miho-heart-arc 재사용).
- **카페라는 무대:** 옥자(상주 사장)·미호(출근)·멜(운영)·바나(밤 바)가 카페에 모인다 — §2.7·§2.8의 합류점이 멜 카페 운영 위에서 성립.

### 2.4 밤 경비·손님 응대 (바나) — 속죄: 밤의 침입자 → 밤의 경비

- **1층 MVP:** 밤 시간대(현재 `clock.phase()`의 "밤")에 카페 경비 — 탈주 잡귀/불청객 막기 + 손님 응대.
- **2층:** 손님 종류·요구 다양성, 바나 호감도 보상(㉠ 밤 안전/손님 흐름 보너스) `[P]`.
- **3층:** 밤 운영 곡선 — 낮(농사) ↔ 밤(경비)의 시간 분업(2.8 직조).
- **서사 역할:** 바나 = **옥자의 행동(목격)** 조각 — 밤 경비라 무방비한 옥자를 본다(CONTEXT).

### 2.5 채광 (순수 활동) — 업화 갱도

- **1층 MVP:** 갱도에서 채굴 → 광물/자원 → 판매·제작 재료. 혼력 소모, 층 내려갈수록 보상↑/위험↑.
- **2층:** 광물 종·깊이·희귀도 `[서랍]`. 도구 업그레이드(곡괭이) `[서랍]`.
- **3층:** 자원이 농사 자동화(도구 제작)·카페(2.7)로 흘러드는 제작 체인 `[서랍]`.
- **서사 역할:** 게이트 열쇠로만(전투와 같은 갱도 공간 공유 가능).

### 2.6 전투 (순수 활동) — 탈주 잡귀

- **1층 MVP:** 갱도/밤 경비에서 마주치는 잡귀와 간단 전투(탑다운). 혼력/체력.
- **2층:** 적 종류, 무기 `[서랍]`.
- **3층:** 채광·밤경비와 엮인 위험-보상 곡선 `[서랍]`.
- **서사 역할:** 게이트 열쇠로만.

### 2.7 가공·서빙 (메뉴 제조) — 루프들의 합류점

> 멜의 1층 **카페 운영**(§2.3 손님 서빙)과 구분: 여기 §2.7은 작물·물고기를 **메뉴로 가공**하는 3층 합류점(고부가가치 2차 산업)이다. 멜 카페 운영이 그 메뉴를 손님에게 낸다.

- **1층 MVP:** 키운 작물/잡은 물고기 → 기계(통·항아리류)로 가공 → 메뉴 아이템(고부가가치).
- **2층:** 레시피 다양성 `[서랍]`, 손님 주문(바나 응대 2.4와 연결).
- **3층:** **농사·낚시·채광의 산출물이 모두 여기로 합류**해 멜 경제(2.3) 위에서 복리 — 스타듀 "artisan goods"에 대응하는 우리의 2차 산업.
- **서사 역할:** 카페가 세 캐릭터(미호 재료·멜 경제·바나 서빙)가 만나는 무대.

### 2.8 루프 간 직조 — 시간을 두고 맞물리기

스타듀의 진짜 재미는 단일 루프가 아니라 **루프들이 시간을 두고 맞물리는 것**이다(작물 자라는 동안 광산 감). 우리 직조 축:

- **낮 ↔ 밤:** 낮=농사(미호)·낚시·채광 / 밤=경비·손님(바나). 하루 사이클(`clock.gd`)이 이미 phase를 가짐.
- **대기시간 메우기:** 작물 자라는 날 = 낚시/채광으로 혼력·시간을 다른 데 투자.
- **자원 합류:** 농사+낚시+채광 산출물 → 가공·카페(2.7) → 멜 경제 → 골드 → 재투자.
- **관계가 전 루프를 묶음:** 셋과 친해질수록 각 루프 효율↑ + 옥자 진실 조각(감정+사실+목격) 수집(CONTEXT).

## 3. 단계적 출시 순서 (Phase 매핑)

> 원칙: **한 번에 한 루프를 끝까지.** 아래는 *코드* 순서. 넓이(이 문서)는 이미 다 그렸으니, 코드는 좁고 깊게 간다.

> **갱신(ADR-0006):** 원안은 농사 단일 루프로 재미 게이트를 통과시킬 계획이었으나, 직접 플레이 결과 *단일 루프로는 평평해 판정 불가*로 결론났다(0·5절이 예측한 대로). 그래서 **멜·바나를 게이트 *앞*(Phase 1.5)으로 당겨**, 세 루프가 어우러지는 통합 그레이박스 위에서 게이트를 재판정한다. 순수활동·합류점·에셋은 그대로 게이트 뒤.

| 시점 | 코드로 만드는 것 | 깊이 |
|---|---|---|
| **Phase 1 (완료)** | 농사 1층 + 여우불 곡선(3층 ㉠ 하나) | 단일 루프 — 게이트 판정엔 평평함 확인 |
| **Phase 1.5-a** | 멜(카페 경제) 1층 MVP + 관계 보상축 | 한 루프 1층까지 |
| **Phase 1.5-b** | 바나(밤 경비) 1층 MVP + 관계 보상축 | 한 루프 1층까지 |
| **Phase 1.5-c** | 낮↔밤 직조 + **★통합 재미 게이트** | 세 루프 어우러짐 위에서 판정 |
| **Phase 2** | 그레이박스 → 에셋 | (게이트 통과 후) 비주얼 완성 |
| **Phase 3** | 낚시(1→3층), 가공·서빙(합류점), 채광·전투, 계절·달력 | 순수활동·합류점·확장 |
| **(상시)** | 농사·멜·바나 2·3층(재생장·품질·숙련 / 시세·이자 / 손님) 재방문 | 기존 루프 심화 |

각 Phase 1.5·3 항목은 **착수 시점에 다시 원자 분해**한다(ROADMAP 해당 머리말). 이 문서는 그 분해의 출발점인 *구조 스케치*이며, 4절 '서랍'(멜·바나 관계 보상축 내용 등)은 그 grill 세션에서 채운다.
