# 스타듀밸리 시스템 대조 카탈로그 (Stardew Systems Catalog)

> **목적.** 스타듀밸리(v1.6, 1.5 Ginger Island/Qi 포함)의 *모든 시스템·콘텐츠·에셋*을 한 곳에 정리하고, 각각이 나라카 밸리에서 **이미 어떻게 처리됐는지**(번역/흡수/리스킨/부분/서랍/기각/미처리)를 대조한다.
> **사용법.** ❓**미처리** 행이 곧 "빠진 것 체크리스트"다. 문서 맨 끝 [부록 A](#부록-a--미처리-체크리스트-모음)에 ❓ 항목만 모아 우선순위와 함께 둔다.
> **권위.** 이 문서는 *외부 레퍼런스 + 갭 분석*이다. 진실의 원천은 [CONTEXT.md](../../CONTEXT.md)·[ROADMAP.md](../../ROADMAP.md)·[docs/adr/](../adr/)이며, ❓ 항목의 실제 채택 여부는 **개별 grill로 결정한 뒤** 권위 문서에 반영한다 — 이 카탈로그가 권위 문서를 자동 갱신하지 않는다.
> **마지막 갱신:** 2026-06-24 (grill-with-docs 세션 — [ADR-0021](../adr/0021-relationship-reward-light-affinity-narrative-primary.md) 관계 보상 모델 대체 반영: 관계=곱셈기 엔진 ❌ → 가벼운 호감도 당근+서사 주력. 미호=농사 XP·바나=전투 XP·멜=카지노. ADR-0010 밤 경비 폐기.)

## 상태 태그 범례

| 태그 | 의미 |
|---|---|
| ✅ **번역** | 저승 세계관 언어로 1:1 대응을 옮김 (예: 4계절 → 저승 절기) |
| ✅ **흡수** | 더 큰 나라카 시스템에 통합 (예: 전문직 트리 → 2축 성장) |
| ✅ **리스킨** | 메카닉은 그대로, 외피만 교체 (예: 마을회관 번들 → 카페 일구기) |
| 🔧 **부분** | 방향·골격은 정해졌으나 미완·일부만 설계됨 |
| 🗄 **서랍** | 의도적으로 미룸 (CONTEXT/ROADMAP의 "서랍" 항목) |
| ❌ **기각** | 명시적으로 안 함 (근거 ADR 존재) |
| ❓ **미처리** | 아직 검토 안 됨 — **체크리스트 항목** |
| ⚠️ | 나라카 기존 결정과 *충돌* 소지 — 채택 전 재검토 필요 |

# 2. 콘텐츠 전수 부록

> 개별 콘텐츠는 나라카에서 **대부분 자체 창작**하므로 항목별 상태를 붙이지 않는다. 대신 카테고리마다 **나라카 설계 규칙** 한 줄을 머리에 둔다. 스타듀 목록은 v1.6 기준 전수(1.5/1.6 신규는 🆕 표시). 정확도가 중요한 수치는 위키 교차검증했으나, 개별 어종의 시간/날씨 토글 등 미세 수치는 각 위키 페이지가 최종 권위.

## 2-1. 작물 (Crops)

> **나라카 규칙:** 절기별 자체 작물. 현재 슬라이스 3종(혼령초·피안화·영혼 호박, [CONTEXT 저승 작물](../../CONTEXT.md))을 절기 4종(피안절·유화절·망연절·성야절)으로 확장. 성장 속도 3단(빠름<중간<느림)으로 리듬, 절기 전환 시 사멸. 품질 등급은 Phase 3 §2.1.

- **봄:** 파스닙, 그린빈(재성장 3일), 콜리플라워, 감자, 마늘, 케일, 쌀(물가), 튤립(꽃), 블루 자스민(꽃), 딸기(에그 페스티벌), 🆕당근(3일)
- **여름:** 멜론, 토마토(재4), 블루베리(재4), 후추(재3), 밀(여름·가을), 무, 적양배추, 스타프루트, 옥수수(재4·여름·가을), 홉(재1), 포피(꽃), 서머 스팽글(꽃), 해바라기(꽃·여름·가을), 🆕여름호박(재3)
- **가을:** 호박, 비트, 아티초크, 봉크초이, 얌, 크랜베리(재5), 가지(재5), 포도(재3), 아마란스, 페어리 로즈(꽃), 🆕브로콜리(재4)
- **겨울(노지):** 🆕파우더멜론(재3) — 1.6 유일 겨울 노지 작물
- **특수/다시즌:** 고대 열매(재7·봄여름가을), 커피 콩(재2·봄여름), 혼합 씨앗, 🆕혼합 꽃씨
- **진저섬/온실:** 파인애플(재7), 타로 뿌리 + (온실은 시즌 무관 전 작물)
- **과일나무:** 살구·체리(봄), 오렌지·복숭아·🆕바나나·🆕망고(여름), 사과·석류(가을), 차나무(Tea)

## 2-2. 채집물 (Foraging)

- **가을:** 일반/차가/노을 버섯, 야생 자두, 헤이즐넛, 블랙베리(8–11일)

- **광산:** 스톤피쉬(20층), 아이스 핍(60층), 고스트피쉬(20·60층), 라바 일(100층)

# 3. 에셋 디자인 레퍼런스

> 스타듀 에셋을 *그대로 가져오지 않는다* — 나라카 컨셉으로 변형한다(ADR-0001). 여기선 "스타듀가 그 에셋을 *어떻게 디자인했나*"(디자인 관습)를 기록해 변형의 출발점으로 삼고, 나라카 변형 방향과 대응 파이프라인/ADR을 붙인다.

## 3-1. 환경 타일셋 (지형 · 오브젝트)

- **스타듀 디자인 관습:** 16×16 기본 타일, 계절마다 같은 맵의 *팔레트·디테일 스왑*(봄 초록↔겨울 눈). 지형(흙/풀/길/물) + 무수한 데코 오브젝트(나무 3성장단계, 바위, 그루터기, 잡초, 꽃). 타일 가장자리 자동 연결(autotile). 따뜻한 채도, 두껍지 않은 외곽선.
- **나라카 변형:** 인스타툰체(플랫+두꺼운 검은 윤곽) → 도트화. 8구역 각 정체성(안식 농원·나루 마을·삼도천·황천해·저승 숲·미혹의 숲·업화 갱도·나락). 절기 4종 팔레트 스왑(지상 계절 아님). 저승 톤(여우불 파랑·혼불·피안화 붉음).
- **파이프라인:** 16px 원본 → 렌더 ×2(ADR-0013 환경 32px native). PixelLab 탑다운 타일셋 + Aseprite 보정(ADR-0001). 절차 그레이박스(TREE/ROCK/WATER) 교체(Phase 2.7).

## 3-2. 작물 스프라이트 (성장 단계)

- **스타듀 디자인 관습:** 작물당 4~5 성장 단계, 단계마다 실루엣이 또렷이 달라짐(씨앗→새싹→성체), 수확 직전이 가장 화려/큼. 16×16~16×32. 재성장 작물은 수확 후 별도 프레임.
- **나라카 변형:** 저승 작물 3종(혼령초·피안화·영혼 호박) → 절기 4종 확장. 빛나는 효과(혼령초=푸른 빛), 얼굴이 비치는 영혼 호박. 여우불 연출(미호 농사 XP 당근의 픽션 — 옛 성장가속 비주얼 유지, 메카닉만 XP, ADR-0021).
- **파이프라인:** PixelLab 타일 + Aseprite. 현재 그레이박스 오버레이 8타일(씨앗/새싹/수확가능-황금, T2.3).

## 3-3. 캐릭터 스프라이트시트 (이동 · 애니메이션)

- **스타듀 디자인 관습:** NPC당 4방향(상하좌우) 워크 사이클 + 정지 + 표정/제스처 프레임. 16×32 정도. 인게임에선 얼굴 디테일 거의 없음(작아서) — 개성은 실루엣·색·헤어로. 일정(schedule) 따라 맵 이동.
- **나라카 변형:** ~32×64(시트 프레임 80px, ADR-0012 2배 상향 — 인게임 얼굴 가독성 1순위). 치비 ~2.5등신. 공통 `proportions` 고정(체형 통일). 테마 데이는 *모자/소품 레이어*(이동 캐릭터) 또는 *통짜 스왑*(상주 NPC) — 풀 의상 세트 안 만듦(ADR-0018·CONTEXT 카페 이벤트 데이).
- **파이프라인:** PixelLab `mode=standard, size=56, detail=high` + 공통 proportions(ADR-0012). 애니 티어 분업(ADR-0018).

## 3-4. 캐릭터 초상화 (대화 표정)

- **스타듀 디자인 관습:** 대화 시 화면 구석에 정면 반신 초상화 + 표정 변형 소수(평소/기쁨/화남/슬픔 등). 인게임 스프라이트와 별개 고해상 에셋. 매력·감정 전달 전담.
- **나라카 변형:** 인스타툰체 풀샷/반신 초상화(ADR-0003 — 표정=별도 일러스트). 테마 데이는 *테마별 초상화 변형*이 본체(매력·디테일이 읽히는 곳, CONTEXT). ⚠️ 플레이어는 평소 인게임 초상화 없음 — 테마 데이·체키 순간만 등장.
- **파이프라인:** 미드저니/제미나이(큰 단일 일러스트 강점) → 표정 변형. P2.4 초상화 채널.

## 3-5. 아이템 / 인벤토리 아이콘

- **스타듀 디자인 관습:** 16×16 단일 아이콘, 인벤토리·툴팁·출하 공용. 작물/물고기/광물/요리/재료 수백 종이 한 결의 작은 아이콘. 품질(은/금/이리듐)은 작은 별 오버레이.
- **나라카 변형:** 자체 콘텐츠 아이콘(저승 작물·물고기·광물·메뉴). 품질 = `quality` 필드(ADR-0020), Phase 3에서 시각화.
- **파이프라인:** PixelLab/Retro Diffusion + Aseprite. 핫바·인벤토리(Phase 2.8 HUD/공통 모듈).

## 3-6. 건물 외관

- **스타듀 디자인 관습:** 건물당 외관 1장(업그레이드 단계마다 별도) + 내부 맵 별도. 농장 건물(닭장/외양간 3단계)·마을 상점 각 개성.
- **나라카 변형:** 안식 농원 건물 + 나루 마을(카페·만물상·혼백관 등) + 메인 3인 집·조연 주민 집(정체성별 재도색, Phase 2.7). 카페는 명소로 클수록 화려해짐(카페 일구기 시각 결실, ADR-0009).
- **파이프라인:** 환경 32px native(ADR-0013). 그레이박스 라벨 → 도트 재도색(Phase 2.7).

## 3-7. UI 요소

- **스타듀 디자인 관습:** 따뜻한 나무/양피지 톤 프레임, 핫바 12칸, 우상단 시계/날짜/금화, 좌하단 에너지/체력 바, 대화창·상점·인벤토리·달력·도감 패널. 픽셀 폰트.
- **나라카 변형:** 공통 인벤토리 모듈 + 컨텍스트 스위칭(상단 레이어 교체, Phase 2.8 §3.4). 혼력 바(우하단) + 체력 바 자리 예약 + 핫바 12칸. 960×540·UI scale 1.5(ADR-0018). 저승 톤 프레임.
- **파이프라인:** Godot UI + 도트 에셋. Phase 2.8 HUD·공통 모듈.

## 3-8. 애니메이션 / 이펙트

- **스타듀 디자인 관습:** 도구질(괭이·물·도끼) 임팩트, 작물 수확 팝, 날씨(비·번개·눈), 계절 전환, 물고기 튀김, 전투 히트, 축제 연출. 짧고 읽기 쉬운 프레임.
- **나라카 변형:** 여우불(미호 농사 XP 당근 연출, ADR-0021), 혼불/도깨비불, 잡귀(나락), 절기 전환, 테마 데이 장식. [그날 밤] 대화재는 컷신(인스타툰체).
- **파이프라인:** PixelLab 애니메이션 + Aseprite. 컷신=미드저니/제미나이.

## 3-9. BGM

- **스타듀 디자인 관습:** 계절별 낮 테마 + 장소별(마을/광산/해변) + 이벤트/축제 + 메뉴. 차분한 어쿠스틱·칩튠 혼합. ~50곡 규모.
- **나라카 변형:** 절기별 낮(밭) + 밤(나라카 바) + 장소별 + 테마 데이 변주 + 컷신. 첫 슬라이스는 최소(낮 밭/밤·메뉴 1~2곡, CONTEXT 에셋 파이프라인).
- **파이프라인:** 생성형 음악(Suno 등 상업 플랜). ⚠️ 상업 약관 확인(licensing-checklist).

## 3-10. SFX

- **스타듀 디자인 관습:** 도구질·물소리·수확·발걸음·UI 클릭·물고기·전투·동물·날씨 등 수백 종. 짧고 만화적.
- **나라카 변형:** 동일 카테고리 + 저승 톤(여우불·혼불·잡귀). 흔한 소리(괭이질·물·UI)는 로열티프리 팩이 압도적으로 빠름(CONTEXT).
- **파이프라인:** 로열티프리 에셋팩 중심 + 필요 시 생성형. ⚠️ 약관 확인(licensing-checklist).
