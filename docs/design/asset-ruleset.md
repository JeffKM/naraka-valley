## 0. 그리드 모델 (토대 — owner 잠금 2026-06-30)

**1 타일 = 16px 논리(authoring) 그레인 → 정수 ×2 nearest → 32px 화면.**

- **저장/렌더 = 32px native PNG, 1:1**(ADR-0013 그대로 — ×2를 PNG에 *구워서* 저장, 엔진 `TILE=TILE_ART=32`).
- **authoring 그레인 = 16px 논리** — PixelLab은 목표의 절반 해상도로 생성(예: 9칸 건물 = 288px 목표 → 144px 생성), `nearest` ×2로 32px 파일에 굳힘. 화면엔 2px 블록 = 스타듀식 굵은 도트.
  - 32px 미만이라 절반 생성 불가한 소형(≤32px 목표)은 동일 크기 생성 후 **÷2(BOX)→알파 임계→×2 nearest** in-place 청키화.
  - 글루: `tools/process_chunky_phaseC.py`.
- **owner의 "16px" 규칙(피벗·footprint 등)은 전부 이 *논리 타일 단위*를 가리킴**(= 화면 32px 1칸 = 논리 16px).
- **적용 범위 = 모든 시각 에셋(타일·객체·캐릭터 포함, 예외 없음)** — 캐릭터도 월드와 같은 도트 밀도(§10.2).

> 이유: 풀해상도(1아트px:1화면px) 생성은 픽셀이 곱아 RPG MV/MZ 룩이 된다(owner 다회 지적). 16-논리 ×2로
> 굵은 핸드메이드 레트로 도트(스타듀)를 강제. ADR-0012(16×2)와 ADR-0013(32native)을 "16논리 저장은 32"로 화해.

### 0.1 ★캐논 청크 단위 = "1 논리px = 2 화면px (2px 블록)" — 전 에셋 보편 강제 (owner 잠금 2026-07-01 grill)

> **한 화면의 모든 픽셀은 같은 그레인이어야 한다.** 건물·바위는 2px 블록(청키)인데 타일·캐릭터·프롭·절벽이
> 1px(고움)로 섞이면 부자연스럽다(owner 지적, 인게임 스크린샷). → **2px 블록을 유일 캐논 단위로** 고정.

- **적용 = 예외 없이 전 에셋** — 타일(터레인 아틀라스·변종·전이 포함)·캐릭터(walk 시트 전 프레임)·프롭·가구·절벽·debris·작물·아이콘·UI. **앞으로 생성·배치되는 모든 것** + 기존 1px 자산 소급 청키화.
- **검증법(정량):** 2×2 블록 비율(픽셀이 오른·아래·대각 이웃과 같은 색으로 2×2를 이루는 비율) **≥ ~95%**면 2px 청키. **< ~70%면 1px 고움 → 재청키화 대상.** (건물·바위 = 100%가 기준.)
- **글루:** `tools/enforce_chunk.py` — 자산을 스캔해 1px(저비율)만 **÷2(BOX 평균)→알파 임계→×2 nearest**로 청키화(이미 2px인 것은 건너뜀, 이중청키 방지). 생성 파이프라인(§0 half-res·`process_chunky_phaseC.py`)과 동일 결과.
- **생성 시:** PixelLab은 목표의 절반 해상도로 생성 후 ×2(§0). 절반 생성 불가한 소형은 동일크기 생성 후 ÷2→×2.
- **타일 함의:** 터레인 아틀라스도 2px 청키 → [ADR-0043] 부드러운 지면이 약간 거칠어짐(owner 수용 — 일관성 우선). 런타임 `_harmonize`·`_ground_detail`은 타일 데이터 기반이라 아틀라스 픽셀만 바뀌면 자동 정합.
- **캐릭터 함의:** 인게임 얼굴 디테일이 2px로 뭉개짐 → **초상화(ADR-0026 #3)가 표정을 전담**하므로 허용(§10.2 부채 청산).

## 1. 광원(Light Source) 일관성 — NW 빛 / SE 그림자 (owner 잠금 2026-06-30)

**모든 독립 오브젝트(건물·프롭·가구·작물·기계·바위)의 자체 음영은 좌상단(NW)에서 빛, 그림자는 우하단(SE)으로 통일.**

> 이유: 오브젝트마다 광원이 제각각이면 한 맵에 모았을 때 따로 놀며 둥둥 뜬다(언캐니). 작은 상자 하나까지 SE로 그림자가 떨어지게 깎는다.

### 1.1 프롬프트 강제 세트 (도트 제어 키워드 — owner 제공)
단순 "light from top-left"는 림라이트·그라데이션으로 쯔꾸르 회귀 위험 → 아래 기술 키워드 세트로 묶어 강제:

```
Flat 2D pixel art, light source from top-left (NW), distinct directional step-shading,
1px highlight on top and left edges, crisp dark shadows cast to bottom-right (SE),
strict adherence to 2-3 color values max, no smooth gradients.
```
→ 거친 레트로 도트(low detail)를 유지하면서 좌상단 엣지에만 1px 하이라이트가 감기는 정통 스타듀식 프롭.

### 1.2 기존 자산 광원 감사(Audit) — 2단계 필터(완주 우선, ADR-0028)
전수 재생성은 완주 지연 리스크라 압축:
- **1단계(즉시 검수): 대형 에셋만**(건물·대형 바위) — 광원 뒤틀리면 바로 띈다. 좌상단 엣지를 1px 밝은 톤으로 리터칭.
- **2단계(패스): 1타일(16×16 논리) 이하 소형 프롭·작물** — 픽셀 밀도가 낮아 유저가 광원 뒤틀림을 거의 인지 못함. 교정 안 하고 **새 생성분부터 철저 통제**.

### 1.3 그림자 = 분리 오버레이 레이어 (굽지 않음 — ★접지 해법)
- 평면 풀·흙 타일은 방향성 없음(예외). **단, 절벽 단면·건물 바로 아래 바닥은 예외**로 그림자 필요.
- **에셋에 큰 그림자를 굽지 않는다** — 타일 배치 바꿀 때 그림자가 잘려 버그. 대신 **반투명 블랙(#000000, 알파 ~30%) 그림자 타일을 별도 오버레이 레이어**로 SE 방향 1~2칸 깐다.
- → NW 광원 시스템 유지 + 배치 100% 자유. **이 분리 그림자가 건물 접지(§ 접지 규칙)와 WALL 패드 문제를 동시에 해결**(건물 밑에 SE 그림자 오버레이 → 떠 보이지 않고 바닥에 앉음).

**★두 종류 그림자 구분 (owner 2026-07-01 grill — 혼동 방지):**
- **(a) 지면 접지/투사 그림자** = 오브젝트가 *바닥에 지는* 그림자. 배치 따라 이동 → **절대 스프라이트에 굽지 않고 별도 오버레이**(위 규칙, 반투명 블랙 SE).
- **(b) 자기(self) 그림자** = 처마가 *제 벽면에* 지는·문이 안으로 쑥 들어간(recessed) 음영 등 **스프라이트 고유 형태의 일부**. 배치와 무관 → **구워도 됨**(§1 step-shading의 일부, [ADR-0036] §4).
- **자기그림자 색 = 재질별**(owner 확정): **warm 건물(주거·농장)=꿀빛 목재 램프의 어두운 끝(dark warm brown)** / **저승 muted 객체(절벽·바위·저승 구조물)=차가운 청보라-슬레이트**. warm 건물에 보라 그림자 금지(따뜻함이 식음).
- **★건물 base = 투명(구워진 지면·잔디 금지):** 건물 아트 하단에 지면·잔디를 굽지 않는다 — 맵 지형과 톤이 달라 "뜬 초록 패치"로 단절된다(owner 2026-07-01 지적). 실제 지형이 비쳐 자연 연결되게 base를 투명 처리(glue: green-strip). 접지는 (a) 오버레이가 담당.

## 2. 건물 방향 — 정면 Facade + 남향 진입 ([ADR-0036] 잠금 2026-06-30)

- **정면(Front-Facing) Facade**: 정면 벽 평면 렌더 + **지붕 Y축 깊이 1~2 논리 타일만** 노출(2.5D 착시). **측면 벽 렌더 금지**(이소메트릭 금지).
- **★지붕 윗면 슬랩 노출 = 필수 검증 (owner 잠금 2026-07-02)**: 정면 지붕 경사면 위에 **하늘을 보는 윗면 슬랩(뒤로 물러나는 깊이면)이 반드시 노출**돼야 한다(본가 지붕이 기준). "정면 삼각형 실루엣만 있고 윗면 0" = **리젝 → 재생성**. 육안 확인(home_dump)에서 윗면이 안 보이면 통과 불가.
  - **생성 파라미터: `view="low top-down"`**(윗면 노출용 — 기존 `view=side`는 윗면이 안 나와 폐기). + §1.1 광원 + `"front-facing facade, facing camera, NOT isometric, NOT angled, symmetrical front elevation, flat roof-top slab receding behind the ridge (roof depth visible)"`.
  - **감블(곡면 gambrel) 지붕 금지** — PixelLab이 곡면 지붕의 윗면을 못 그린다(축사 v1~v4 실패). **박공(gable, 삼각 경사) 지붕**을 써야 윗면이 나온다(본가·창고 검증). 축사도 박공 + 넓은 슬라이딩 도어로 정체성 구분.
  - **선명도: half-res 네이티브 생성 필수** — PixelLab 캔버스를 목표의 절반(예: 6칸 창고=96px)으로 잡아 생성 → `tools/facade_halfres_x2.py`(×2 nearest)로 굳힘. `place_facade.py`(풀해상 → ÷2 청키화)는 **금지**(÷2가 기와·판자를 뭉개 흐릿해짐 — [§0.1] 위반, gemini-shed-barn-spec [REVISION 3] 참조).
- **남향 고정**: 모든 상호작용 문 = 남쪽(아래), 진입 벡터 북향(아래→위).
- **DOOR 1:1**: art 문 중심 픽셀 열 = `*_DOOR` 타일 열(좌우대칭 facade라 중앙). `_carve_paths`가 그 좌표 남쪽 인접까지 경로 생성. (안식 정합: 본가 문열 44 / 창고 30 / 축사 6 — rect 중앙·하단.)
- **입체감** = 지붕 깊이 + NW 음영(§1) + 앞쪽 풀/덤불 Y-Sort 겹침(이소메트릭 아님).

## 3. 피벗/앵커 체계 — 배치 컨텍스트별 기준점 ([ADR-0037] 잠금 2026-06-30)

| 배치 컨텍스트 | 피벗/앵커 | 정렬 결과 |
|---|---|---|
| **바닥 배치**(건물·바위·그루터기·바닥가구·작물·기계) | **Bottom-Center** | art 바텀 = 타일 하단 경계 → **flush 접지**, art는 위로 솟음(지붕·키큰 부분은 위 타일로 오버행 → Y-Sort) |
| **벽 부착**(창문·달력·벽선반) | **호스트 벽의 안쪽 면**(아래 표) | 벽에 딱 붙고 방을 향함 |
| **들고있는 도구/장착물**(괭이·낚싯대) | **Center** | 16×16 또는 16×32 프레임 정중앙 → 스윙 회전 시 손에서 안 떨어짐(Phase 3 도구 애니 대비 선제 규격) |

**벽 부착 = 어느 벽인지(N/E/S/W)까지 따짐:**

| 호스트 벽 | 앵커 | facing | 비고 |
|---|---|---|---|
| **북(상단) 벽** | art 바텀을 벽 띠 하단 모서리에(현 `WALL_PROP_LIFT`) | 남(뷰어 향함) | **실사용 기본** |
| 서/동 벽 | 그 벽 안쪽 세로 모서리 | 동/서(방 안으로) | 측벽 가구 시 |
| 남(하단) 벽 | art 탑을 남벽 상단 모서리 | 북 | 드묾(가림 주의) |

**구현:** 프롭 데이터 테이블 각 엔트리에 **pivot 타입**(`floor`/`wall:N|E|S|W`/`held`) 필드 → 그리기·충돌이 그 앵커를 씀. 임시 `WALL_PROP_LIFT(-18)`를 `wall:N` 케이스로 흡수. facade 블릿을 **bottom-center 앵커**로 변경(현 top-left → art 바텀=footprint 하단행). 지금은 N벽만 실사용(E/S/W 필드 예약).

> **★야외 바닥 프롭 = 발치 기준(foot-referenced)으로 등가 충족(Slice 0 종료, owner 확정 2026-07-01):** facade는 위 `_blit_facade_anchored`로 bottom-center 앵커 적용 완료. **야외 바닥 프롭(나무·바위·debris·그루터기 등)은 좌표를 `top-left`로 두되, 발치(base=`tile.y*TILE + yo + art_h`)를 기준으로 [§5 발치 충돌 바]·[§6 Y-split base_y]·[§11 접지 그림자]가 전부 그 발치를 참조**하므로, 피벗/충돌/그림자가 발치 정렬이라는 §3의 *목적*이 충족된다(인게임 접지 정확·⑦/S0-8 사인오프). 좌표 표기를 foot-tile로 바꾸는 리터럴 재배치는 **시각 결과 동일(payoff 0) + 회귀 리스크**라 하지 않는다. **신규 프롭 배치·후속 pivot 필드 도입 시엔 foot-referenced를 기준으로 설계**한다(이 등가가 규범).

## 4. 방향성 에셋의 면별 변형 (Directional Variants) ([ADR-0037] 잠금 2026-06-30)

> **모서리·면을 가진 방향성 에셋(절벽·벽·울타리·다리 등)은 면 방향마다 *올바르게 향하는* 변형을 따로 생성한다.** 한 면을 전 모서리에 복붙하거나 식생으로 가리는 핵 **금지**.

- 필요한 면: **남(정면 face)·동·서·북(상단 lip)** + 외부/내부 코너.
- **flip 허용은 진짜 거울대칭 쌍만**(좌코너↔우코너). 단 **NW 광원(§1) 때문에 동면↔서면은 음영이 달라** 단순 flip이 아니라 **광원 재보정**(밝은 엣지가 항상 좌상단)이 필요.
### 4.1 다단 절벽 pseudo-Z 규격 (H=2) ([ADR-0044] 잠금 2026-07-01)

> 절벽은 **1타일 돌띠가 아니라 2행 pseudo-Z**로 "벽처럼" 읽혀야 한다(owner: "맵이 단조롭다"). z축 아님 — [ADR-0013] 2D 평면 데이터·세이브·카메라 불변, 타일 셋업으로만 고저차 연출.

- **표준 수직 구조(남향, H=2행=64px):** `PLATEAU_GRASS`(고지 풀, 걷기 O) → **`CLIFF_LIP`**(고지 밑단, **걷기 O** · 밝은 상단 라인 · 고지 오브젝트 배치 하단 한계선) → **`CLIFF_FACE`**(절벽 상단, **SOLID**) → **`CLIFF_FACE_BASE`**(절벽 접지, **SOLID** · 접지 그림자 self-shadow 베이크 = §1.3(b)) → `LOWER_GRASS`(저지 풀, 걷기 O).
- **H=2 마스터 표준**(Face+Base), 구역별 상향 가능 파라미터(하한 2).
- **동/서향 = 높이의 가로 치환:** 세로 런은 H를 *너비*로 바꿔 `Lip 1열 + Face 2열`. 단순 flip 금지 · **NW 광원 재보정 필수**(밝은 엣지 항상 좌상단, 위 규칙).
- **계단 절개:** 절벽 종단 계단 = `y+1`·`y+2` SOLID 일시 해제 **2폭 세로 트리거**(걷기 O).
- **코너 = edge-to-edge 마감:** 고지/저지 엣지를 완전히 채운 외/내 코너 타일로 초록 새어나옴 방지. **넝쿨·덤불 seam 가림 핵 폐기**(ADR-0035/phaseB 개정) — 넝쿨은 순수 장식으로만 강등.
- **물가 변종:** 강물 상단 `CLIFF_FACE` 강둑 단차 ≥1행 · 바다 진입부 수평 절벽 런(고지 수풀↔백사장). [ADR-0044] §2.
- **구현 이관:** 안식 농원=Slice 1 · 나루=Slice 2 · 삼도천/황천해=Slice 3.

## 5. footprint & 충돌체 — "머리는 통과, 발만 막는다" ([ADR-0038] 잠금 2026-06-30)

> **충돌 박스 ≠ 아트 실루엣.** 아트는 크고 높아도(실루엣), 충돌은 발치만 막아 플레이어가 뒤로 지나가게 한다.

- **충돌 = 밑단 바(bottom bar) = 16×8 논리**(타일폭 × 반높이), 오브젝트 발치에 둠. 키 큰/부피 큰 프롭·바위·나무·파괴가능 객체(업화석·석화고목) 공통. 위쪽은 통과(머리 겹침 → Y-Sort §6).
- **작물·기계 추가 strict:** 최종 성장형 아트의 **가로 폭이 타일 경계(16 논리)를 1px도 넘지 않음**(세로로 크는 건 OK). 옆칸 클릭미스·Y-Sort 깜빡임 방지.
- **구현:** 현 `SOLID_PROPS` 풀타일 충돌 → **서브타일 16×8 도형**으로 점진 전환(키 큰 프롭부터, 작물·파괴가능은 해당 메카닉 빌드 시). 현 풀타일 하드게이트 debris(업화석·석화고목)는 개간 Slice 1에서 전환.

## 6. Y-Sort 깊이 정렬 — "머리는 통과" 의 시각 짝 ([ADR-0037] 입체감 산하)

> 충돌(§5)이 발만 막는다면, Y-Sort는 *머리가 뒤로 겹쳐 지나가는 것을 보이게* 한다. 둘은 짝.

- **규칙:** **base 피벗 Y로 깊이 정렬** — 플레이어 발치 Y가 프롭 피벗(발치) Y보다 *작으면(위)* 프롭이 앞(플레이어 머리 가림), *크면(아래)* 플레이어가 앞. 키 큰 나무·건물 지붕 뒤로 자연스럽게 지나감.
- **구현 = 수동 Y-split**(owner 잠금): 현 즉시모드(`main._draw()`) 유지하되, 프롭을 플레이어 피벗 Y 기준 둘로 나눠 **[뒤 프롭 → 플레이어 → 앞 프롭]** 순서로 그림. (노드 Y-Sort 전환은 과한 리팩터라 보류.)
- **새 ADR 없음** — [ADR-0037]의 "입체감 = Y-Sort 겹침" 산하 구현 규칙.

## 8. 렌더 안정성·특수 연출 규격 (Render & Shader) ([ADR-0039] 잠금 2026-06-30)

- **8.1 알파값 제어:** 에셋 스프라이트 픽셀은 **풀 불투명 OR 풀 투명(하드 알파)** — 반투명 안티에일리어싱 엣지 금지(헤일로 방지). 후처리에서 알파 임계(threshold). **플레이어 가림 투명화는 베이크하지 않고 엔진 Modulate/셰이더에 위임**(런타임). ※§1.3 그림자 레이어(반투명)는 스프라이트가 아닌 별도 오버레이라 예외.
- **8.2 전이 픽셀(Seam Transition):** **타일·절벽 면 한정** — 인접 지형과 결합 위해 외곽 **1~2 논리px에 2px 디더(체커) 마진** 강제(알파 블렌드 아님 — 청키·하드알파 정합). 독립 프롭은 불필요.
- **8.3 발광 마스크(Emission Mask):** 야간 발광 부위(건물 창·영혼빛 오브젝트)는 생성 시 **별도 레이어/마스크로 분리 추출**(`*_emit.png`) → 야간 라이팅(Phase 3-6 절기·시간) 때 재작업 0. 본가 앰버 창이 첫 대상.

## 9. 팔레트 정책 — 따뜻한 베이스 + 저승 에셋 (owner 잠금 2026-06-30, [ADR-0035] 개정)

> **따뜻한 농장 위에 음울한 저승 구조물.** 농장 맛(스타듀 따뜻함)은 *바닥*에서, 저승 정체성은 *그 위 객체*에서.

- **베이스 = 걸어다니는 땅 지형**(풀·흙길·밭흙·자갈·모래·숲바닥) → **"살짝 가라앉힌 warm"**(스타듀 따뜻한 녹색·갈색에서 채도·명도만 소폭 down — 캔디처럼 쨍하지 않게). 따뜻한 농장 느낌 + 음울한 세계 공존.
- **물(연못·강·바다) = 저승 에셋 취급** → 영혼빛 파랑 액센트(tileset-ruleset §3.4b). 따뜻한 베이스에서 **제외**.
- **모든 독립 오브젝트**(건물·나무·바위·debris·가구·작물·기계) → **저승 muted desaturated**(§1 광원 + 공유 꼬리).
- **함의:** tileset-ruleset 마스터 팔레트(현 풀 `#306033` 어두운 이끼)를 따뜻한 녹색으로 상향 → **바닥 타일 warm+청키 재생성** 필요. ADR-0035 §3.4 "전부 muted" → 객체용/베이스용 분리.

## 10. 스케일/비례 일관성 + 도트 밀도 통일 (owner 잠금 2026-06-30)

**10.1 스케일 anchor:**
- 기준 = **캐릭터 ≈ 2 타일(~64–70px) 높이**(char_sprite FRAME 80px·콘텐츠 ~70px). 모든 에셋을 이 기준에 비례.
- **문 높이 ≥ 캐릭터 키**(캐릭터가 통과하는 게 보이게). 가구·작물은 캐릭터에 그럴듯하게 비례, 건물은 6~8타일로 압도(정상).
- 16-논리 공유 그리드라 상대 스케일은 자동 — 단 **생성 검수 시 캐릭터 실루엣을 옆에 놓고 비율 확인**(AI 내부 비율 오류 방지).

**10.2 도트 밀도 통일 (★캐릭터 예외 없음):**
- §0 청키 그레인은 **캐릭터에도 적용** — 캐릭터·월드가 **같은 도트 밀도**(2px 블록). "주인공 디테일 예외" 폐기(owner: 한 화면에서 캐릭터만 매끈하면 따로 놂).
- **부채:** Slice 0 캐릭터 5종(+walk)은 현재 1:1 fine(80px) → **PIL 청키화(÷2 BOX→알파임계→×2 nearest, 프레임 80px 유지)** 로 그레인만 맞춤(재생성 X, 디자인·애니 보존). 인게임 얼굴 디테일 손실은 초상화(ADR-0026 #3)가 담당하므로 허용.

## 16. 마스터 팔레트 램프 (고정 공유 팔레트) ([ADR-0041] 잠금 2026-06-30)

> 통일은 *방향*이 아니라 *고정 색*에서 나온다. AI가 생성마다 제 색을 고르면 통일이 깨짐 → 재질별 램프를 잠그고 생성 후 그 팔레트로 양자화.

  - **저승 객체**: 슬레이트 청회 · 목재 갈회 · 석재
  - **영혼빛**: `#60d8f0→#2068e8`(물·영혼·차가운 발광)

## 17. 가독성/대비 — 게임플레이 pop (owner 잠금 2026-06-30)

> 통일(§9·16)과 별개로, **게임플레이 중요 오브젝트는 배경에서 눈에 띄어야** 한다. 저채도 세계라도 채도가 아니라 **명도(value)로** 구분.

- **중요 오브젝트(작물·드롭 아이템·인터랙터블)는 배경 타일과 명도 대비 보장.**
- 저승 객체는 보통 밝은 warm 풀 위 어두운 저채도라 실루엣으로 잘 튐(dark-on-light). **위험 = dark-on-dark**(어두운 밭/흙 위 작물, 풀 위 작은 드롭):
  1. **작물** = 어두운 밭 위에서 **더 밝거나 영혼빛 톤**으로(dark-on-dark 회피).
  2. 단일 외곽선(`#401818`, §1)이 형태를 배경에서 뗌.
  3. **드롭템·인터랙터블** = §11 접지 그림자 + 필요 시 **미세 림/영혼빛 글로우**.
- **검증:** §12 실제 줌에서 "작물이 흙 위에, 드롭템이 풀 위에 한눈에 보이나" 확인.
