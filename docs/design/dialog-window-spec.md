# 대화창 「태운 한지」 프레임 규격 (S0-6)

> **워크플로우:** owner가 제미나이로 프레임 아트 1장 생성 → `make_dialog_window.py`(글루: 배경제거·크롭) → `assets/ui/dialog_window.png` → `portrait_preview.gd`가 측정한 내부 칸 프랙션에 초상화·본문·이름 오버레이. (ADR-0001 허용, [ADR-0025] 생성 전 규격 게이트.)
> 관련: 초상화 자체 규격은 [portrait-spec-card.md](./portrait-spec-card.md).

## 개정 이력
- **2026-07-01 (2) — 현재 채택 = B(기존 칸에 여백 없이 꽉).** "큰 초상화 + 옛 작은 프레임이 따로 논다" 지적 → 프레임 재생성(아래 A) 전까지는 **기존 프레임 칸을 그대로 두고 초상화만 여백 없이 채움**으로 일관성 확보. 코드 변경: 초상화 매팅(grow −5) 제거 + `STRETCH_KEEP_ASPECT_COVERED` + `clip_contents`(칸을 꽉, 넘침 클립). 레이아웃 상수(F_PORT/F_TEXT/F_NAME)·하단 이름판은 원래대로.
- **2026-07-01 (1) — 초상화 칸 대형화(A안, 향후 업그레이드 옵션·보류).** "초상화가 몰입의 핵심인데 너무 작다" → 우측 칸을 폭 ~28%·높이 ~82%로 키우는 안. 초상화 에셋은 이미 칸을 꽉 채우고 있었고(내용이 320²의 폭 100%·상하 여백 22px뿐), **병목은 칸이 윈도우 폭 15%뿐**이던 것. **owner가 아래 프롬프트로 큰-칸 프레임을 재생성하면** 이 안으로 전환(그때 F_PORT=`(0.6979,0.0889,0.2764,0.8222)`·본문 폭 0.608·이름은 칸 하단 리본). 프레임이 제미나이 아트라 코드만으론 불가해 보류.

## 레이아웃 (프레임 텍스처 = 윈도우 프랙션)

프레임 전체를 윈도우 Rect(논리 608×176, CanvasLayer scale 1.5)에 STRETCH. 내부 칸은 프레임 텍스처의 아래 프랙션 위치에 그린다 → **owner는 아트에서 각 칸을 이 프랙션에 정확히 맞춰 그릴 것**(그래야 코드 오버레이와 정렬, 재측정 불필요).

| 칸 | 프랙션 Rect2(x, y, w, h) | 용도 |
|---|---|---|
| **초상화 칸** `F_PORT` | `(0.6979, 0.0889, 0.2764, 0.8222)` | 우측 대형 초상화. 초상화 내용 비율 **1.16:1**(320:275)에 맞춰 칸도 1.16:1 → 꽉 참 |
| 이름 리본 `F_NAME` | 초상화 칸 하단 오버레이(코드가 그림, 반투명 먹빛 띠 + 가운데 이름) | 화자 이름 |
| 본문 칸 `F_TEXT` | `(0.0523, 0.1372, 0.6080, 0.7306)` | 좌측 대사(초상화 칸과 안 겹치게 폭 축소: 우끝 0.66) |

## 제미나이 프레임 프롬프트 요지 (재생성 시)
- **가로형 「그을린 한지」 대화창 통짜.** 따뜻한 종이 바탕 + 가장자리 탄 자국 + 먹 나비 장식(좌상단). 기존 룩 유지.
- **좌측 ~62% = 본문 패널**(빈 한지, 텍스트는 게임이 얹음).
- **우측 ~28% 폭 × 거의 전체 높이 = 큰 정사각~약간가로(1.16:1) 초상화 칸.** 테두리는 한지·먹 톤. **이 칸이 주인공 — 크게.**
- **초상화 칸 하단에 얇은 이름 리본**(이름은 게임이 얹음) 또는 칸에 통합. 기존의 작은 정사각 초상화칸 + 별도 하단 이름판은 폐지.
- 배경 투명(안 되면 무채색 회색 체커 — 글루가 제거).
- 규격: 가로 ≥ 2800px(다운스케일 여유), 최종 `dialog_window.png`는 폭 1400로 정리됨.
