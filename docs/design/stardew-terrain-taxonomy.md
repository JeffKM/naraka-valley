# 스타듀 지형 변주 taxonomy — deep-research 결과

> **목적:** 안식 농원 등 지상 구역의 "자연 지형 변주를 더 풍성하게" 만들려면 무엇이 필요한지, 스타듀밸리가 *실제로* 어떻게 하는지 로컬 위키 미러(`docs/reference/stardew-wiki/`, 1,999장) + 웹 리버스엔지니어링으로 조사한 결과. 2026-07-17.
> **핵심 결론 미리:** 스타듀는 **WFC도, 하드 인접-제약 솔버도 쓰지 않는다.** 자연스러움은 [지형-타입 유효성 마스크 + 구역·계절별 가중 스캐터 테이블 + 풀의 확률적 이웃-확산 + 충분한 변종 수]에서 나온다. WFC 인접규칙 엔진은 이 목표엔 **오버킬·오프모델**.

## 2. 분포 메커니즘 (어떻게 배치하나) — WFC 아님

- **Forage/debris = 독립 균일 랜덤 롤 + 거부표집.** `GameLocation::spawnObjects`가 밤에 맵 전역에서 **완전 랜덤 좌표 타일을 뽑아** 유효 지형이면 배치. **공간 상관 0** — 뭉쳐 보이는 건 유효 타일이 몰린 자리의 창발적 산물이지 클러스터 규칙 아님. 맵당 주 6개 상한.
- **유일한 "인접-비슷" 게이트 = 지형-타입 유효성**(칸별 테스트). 대부분 맵: 초록(잔디형) 타일에만. 해변/사막: 모래 타일. 점유 타일 제외.
- **Weeds 확산:** 방치 잡초가 **인접 농사가능 타일로 확률 번식**(계절 1일·비·여름에 ↑). 단순 "빈 이웃으로 롤" — 패턴 검증 아님. Gold Clock = 전역 억제.
- **Grass 확산 = 유일한 진짜 이웃 메커닉, 그러나 그냥 확률.** 각 풀타일 4 tuft, 매일 65% 성장, **4 tuft 만개 시에만** 직교 4이웃 중 **농사가능 흙**에 25% 확률로 새 tuft 파종. **기존 초록엔 확산 안 함**(→플레이어가 source 패치를 울타리로 가둬 재배). Spring 1(2년차+)엔 40회 iterate 벌크필. 확률적 셀룰러오토마타 확산이지 **WFC/변종-합법성 검사 없음**.
- **구역/계절 가중 테이블 = 있음.** 맵마다 유효 타일 비율·forage 리스트·debris 스케줄이 다름(`Content/Locations.xnb`). Forest Farm·Hilltop 채석장은 아예 별도 스포너.

> **VERDICT:** 스타듀 지면 자연스러움 = [지형-타입 유효 마스크 + 구역·계절별 가중 랜덤 스폰 테이블 + 풀(과 약한 잡초)의 확률적 이웃-확산 + 넉넉한 변종]. **WFC·하드 인접규칙·전역 패턴 검증 전무.** 스타듀식 룩에 WFC 기계는 오버킬.

## 3. 변주 레버 (풍성함 순위) — 우리 설계의 우선순위 지도

1. **계절 타일시트 스왑** — 봄초록/여름마름/가을주황/겨울눈 4벌을 28일 주기로 통째 교체(필터 아님). `Seasons.wikitext` → **Slice 7(계절) 영역**
2. **구역·계절별 가중 스캐터 테이블** — 각 구역 고유 forage/debris 정체성(공유 풀 아님). ← *현재 게임이 가장 덜 쓰는 레버 = "심심함"의 주범*
3. **구역별 상이한 base 지형** — 잔디 vs 모래 vs 돌 vs 물
4. **타일당 절차적 풀** — 4 tuft + 매일 성장/확산으로 필드가 매일 다름
5. **계절 debris/disrepair 주기** — 겨울 소멸·Spring 1 재출현으로 가시적 순환
6. **손배치 매크로 피처** — 절벽·연못·조수웅덩이(그리드 깨기) ← 이미 저작
7. **플레이어 저작 바닥/길 레이어** — 13종 오토타일·풀 재성장 억제 ← Wang([ADR-0057])이 유사
8. **계절 한정 clutter 악센트** — 특정 계절에만 나오는 spice 아이템

## 4. 우리 코드와의 매핑 (현재 상태 진단)

| 스타듀 레버 | 현재 `main.gd` 구현 | 격차 |
|---|---|---|
| 지형-타입 유효 마스크 | `_g16_blend_scatter`가 GROUND/PATH만·점유 회피 | ✅ 있음 |
| 풀 이웃-확산(CA) | `_g16_cluster_cleanup`(잔디<2 사멸·흙≥5 생성) | ✅ CA 있음(현재 `_G16_GRASS_PATCHES=false`로 휴면) |
| 저주파 클러스터 | `_gd_cluster`(bilinear 블록해시) | ✅ 있음 |
| 가중 스캐터 롤 | `_GD_TABLES`/`_GD_SPARSE` | ⚠️ **단일/전역** — 구역·계절별 분화 없음 |
| **구역·계절별 테이블** | 없음 | ❌ **최대 격차(레버 #2)** |
| **변종 수** | 소수 | ❌ 레버 #4·#1 — 병렬 아트 트랙 |
| 계절 스왑 | 없음(Slice 7) | ❌ Slice 7 |

> **결론:** "자연 지형 변주를 더 풍성하게"의 실효 레버는 **[구역·계절별 가중 테이블 도입] + [변종 확대]**이지 WFC 인접규칙 엔진이 아니다. 이미 있는 CA(풀 확산)·유효마스크·클러스터는 스타듀 레시피와 정합 — 되살리고 데이터를 분화하면 된다.
