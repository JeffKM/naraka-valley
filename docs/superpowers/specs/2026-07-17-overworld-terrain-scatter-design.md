## 1. 목표와 배경

owner의 요청: **"안식 농원 등 지상 구역의 자연 지형 변주를 더 풍성하게."** 지금 라이브 지면이 부분적으로 **부자연스럽거나 심심하게** 보인다.

출발점은 WFC(파동함수붕괴)였으나, **deep-research 결과 스타듀밸리는 WFC도 하드 인접-제약 솔버도 쓰지 않는다**는 게 세 독립 리서치에서 수렴 확인됨(taxonomy §2 VERDICT). 스타듀 지면 자연스러움의 실체:

1. **지형-타입 유효 마스크**(칸별 "여긴 스캐터 가능 지형" 테스트)
2. **구역·계절별 가중 스캐터 테이블**(각 구역 고유 clutter 정체성)
3. **풀의 확률적 이웃-확산**(4 tuft·만개 시 흙 이웃에 25% 파종) — 유일한 정당한 인접 메커닉
4. **넉넉한 변종 수**

owner가 느낀 "심심함/부자연"의 진단(taxonomy §4): 현재 `_GD_TABLES`가 **단일·전역**이라 구역마다 clutter 믹스가 비슷하고(레버 #2 미사용), **변종 수가 부족**(레버 #1·#4)하다. 이미 있는 CA(풀 확산)·유효마스크·클러스터는 스타듀 레시피와 정합.

**설계 결정(owner 승인 2026-07-17):** WFC 인접규칙 엔진을 짓지 않고, **스타듀-정통 스캐터**로 간다.

## 2. 무엇을 짓나 (3부)

### A. 구역-키드 가중 스캐터 테이블 — *심심함의 최대 레버*
현재 `main.gd`의 `_GD_TABLES`(전역 `{GROUND, PATH: [[tex, weight, shadow]...]}`)와 `_GD_SPARSE`를 **구역별로 분화**한다.

- 새 구조: `_REGION_GD_TABLES := { "home": {GROUND:[...], PATH:[...]}, "jeoseung_forest": {...}, ... }` + 구역별 `_REGION_GD_SPARSE`.
- 폴백: 구역 키 없으면 현행 전역 테이블(= 기본값) 사용 → **점진 이관·회귀 0**.
- `_g16_blend_scatter(out)`가 `_region`으로 테이블을 골라 쓰도록 조회 지점만 교체(로직 불변).
- **안식 농원**은 자기 테이블을 받아 지금보다 풍성한 믹스로 튜닝. **타깃(owner 2026-07-17):** *지금보다 풀무리(풀 tuft 클러스터) 증가.* ⚠️ 단 이는 **스캐터 tuft 클러스터 밀도·가중 ↑**이지 **base 잔디패치 부활 아님** — 흙-지배 base([ADR-0053]) 유지·`_G16_GRASS_PATCHES=false` 유지(스타듀 농장도 tan-지배 base + 풀 tuft 무리 산포). 구조가 서면 다른 구역은 지어질 때 각자 테이블을 채운다(숲=나뭇가지↑, 물가=이끼, 광산입구=돌↑ 등 — 구역 정체성).
- **밀도·클러스터 레버**(`_GD_SPARSE_DENSITY`·`GD_CLUSTER_CUT`·`GD_CLUSTER_BLOCK`)도 구역별 오버라이드 허용.

### B. 풀 이웃-확산 CA 되살리기 — *유일한 정당한 인접 메커닉*
`_g16_cluster_cleanup(surf)`(잔디<2 사멸·흙≥5 생성 CA)는 이미 있으나 현재 `_G16_GRASS_PATCHES=false`로 **휴면**. 스캐터-only 스타듀 룩(2026-07-16 P2)과 병행 가능한 형태로 **스캐터 클러스터 판정에 스타듀식 확산 규칙을 반영**한다.

- 현행 `_gd_cluster`(저주파 블록노이즈)는 "어디가 풀 무리냐"를 정하지만 이웃-상관이 약하다. 스타듀 확산(만개 풀이 흙 이웃으로 번짐)을 본떠, **풀 tuft 클러스터가 이웃 방향으로 유기적으로 자라나는** 결정적 성장 패스를 스캐터 전에 1회 적용.
- 순수 시각·결정적(좌표 해시)·셀 단위. 백트래킹 없음(퍼프 상한).
- **파괴적 확산(잡초 번식)은 여기 넣지 않는다** — 그건 [ADR-0055] 재점령 시스템의 동적 레이어. 본 스펙은 *정적 base 변주*만. (§4 공존 규칙)

### C. 변종 확대 — 병렬 아트 트랙 (레버 #1·#4)
스타듀 풍성함의 두 상위 레버는 변종 수(풀 4-tuft 하위구성·잡초/twig/stone 다종). 현재 소수.

- 본 스펙은 **변종 로스터 + 스펙카드([ADR-0025])만 산출** — 어떤 변종을 우선 추가할지 우선순위 목록. 아트 생산 자체는 **owner-Gemini/PixelLab 병렬·페이스**(코드 트랙과 비차단).
- A(테이블)가 변종 슬롯을 데이터로 이미 여니, 변종이 들어올 때마다 **재빌드만으로 자동 반영**(엔진 변경 0).
- 우선순위(taxonomy 기준): ① 풀 tuft 변종 2~3종 추가(레버 #4·현재 GD_GRASS1~3) → ② 잡초 변종 → ③ twig/stone 변종. 계절 recolor는 §3 아웃오브스코프.

## 3. 스코프 — 지금 짓는 것 / forward 훅

**In(지금, 기존 에셋으로):** A(구역-키드 테이블 구조 + 안식 튜닝) · B(풀 확산 CA) · C(변종 로스터·스펙카드). 셋 다 안식 농원에서 즉시 육안 개선.

**Out(forward 훅, 지금 안 지음):**
- **계절-키드 테이블 + 계절 타일시트 스왑**(레버 #1) → **Slice 7**(계절·날씨). 테이블 구조를 `{region: {season: {...}}}`로 나중에 한 겹 더 키잉할 수 있게 A를 설계하되, 지금은 region 레벨까지만. 계절 시스템 미존재라 지금 키잉하면 헛빌드([ADR-0028]).
- **던전 절차 생성** → **Slice 5**(채광·나락). *forward 노트 수정:* 스타듀 광산조차 WFC가 아니라 **프리팹 방 조립**이다. 따라서 S5 던전은 "정식 WFC 확정"이 아니라 **"절차 생성 — WFC는 한 옵션, 프리팹-스티칭이 스타듀 실제"**로 완화 기록. S5 착수 grill에서 [ADR-0031] 서랍 수치와 함께 결정.

## 4. 보존·공존 불변식 (어기면 안 됨)

1. **저작 맵 불가침([ADR-0005/0015]):** 스캐터는 GROUND/PATH 빈 셀만. 건물·프롭·밭·절벽·물·워프 seam·구역 가장자리는 **읽되 절대 안 덮음**. `_g16_blend_scatter`의 점유 회피(`occupied`) 유지.
2. **결정성:** 시드=`_gd_h01(x,y,salt)` 기반, 매 로드·재방문 동일. save.dat·회귀 테스트 오탐 방지(메모리 다수 기록). **난수(`Math.random`류) 금지** — 좌표 해시만.
3. **퍼포먼스:** 로드 시 `_ground_detail_tex`로 **1회 bake**(프레임마다 아님). 밴드/구역 스코프. 백트래킹 없는 성장(전체셀 무거운 처리=홈빌드 17s·bana_test 행 회피 — 메모리).
4. **재점령([ADR-0055]) 공존:** 본 스펙=정적 base 변주(스캐터 데칼), 재점령 잡초=동적 상위 레이어(`_draw_encroach_weeds`). 서로 안 싸우게 — 스캐터는 시각 데칼, 재점령은 grid/게임플레이. 겹침 시 재점령이 위.
5. **Wang 경계([ADR-0057]) 무간섭:** Wang=지형-타입 엣지 전환, 스캐터=디테일 레이어. 관심사 분리 유지.
6. **현행 룩 방향 유지:** `_G16_GRASS_PATCHES=false`(스캐터-only 스타듀). 채움 잔디패치 부활 아님 — **스캐터 분포를 풍성화**하는 것.

## 5. 아키텍처 & 데이터 흐름

```
region 진입 → _build_ground16()
   ├─ (기존) tan 베이스 + Wang 경계 + 절벽/LIP/apron 오버레이   [불변]
   ├─ [B] 풀 확산 성장 패스(결정적·이웃-상관) → 클러스터 마스크 강화
   └─ [A] _g16_blend_scatter(out):
          table = _REGION_GD_TABLES.get(_region, _GD_TABLES)   ← 구역 키드(신규)
          sparse = _REGION_GD_SPARSE.get(_region, _GD_SPARSE)
          for 각 GROUND/PATH 빈 셀(점유 회피):
             클러스터 게이트 → full/sparse 테이블 선택 → 가중 롤 → 변종 데칼 blend
   → _ground_detail_tex (단일 텍스처·단일 draw call)   [불변]
```

- 변경 파일: `main.gd`(테이블 구조·조회 지점·성장 패스). 신규 데이터는 상수/딕셔너리.
- **격리·테스트 가능:** 테이블 조회·성장 패스는 순수 결정적 함수 → 고정 입력+구역+시드 → 출력 안정성·점유 불침범을 헤드리스로 단언.
- 에러 핸들링: 구역 키 없음→전역 폴백. 변종 텍스처 null→맨 타일(현행 동작). 성장 패스 모순→현행 클러스터로 폴백.
