# 밤 경비 루프와 전투 심화 경로 — 막기는 곱셈기 위 얇은 방어, 전투는 Phase 3 구현 교체

> ⛔ **폐기됨(2026-06-24): [ADR-0021](./0021-relationship-reward-light-affinity-narrative-primary.md) grill에서 *밤 경비 루프 자체*를 삭제했다.** 전투 공간이 던전(갱도·[나락])으로 따로 확보됐고(ADR-0015), 밤 시간대는 *바 서빙*(오후 3시~밤 12시 서비스 블록, 네 명 다 출근)으로 채워지므로 별도 방어 루프가 중복이라 손절. **바나는 밤 경비가 아니라 던전 전투/보안 담당**(침입자 → 나락의 위협을 *근원에서* 막는 수호자)이 됐고, 보상은 *보호 곱셈기*가 아니라 *전투 경험치 당근*(ADR-0021)이다. **[잡귀]는 바에 침입하지 않고 [나락]에만 남는다**(밤 약탈 리스크 삭제). [나라카 바]는 *멜/공동 밤 서빙*으로 재정의된다. 아래 본문(옵트인 밤 영업·이중 손실·이중 보호 곱셈기·막기↔응대 경쟁)은 *역사적 기록*. **단 #8 "막기 해소를 반환 계약 뒤에 숨겨 전투로 구현 교체"의 디커플링 패턴은 던전 전투 구현에 그대로 재사용**할 것.

[ADR-0008](./0008-growth-model-relationship-multiplier.md)이 바나 곱셈기의 *종류*를 "보호(손실 방지)"로 확정했고, [ADR-0006](./0006-fun-gate-multi-loop.md)이 전투(순수 활동)를 Phase 3로 손절했다. 그런데 [game-loops](../design/game-loops.md) §2.4 밤 경비 1층 MVP 안에는 "탈주 잡귀 막기"가 들어 있고, §2.6 전투는 "갱도/밤 경비에서 마주치는 잡귀와 전투"라고 적혀 있다 — **밤 경비 MVP(Sprint 6, Phase 1.5)와 전투(Phase 3)가 같은 "잡귀"를 두고 겹쳐**, "밤 경비의 막기를 전투로 만들면 Sprint 6이 몰래 Phase 3 전투를 끌어와 ADR-0006 범위 통제를 깬다"는 긴장이 문서에 그대로 박혀 있었다. Sprint 6 착수 grill에서 이 가지를 끝까지 따라 내려가, 밤 루프의 메카닉 골격과 전투 심화 경로를 확정했다.

## 결정

**밤 경비를 "플레이어가 수행하는 옵트인 손실-방어 루프 + 바나 보호 곱셈기"로 짜고, 전투는 그 막기의 *구현을 교체*하는 Phase 3 심화로 둔다.**

1. **수행 주체는 플레이어, 바나는 곱셈기다.** 밤에 잡귀를 막는 행위는 *플레이어*가 한다(미호=농사, 멜=카페와 정확히 대칭). 바나는 밤 루프의 담당 NPC로서 보호 곱셈기 + 속죄 서사(옥자 '목격' 조각)를 얹을 뿐, **검을 든 액션 동료(파티원)가 아니다.** 바나를 함께 싸우는 NPC로 만들면 ADR-0008 곱셈기 모델이 깨진다.

2. **1층 MVP의 막기 = 얇은 방어, 전투 엔진 0.** 막기는 "잡귀 접근 → E → 쫓아냄"(즉시 판정, 격퇴 성공/실패 이진). HP·무기·적 패턴 없음. 진짜 전투(§2.6)는 Phase 3다 — 밤 경비 MVP는 "밤이 농사와 다른 활동으로 채워진다"만 그레이박스로 증명하면 된다(ADR-0006 범위 통제 유지). 적을 *썰어죽이는* 것은 오히려 바나 테마(보호=수비)에서 벗어난다.

3. **밤 경비는 혼력을 쓰지 않고 시간(밤 창)으로만 제한한다.** 카페 서빙과 같은 결(ADR-0008: 혼력 = 노동 전용, 응대·장사는 시간 희소성). 혼력을 밤까지 끌면 아침 농사와 혼력 풀을 두고 싸워 ADR-0008이 피하려던 "혼력 부풀리기"가 도로 생긴다. (혼력/체력은 Phase 3 전투가 들어올 때 재논의 — Consequences 참조.)

4. **막기와 응대는 *경쟁*한다(기회비용).** 응대 중 잡귀가 뜨면 *카운터를 비우고 막으러 갈까(손님 이탈) vs 손님부터 받다 뚫릴까(밤 손실)*. 순차로 두면 기회비용 0 = 긴장 0이라 바나 "보호" 곱셈기가 보호할 대상이 사라진다. 이로써 §2.8 직조의 밤쪽 절반이 낮("밭 vs 카운터")과 같은 기회비용 문법으로 완성된다.

5. **손실은 시간 지평이 다른 *이중*이다.** 잡귀는 바나 죄목(주거침입·흡혈)의 거울로서 *약탈*한다 — 무엇을 약탈하느냐가 둘:
   - **막기 실패 → 재고 약탈:** 낮에 농사로 쌓은 카페 재료를 훔침 → *내일* 카페가 굶음(미래 자산).
   - **응대 실패(카운터 비움) → 현장 매출·단골 이탈:** *지금* 눈앞 손님을 놓침(현재 자산).
   택일이 "현재 매출 vs 미래 재고"라 계산기가 아닌 직관적 딜레마가 되고, 잡귀가 *낮의 농사 산물*을 노리니 밤이 낮 농사·카페 사슬(밭→재고→서빙)에 자원으로 직접 묶인다(직조).

6. **밤 약탈은 옵트인 리스크다(매일 세금 아님).** 잡귀 약탈은 플레이어가 *밤 영업(바)을 열고 깨어 있을 때만* 발생한다. 자정 전에 자면 손실 0·밤 매출 0. 밤 경비를 "안 하면 손해"인 매일 세금으로 두면 사실상 강제 게이트가 되어 은둔 농사파를 매일 처벌한다(ADR-0008 "평평 ≠ 막힘" 위반). 옵트인은 밤을 *선택적 고위험-고보상 루프*로 만들어, 기회비용을 하루 단위로 한 층 더 쌓는다("오늘 밤 더 벌래(+위험) vs 푹 자고 내일 밭에 올인").

7. **보호 곱셈기 = 두 손실에 대응하는 이중 축**(ADR-0008 "같은 +% 반복 = 그라인드" 회피, 미호 가속/범위 이중 축과 대칭):
   - **㉠ 재고 방어:** ♡↑ → 잡귀가 훔치는 재고량 ↓ / 창고 쪽 잡귀를 N마리까지 자동 차단(미호 *범위*의 대칭 — "내가 못 가는 곳을 바나가 받쳐줌").
   - **㉡ 응대 보호:** ♡↑ → 카운터 비운 사이 손님 인내심이 더 오래 버팀(이탈↓, 미호 *가속*처럼 강도 축).
   ♡0이면 둘 다 0 = 바나 잠듦(밤은 거칠지만 base로 굴러감). ♡↑일수록 양쪽 바닥을 받쳐 밤이 점점 수월+짭짤 = ADR-0008 "관계 = 의도된 최적 경로"의 밤판. 곱셈기 동사가 미호(대신 농사=자동화)·멜(더 비싸게=증폭)과 명백히 다른 *손실 방지/안전망*이라 그라인드를 피한다.

8. **전투 심화 경로 = "막기 해소"를 반환 계약 뒤에 숨긴다.** 밤 경비 코드는 계약 하나만 든다:
   ```
   잡귀 위협 → [막기 해소] → { 격퇴 성공/실패, 약탈량 }
   ```
   다운스트림(이중 손실 적용·바나 ㉠㉡ 곱셈기·막기↔응대 경쟁)은 이 반환값만 소비하고 *어떻게* 격퇴했는지 모른다. (B) MVP 구현 = "접근→E", Phase 3 전투 구현 = 실제 전투(§2.6). 반환 계약이 같으면 전투로 갈아껴도 다운스트림 재설계 0 — 이미 `field.gd advance_day(accel, reach)`가 Foxfire를 *모른 채* 값만 받는 그 패턴이다. 바나 곱셈기를 *판정 결과 위 레이어*로 얹었으므로(㉠ = "그 N마리는 전투 자체가 안 일어남", ㉡ = "져도 동일 적용") 전투로 가도 그대로 산다.
   이음매는 **미호 수준 데이터 경계까지만** 판다 — 해소가 `{격퇴, 약탈량}`을 반환하고 다운스트림이 그것만 소비하는 선. 풀 전투 인터페이스(적 클래스·무기 추상)를 MVP에 미리 파는 것은 야크 셰이빙(ADR-0001 정신 위반)이므로 Phase 3로 미룬다.

## 폐기한 대안

- **바나 = 함께 싸우는 액션 동료 NPC** — ADR-0008 곱셈기 모델을 깬다(캐릭터는 base 위 곱셈기지, 활동을 대신 수행하는 파티원이 아니다).
- **밤 경비 MVP 막기 = 진짜 전투** — Sprint 6이 Phase 3 전투를 앞당겨 ADR-0006 범위 통제를 무력화하고, 그레이박스 슬라이스에 전투 엔진 제작을 끼워 넣는다.
- **단일 손실(골드만)·단일 보호 축** — 막기·응대가 같은 통화를 다퉈 (나) 경쟁이 밋밋해지고, "손실 -%" 한 덩어리는 ADR-0008이 경고한 그라인드.
- **매일 세금형 약탈** — 밤 경비를 강제 게이트로 만들어 은둔 농사파를 처벌(ADR-0008 위반).
- **MVP부터 풀 전투 인터페이스 추상화** — 안 쓸 추상을 미리 파는 야크 셰이빙(ADR-0001).

## Consequences

- **game-loops §2.4가 *각도+종류*에서 *메카닉 골격*까지 채워진다** — 주체(플레이어)·자원(시간 제한)·재미(경쟁)·판돈(이중 손실)·노출(옵트인)·보상(이중 보호 축). ROADMAP Sprint 6의 "바나 관계 보상축 *내용은 grill로*" 서랍이 이 ADR로 닫힌다(구체 수치는 구현 grill).
- **`foxfire.gd` 패턴이 또 한 번 일반 법칙으로 적용된다** — 바나 보호 곱셈기도 static 매핑 + 세이브 무상태 + base 위에 얹기(♡0=잠듦). 막기 해소는 `advance_day(accel, reach)`처럼 다운스트림이 메카닉을 모른 채 값/결과만 받는 디커플링.
- **전투 심화 단계(Phase 3 확장) 진입 grill의 숙제 두 개(플래그) — ✅ [ADR-0011](./0011-combat-health-resource.md)에서 해소:**
  1. ✅ **혼력 충돌** — §2.6 전투는 "혼력/체력" 사용이나 ADR-0008은 "혼력 = 노동 전용". → ADR-0011: 전투 자원 = **별도 체력(HP)**(혼력과 종류가 다른 피해 자원), 혼력은 노동 전용 그대로. 밤=체력만(혼력 안 씀), 갱도=채굴 혼력+전투 HP 동시.
  2. ✅ **밤 시간 창 유지** — (B)의 시간 제한(밤 창)은 전투가 들어와도 유지. 전투는 창 *안의 행위*만 바꾼다(ADR-0011 #3 재확인).
- **전투는 밤 경비 전용이 아니라 갱도(§2.5)와 공용** — Phase 3에서 전투를 한 번 만들어 두 무대(밤 경비·갱도)가 같은 "막기/조우 해소" 계약으로 호출한다.
- **이 분리(막기=얇은 방어 / 전투=Phase 3 구현 교체)나 옵트인·이중 손실·이중 보호 축을 되돌리려면** 이 ADR을 새 ADR로 대체해야 한다.
