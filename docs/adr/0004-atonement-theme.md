# "속죄" — 죄의 힘을 정반대로 써서 갚는다 — 를 게임 관통 테마로 삼는다

각 죄인 직원은 **생전의 죄를 저지른 그 힘을, 나라카에서 정반대 목적으로 써서 속죄한다.** 미호: 여우불 방화 → 여우불 작물 양육. 멜(도박·사채): 돈으로 망침 → 카페 경제·회계. 바나(주거침입·흡혈): 밤의 침입자 → 밤의 경비·손님 응대.

이 테마를 뼈대로 고정하면 모든 캐릭터·시스템 기획이 "이 캐릭터의 죄는 무엇이고, 그 힘을 어떻게 뒤집어 게임 시스템에 붙이는가?"라는 **단일 질문으로 수렴**한다. 호감도 스토리 = 속죄 서사가 되고, 각 캐릭터 능력이 게임 메카닉(농사·경제·밤 이벤트)에 직결된다. 1인 개발의 의사결정 비용을 크게 줄이는 설계 제약이라 의도적으로 채택한다.

## Consequences

- 멜·바나의 구체 능력은 각자의 슬라이스에서 정하되, 반드시 이 테마(죄→역전)를 따라야 한다.
- 테마를 어기는 캐릭터/시스템은 추가하지 않는다. 이 제약이 곧 게임의 정체성이다.
