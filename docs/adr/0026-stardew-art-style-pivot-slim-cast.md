# 스타듀밸리 룩 전면 채택 — 슬림 캐스트 + 월드 전체 룩 재생성 (ADR-0003/0012/0018 개정)

기획 단계로 되돌려 방향을 재구체화하던 중, **직접 플레이 누적 피드백으로
"지금 캐릭터가 선명하고 큼직해서 아기자기한 감성을 죽인다"**가 owner 결론으로 굳었다.
진단(ADR-0018)상 캐릭터:타일 비율은 이미 1×2(스타듀 동일)라 *크기*가 원인이 아니라,
**① `detail=high` 또렷함 ② 치비 2.5등신 큰머리 ③ 또렷한 팔레트** 세 가지가 "큼직·선명"의
실제 범인이었다. owner는 캐릭터뿐 아니라 **타일·가구·UI를 포함한 월드 전체 룩**을
스타듀밸리식으로 통일하기로 결정했다.

이는 ADR-0018이 비용 과다로 **철회했던 "결정 3(스타듀풍 캐릭터 재생성)"을 다시 여는**
결정이며, 그걸 캐릭터를 넘어 월드 전체로 확장한다.

## 결정

**아트 룩을 스타듀밸리식으로 전면 교체한다.** 그리드·`TILE=32` 월드 좌표·게임 로직·
회귀 테스트는 불변(타일 *개수*·좌표·1×2 비율 그대로). 바뀌는 건 **아트 스타일**뿐이다.

1. **캐릭터 5종 재생성 (슬림·단정 ~3등신).** 치비 2.5등신을 폐기하고 머리:몸 ≈ 1:3의
   **날씬·단정한 기준선 체형**으로 간다. 이유는 미감만이 아니라 **표현력** — 슬림 기준선이
   깔려야 통통·건장 등 *체형 차이가 대비로 읽혀* 캐스트를 다채롭게 표현할 수 있다.
   생성 = PixelLab 저디테일(`detail=low`/`medium`) + 슬림 `proportions` 공통 고정값
   (정확한 수치는 재생성 시 [p2.0-spike-prompts.md](../design/p2.0-spike-prompts.md)에 잠금).
2. **월드 전체 룩 스타듀화 (타일·가구·UI).** 32px native(ADR-0013)는 유지하되,
   **스타듀식 소프트 제한 팔레트 + 저디테일·둥근 톤**으로 지형·가구·UI를 통일한다.
   또렷·고대비 룩을 버리고 "아기자기·코지"한 색·선으로.
3. **얼굴은 대화 초상화 전담 유지(ADR-0003/0018).** 인게임 도트를 작고 소프트하게
   둘수록 이 분업이 오히려 강화된다 — 매력·표정은 초상화가, 인게임 도트는 분위기가 진다.

## 근거

- **"큼직·선명"의 원인은 크기가 아니라 디테일·비례·팔레트**(ADR-0018 진단). 크기(16×32로
   축소)를 내리면 ADR-0012가 올린 이유("얼굴 안 보임")가 재발하므로, 손대는 건 스타일이다.
- **슬림 기준선 = 체형 표현력.** 치비는 모든 캐릭터 머리를 키워 체형 차이를 뭉갠다.
   슬림·단정을 baseline으로 깔아야 다양한 캐스트가 구별된다(owner 논거).
- **지금이 가장 싼 재생성 시점.** 캐릭터는 아직 5종, NPC가 15명으로 늘기 전. ADR-0018이
   "비용 과다"로 미룬 재작업을 기획 재구체화 시점에 한 번에 치른다.
- **그리드 불변** → 좌석·밤 스폿·NPC 출근·밭·워프 좌표와 회귀 테스트가 전부 불변(시각만 드롭인).

## 관계 (ADR-0001/0003/0012/0013/0018)

- **ADR-0003 개정:** "치비 ~2.5등신"을 **슬림·단정 ~3등신**으로 바꾼다. 탑다운 3/4뷰·
   정수배 스케일·**표정=별도 일러스트 초상화**는 그대로(오히려 강화).
- **ADR-0012 개정:** `detail=high` + 치비 `proportions` 고정값을 **저디테일 + 슬림 proportions**로
   바꾼다. 내부해상도·`TILE=32` 그리드는 ADR-0018 기준 유효.
- **ADR-0013 유효(스타일만 조정):** 환경 아트 32px native·렌더 1:1은 유지하되, 팔레트·톤을
   스타듀식 소프트로 통일한다(해상도가 아니라 색·디테일을 바꿈).
- **ADR-0018 결정 3 재개봉:** "비용 과다"로 철회했던 캐릭터 재생성을 다시 열고, 월드 전체
   룩까지 확장한다. 카메라(코지 와이드 960×540)·맵 확대 결정은 그대로 유효.
- **ADR-0001 유효:** 도트는 PixelLab/Retro Diffusion 생성 + Aseprite 보정. 자체 도트화 툴 금지.

## 결과

- **에셋 파이프라인 리셋(비용 인지):** 기존 P2 캐릭터·초상화·환경 아트와 **진행 중인
   Phase 2.8 에셋(테두리 프롭·집 facade 등)이 무효화**된다. 새 스타듀 룩 기준으로 재생성한다.
   → 비주얼-우선 마일스톤(옥자 데모, ADR-0023)과 P2.8 작업 순서를 이 룩 기준으로 재정렬할 것.
- **재생성 스펙은 재시작 시 잠근다:** 슬림 `proportions` 수치·`detail` 등급·스타듀 팔레트 토큰을
   p2.0-spike-prompts.md에 박고, ADR-0025의 스펙 카드 게이트(에셋 생성 전 승인)를 그대로 따른다.
- 검증: 회귀 8종 PASS(그리드 불변이므로 로직 무영향), map_dump 육안으로 룩 일관성 확인.
