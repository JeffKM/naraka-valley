# 인게임 픽셀 규격: 16×16 타일 / 16×32 캐릭터 / 320×180 내부해상도 / 탑다운 3/4뷰

에셋 제작을 시작하기 전에 픽셀 규격을 고정한다. 한번 정하면 모든 캐릭터·타일·아이템이 여기 묶이고, 변경 시 전체 에셋 재제작이 필요하기 때문이다.

- **카메라 시점**: 탑다운 3/4 뷰 (스타듀밸리·젤다식). AI 도구 학습이 가장 잘 된 뷰.
- **타일**: 16×16 px (스타듀밸리 동일). 32×32 대비 에셋당 작업량 1/4.
- **캐릭터**: 약 2.5등신, 16×32 px 안팎. 원작 인스타툰체의 치비 비율과 호환.
- **내부 해상도**: 320×180(16:9)을 정수배 스케일(×4=1280×720, ×6=1920×1080). Godot 뷰포트로 설정.

## 캐릭터 매력 표현 방식

인게임 도트(16×32)는 얼굴이 몇 픽셀 안 되므로 표정을 도트에 욱여넣지 않는다. 대신 **대화 시 미드저니 기반 큰 초상화(일러스트)를 별도로 띄운다**(스타듀밸리 방식). 인게임 도트 제작 부담을 최소화하면서 캐릭터 매력은 일러스트로 살린다. 이는 ADR-0001의 "미드저니=일러스트, 픽셀AI=인게임" 역할 분담과 일치한다.
