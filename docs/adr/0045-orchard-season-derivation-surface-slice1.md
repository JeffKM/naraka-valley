# 과수 절기 유도 표면 — Slice 1 선도입 (Slice 7 절기 시스템 경계)

> **개정 대상:** [homestead-farming-greybox-spec.md §5.6](../design/homestead-farming-greybox-spec.md)의 "절기 연계는 `multi_seasonal: bool` 단일 수렴, 상세 절기 매핑·사멸 판정은 **Slice 7 이관**" 경계를 부분 관통한다. 절기의 *사멸·날씨·축제·예보*는 Slice 7 그대로 두되, **읽기 전용 `day→절기` 파생 표면만** Slice 1(S1-5b 과수)에서 선도입한다.
>
> **성격:** 되돌리기 비쌈(`clock.gd` 공개 표면·`orchard.gd` 의존·세이브 무관하나 API 계약) + 놀라움(왜 Slice 7 전에 절기가 `clock.gd`에 있고, orchard는 절기를 읽는데 작물은 안 죽지?) + 실 트레이드오프(스텁 A/B/C) → ADR.

## 맥락

S1-5b 혼의 나무 과수의 **정의 기능 두 개**가 절기 개념에 의존한다:

- **제철 결실** — 나무는 자기 절기에만 결실한다(비제철엔 정지).
- **절기 영속** — 나무는 절기 전환에도 죽지 않는다(작물 사멸의 예외, 다절기 프레스티지의 나무판).

그런데 절기 런타임은 아직 없다. `clock.gd`엔 `day`(1부터)만 있고, greybox-spec §5.6이 절기 시스템(사멸 판정·날씨·축제·예보 UI)을 통째로 **Slice 7(계절·날씨·축제)** 로 이관했다. 문제는 위 두 기능이 절기 없이는 **검증 자체가 불가**하다는 것 — S1-5b 완료기준("제철 결실·나이별 품질 검증 통과")이 성립하지 않는다.

## 결정

`clock.gd`에 **읽기 전용 `day→절기` 파생 함수만** 도입한다. Slice 7 본체는 건드리지 않는다.

- **상태 변수 없음.** `clock.gd`는 `season` 인스턴스 상태를 갖지 않고, 기존 `day`에서 계산되는 순수 파생 함수만 노출한다:
  ```gdscript
  static func season_index_for_day(d: int) -> int:
      return ((d - 1) / 28) % 4    # 0=피안 · 1=유화 · 2=망연 · 3=성야
  ```
  게임 시작 = 피안절 1일이므로 `day 1 → index 0 → 피안절`. 절기 순환 = 피안(봄)→유화(여름)→망연(가을)→성야(겨울) — [CONTEXT §저승 절기] 정합.
- **소비자는 `orchard.gd` 하나.** 나무가 "지금 내 결실 절기인가?"만 판단하는 데 쓴다.
- **절기 판정은 매 틱 무상태 재계산.** `clock.day`에서 그때그때 유도(세이브 시점 캐시 아님) → 로드-틱 유령 결실 버그 원천 차단.

**불가침 경계(Slice 7 소유 유지):**
- **작물 사멸 판정을 심지 않는다.** orchard는 절기를 *읽기만* 하고, 절기 전환 사멸 트리거를 만들지 않는다. 작물 사멸(`multi_seasonal==false` 일괄 죽음)은 Slice 7 그대로다. 영속은 orchard가 사멸 판정에 *참여하지 않음*으로 자명하게 성립한다.
- **날씨·축제·예보 UI·절기별 콘텐츠 배치**(어느 과일이 어느 절기에)는 Slice 7.

## 근거 (기각한 대안)

- **(B) 절기 게이팅을 Slice 7로 통째 미룸** — 나무는 절기 무관 매일 결실, 영속은 "아무도 안 죽이니 자명". 하지만 S1-5b의 정의 기능 두 개가 스텁이 되어 **슬라이스가 속 빈다**(완료기준 검증 불가). 기각.
- **(C) Slice 7 절기 시스템을 먼저** — 순서 역전. Slice 1은 완성 우선순위 1위(데모 1)라 Slice 7을 앞세우면 시퀀싱이 깨진다. 기각.
- **채택 (A)** — 얇은 파생 표면만. S1-4가 `multi_seasonal: bool`을 단일 수렴점으로 스텁하고 S1-5가 소비한 **데이터/메카닉 분리와 동형**. Slice 7은 이 파생 위에 사멸·날씨·축제를 *얹는다*(중복 아님, 토대).

## 파급

- **Slice 7 진입 시:** `season_index_for_day`는 이미 존재하는 토대다 — Slice 7은 여기에 사멸 판정·날씨 풀·축제 슬롯·예보 채널을 얹는다. 이 함수를 재작성하지 않는다.
- **미래 독자 대비:** orchard가 절기를 읽는데 작물이 안 죽는 것은 *의도*다(영속 vs 사멸의 분리). quality_tier_for_age가 S1-6까지 미사용 파생인 것도 의도(품질 소스만 정의, 인벤토리 배선은 S1-6).
