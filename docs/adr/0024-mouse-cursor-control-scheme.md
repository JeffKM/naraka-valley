# 마우스 커서 조작 스킴 (스타듀 동일) — ADR-0020 보강

> **보강:** [ADR-0020](./0020-item-tool-architecture.md)이 "단일 컨텍스트 E → 선택 도구 모델"을 못 박았으나, 그 *바인딩*(어떤 입력으로 도구를 쓰나)은 열어뒀다. Phase 2.7 코어 시스템 착수 grill(2026-06-24)에서 키보드-facing vs 마우스-커서를 따져, **마우스 커서 조작(스타듀 동일)** 으로 확정한다.

## 맥락

지금까지 게임은 **키보드 온리 + "바라보는 칸"** 모델이었다(이동 WASD, 단일 `interact`=E가 앞칸/앞NPC에 작용). 도구 피벗(ADR-0020)을 얹을 때 두 갈래가 있었다:

- **(A) 키보드 2-키 (facing 칸):** `도구 사용` 키 + `액션/대화` 키, 둘 다 바라보는 한 칸에 작용. 현재 구조에서 가장 작은 피벗 — 마우스 커서→타일 변환을 안 짜도 됨.
- **(B) 마우스 커서 (스타듀 네이티브):** LMB=든 도구를 *커서 밑 타일*에 사용, RMB=액션·대화. 이동은 WASD 유지.

## 결정

**(B) 마우스 커서 조작으로 간다 — 조작 체계를 스타듀밸리와 동일하게 한다.**

1. **이동 = WASD/방향키, 조준 = 마우스 커서** (별개 축). 도구를 쓸 때 캐릭터가 커서 쪽으로 돌아본다.
2. **LMB = 손에 든 도구 사용** — 핫바에서 선택한 것(괭이·물뿌리개·씨앗봉지)의 동사만 작용. 안 맞으면 무동작(자동 분기 없음 — ADR-0020 "선택 도구가 장식이 되지 않게" 정합).
3. **RMB = 액션/대화** — 맨손 동작: 자란 작물 맨손 수확·NPC 대화·좌석 서빙·밤 막기·출하함/매대 열기·취침. (스타듀에서 수확·대화가 액션 버튼인 것과 동일.)
4. **작용 칸 = 커서 밑 타일, 단 플레이어 인접 1칸 반경(주변 8칸+발밑)으로 제한.** 그 밖이면 무효/클램프. 커서 밑 유효 타일에 **하이라이트**(스타듀의 그 칸 표시).
5. **사거리 ≠ AoE.** base 사거리는 인접 1칸. 도구 충전(LMB 홀드)으로 펼쳐지는 AoE 범위(1×3·3×3, [ADR-0019])는 *별개 축*이며 Phase 3 스킬에서 추가 — 2.7은 1×1만, 자리만 예약.

## 폐기한 대안

- **(A) 키보드 2-키 (facing):** 구현 표면이 가장 작지만 "진짜 스타듀형 게임"의 조작 손맛(마우스 조준·임의 칸 작용)을 못 준다. 마우스 조준은 후속 폴리시가 아니라 *조작 정체성*이라 처음부터 넣는다.
- **(C) 단일 키 스마트 라우팅:** ADR-0020이 이미 기각한 (B) 하이브리드(선택 도구가 장식). 제외.

## Consequences

- **커서→타일 좌표 변환 + 사거리 클램프 + 하이라이트 렌더가 신규 표면**으로 들어온다(Phase 2.7 C1).
- **검증된 농사 입력(`field.next_action` 강제순서 → 단일 E)이 전면 재배선된다** — `field.hoe/water/plant/harvest` 직접 호출, T2.1~T2.3·서빙·밤·온보딩 회귀 재작성(ADR-0020 Consequences의 "최대 리스크"가 이 스킴으로 구체화).
- **`Q` 작물 순환 소멸** — 씨앗은 핫바에서 선택(ADR-0020 ① 데이터 주도 아이템 위에서).
- **다시 키보드-facing으로 되돌리거나 다른 조작 체계로 바꾸려면** 이 ADR을 새 ADR로 대체해야 한다.
