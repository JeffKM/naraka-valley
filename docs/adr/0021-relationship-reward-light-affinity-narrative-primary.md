# 관계 보상 = 가벼운 호감도 당근 + 서사 주력 — ADR-0008 대체

> **대체:** [ADR-0008](./0008-growth-model-relationship-multiplier.md)(관계 = 명백히 우월한 곱셈기 엔진)을 *대체*한다.
> **개정:** [ADR-0019](./0019-physical-skill-relationship-multiplier-two-axis.md) #2/#4(미호=자동화·바나=화력 곱셈기) — 곱셈기 *종류*가 아니라 *스킬 경험치 당근*으로 바뀐다.
> **연쇄:** [ADR-0010](./0010-night-guard-combat-layering.md)(밤 경비) 폐기, [ADR-0011](./0011-combat-health-resource.md)·[ADR-0007](./0007-mel-cafe-operation-loop.md) 부분 개정 — 아래 Consequences·각 ADR 헤더 참조.

## 맥락 — ADR-0008의 전제가 낡았다

[ADR-0008](./0008-growth-model-relationship-multiplier.md)은 관계를 **"항상 명백히 우월한 의도된 최적 경로"(§17)인 기계적 곱셈기 엔진**으로 못 박고, 곱셈기 *종류*를 캐릭터마다 다르게(미호=자동화·멜=마진·바나=보호) 갈라 그라인드를 피했다(§3·§21). 이 결정의 *근본 전제*는 **"플레이어가 관계를 추구할 *이유*를 메카닉이 떠받쳐야 한다"** 였다 — 안 그러면 순수 스타듀(관계=덤)로 전락하니까(§44 폐기 대안 b).

그러나 ADR-0008 이후 서사 구조가 크게 바뀌었다. [ADR-0016](./0016-converging-night-ensemble.md)·[ADR-0017](./0017-main-affinity-activity-gate.md)이 **관계를 메인 스토리의 *필수 척추*로 만들었다** — ♡3 [그날 밤] 고백이 클라이맥스 재구성의 *필수 증거*(코러스 모델), ♡4 서사 조각·♡5 연애·[해방]이 *스토리 완주 자체에 필요*하다. 즉 **"관계를 추구할 이유"를 이제 *서사*가 떠받친다.** 메카닉이 그 짐을 *중복으로* 질 필요가 없어졌다.

Phase 3 수정 grill(2026-06-24)에서 이 가지를 끝까지 따라 내려가, 관계의 *기계적* 보상을 가벼운 당근으로 내려놓고 *서사*를 주력 보상으로 두기로 했다.

## 결정

**관계의 *기계적* 보상 = 가벼운 호감도 차등 당근(초·중반 편의). 관계의 *진짜* 보상 = 서사(♡4 조각·♡5 연애·해방). "평평 ≠ 막힘"은 그대로 보존한다.**

1. **미호 = 농사 경험치 당근 / 바나 = 전투 경험치 당근.** 호감도가 오를수록 해당 [스킬](./0019-physical-skill-relationship-multiplier-two-axis.md)(농사·전투) 경험치를 조금 더 빨리 쌓는다(여우불·사역마 등 픽션 연출은 유지, 메카닉만 경험치). **스킬 트랙이 있는 활동에만 붙는다**(ADR-0019 스킬 = 농사·채광·채집·낚시·전투).

2. **멜 = 별도 경제 보상(카지노 보너스, 후속).** 멜의 도메인(카페 서빙)엔 스킬 경험치 트랙이 없어 XP 당근이 안 붙는다. 멜의 기계적 보상은 *카지노*(후속 추가)의 호감도 차등 보너스로 둔다. 카지노 출시 전까지 멜은 **서사 보상(♡4·♡5)만** 갖는다(이 공백은 의도된 것 — 멜의 주력은 어차피 서사). 카지노 = 죄(도박·사채)와 글자 그대로 닿으므로 [ADR-0004](./0004-atonement-theme.md) 정합을 위해 *"속죄한 도박꾼이 공정한 하우스를 운영(등치지 않고 절제를 가르침)"* 방향의 *반대로 쓰기*를 카지노 설계 grill에서 확정한다(서랍).

3. **기계적 보상은 *의도적으로 가볍고*, 만렙에서 증발한다 — 수용한다.** 경험치 당근은 스킬을 다 찍으면 가치 0이 된다. 이를 결함이 아니라 *설계*로 받아들인다 — 그 시점(엔드게임)의 관계 보상은 **연애·해방·서사**이지 숫자가 아니기 때문이다. 게다가 ADR-0017상 활동이 스킬 XP와 호감도를 *나란히* 올리므로, 당근이 일하는 창은 초·중반으로 자연히 한정된다("가벼운 당근"이 곧 정확한 기대치).

4. **비관심 플레이어 보호 (ADR-0008 §17의 *완화*).** 관계를 외면해도 base rate로 멀쩡히 플레이된다("평평 ≠ 막힘" 불변, ADR-0008 #1 보존). 당근은 *벌이 아니라 당근*이라 — 같이 일하는 플레이어에게 "조금이라도 이득"을 줄 뿐, 외면 경로를 *명백한 차선*으로 처벌하지 않는다. 관계의 당김은 메카닉이 아니라 *서사*가 진다.

## ADR-0008에서 *뒤집은* 것 / *보존한* 것

- **뒤집음 §17 "관계 = 항상 명백히 우월한 의도된 최적 경로"** → 관계는 *서사의* 의도된 경로이고, 기계적으로는 가벼운 당근(near-neutral)이다. T4.4 판정서의 "여우불이 밭 2배 노동을 이긴다"는 *옛 성장가속 모델*의 정량 결과이며 이 대체와 함께 효력을 잃는다.
- **뒤집음 §3·§21 "곱셈기는 캐릭터마다 종류가 달라야 한다(같은 +% 반복=그라인드)"** → 미호·바나가 같은 *경험치 당근*을 공유해도 무방하다. 이유: 캐릭터를 가르는 것은 이제 *스프레드시트(곱셈기 종류)*가 아니라 *서사(속죄 아크·연애)*다. 게다가 둘은 *다른 루프*(농사 vs 전투)에 붙어 반복 체감이 옅고, 솔로 파트타임 개발에 세 벌의 고유 곱셈기 시스템(가속 엔진·마진 엔진·전투 AI)을 만드는 부담을 던다.
- **보존:** ADR-0008 #1 "평평 ≠ 막힘"(♡0 base rate), #4 "서사는 게이트"([ADR-0005](./0005-main-story-emergent.md) "서사는 캐릭터에만"), 활동파 자생 곡선 = 활동 깊이 + 스킬(ADR-0019).
- **순수 스타듀 회귀가 *아닌* 이유:** ADR-0008이 막은 (b)는 "관계 = 무시 가능한 덤"이었다. 여기선 관계가 *스토리 완주에 필수*(ADR-0016/0017)라 덤이 아니라 척추다 — 기계적 층만 가벼워질 뿐, 관계의 무게는 서사로 이전·증대됐다.

## 폐기한 대안

- **(a) ADR-0008 유지(고유 곱셈기 3종 엔진)** — 서사가 이미 관계 추구 동기를 떠받치므로 메카닉 중복. 솔로 개발 부담 큼. 바나의 *만렙에도 안 죽는 고유 전투 곱셈기*(반려 화력/생존)가 미해결로 남아 있었고, 이를 풀어도 위 중복은 해소 안 됨.
- **(b) 순수 서사(기계적 보상 0)** — 같이 일하는 플레이어에게 "조금이라도 이득"을 주는 가벼운 당근의 가치를 버린다(당근은 옵트인 압박 없이 협업을 살짝 보상). 구현 비용도 XP 당근이 고유 곱셈기보다 훨씬 작아 굳이 0으로 갈 이유 없음.
- **(c) XP 당근을 *명백히 우월한* 강한 보너스로** — 만렙 증발 + ADR-0008 §17 부활이라 자기모순. 당근은 가벼워야 비관심 플레이어를 안 민다.

## Consequences

- **`foxfire.gd` 메카닉 교체(Phase 3).** 현재 T3.4 구현은 *작물 성장가속*(타이머)이다 — 이를 *농사 경험치 당근*으로 갈아끼운다. 다운스트림(다운스트림이 값만 받는 디커플링)은 ADR-0010/0011이 세운 그 패턴 그대로 유지.
- **CONTEXT 갱신** — 여우불 성장 촉진(타이머→경험치)·호감도(이중보상→가벼운 당근+서사 주력)·밤 경비(폐기)·나라카 바(밤 서빙으로 재정의)·잡귀(나락 한정)·바나(전투/보안 속죄)·멜(출하대 해제+카지노)·출하대(무인화)·카페 운영(단일 주인 없음)·체력(무대 프로필에서 밤 제거).
- **밸런싱 부담↓:** 우상향 엔진이 *스킬* 하나로 단순화(관계는 스킬을 살짝 가속할 뿐). ADR-0019의 "스킬 곡선이 관계 곡선을 가리지 않게"라는 재검증 숙제가 *소멸*(관계가 더는 독립 엔진이 아님).
- **이 모델(가벼운 당근 + 서사 주력)을 다시 *기계적 곱셈기 엔진*으로 격상하거나 순수 서사로 내리려면** 이 ADR을 새 ADR로 대체해야 한다.
