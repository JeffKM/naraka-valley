# 게임 엔진으로 Godot을 채택한다 (Steam/PC 타겟)

개발자는 TypeScript/React/Next.js 전문가지만, 게임 본체는 익숙한 웹 스택(Phaser 등) 대신 **Godot**으로 만들기로 했다. 스타듀밸리형의 핵심인 타일맵 에디터·캐릭터 이동·충돌·2D 애니메이션·세이브·오디오가 Godot에 전부 내장되어 있어, 웹 스택으로 이 게임 인프라를 직접 재발명하는 수개월보다 "전용 엔진 학습 2~3주"가 훨씬 싸기 때문이다. 배포 타겟은 **Steam/PC**.

## Consequences

- 새 언어(GDScript)·새 에디터 학습으로 초반 2~3주는 속도가 느리다.
- 개발자의 TS 역량은 게임 본체와 분리해 **홍보 홈페이지·위시리스트·디스코드 연동·웹 데모** 쪽에서 활용한다.
- 엔진 교체는 사실상 전면 재작성이므로 이 결정은 되돌리기 매우 비싸다.
