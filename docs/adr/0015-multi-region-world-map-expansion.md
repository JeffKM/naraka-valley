# 멀티 구역 세계 + 맵 확장을 활동 루프보다 먼저

검증된 **단일 무대**(밭+집+카페)를 **8개 분리 구역의 세계**로 확장하고, 이 맵 확장을 **낚시·채광·전투 등 활동 루프 *앞*에** 끼운다. P2.3 grill의 "멀티맵·새 지역은 Phase 3 손절, 단일 무대 유지"와 ROADMAP Phase 3의 "낚시 먼저" 순서를 개정한다.

## 결정

- **순서 재정렬.** 활동 루프(낚시·채광·전투)·기존 기능 심화 *전에* 맵 확장을 먼저 한다. 이유: 각 미래 루프가 들어올 *집(구역)*을 먼저 정의하면 루프가 떠도는 무대에 임시로 박혔다 옮기는 재작업이 없다. 단 **설계(지도)는 8구역 다 그리되, 빌드(코드·에셋)는 한 구역씩**(빈 맵 위험·번아웃 통제).
- **세계 구조 = 분리 맵 + 가장자리/길 전환**(스타듀식). 거대 연속 맵(×)·하이브리드(×) 기각 — 기존 문 전환(`_transition_to`)·카메라 rect 패턴의 확장이고, 현재 구역만 로드해 8구역에 깔끔하게 확장된다. 세이브는 현재 구역을 추적.
- **8구역**: 안식 농원(농사·목축) · 나루 마을(허브, 강+다리 분할, 카페·서비스·거주) · 삼도천(강 낚시·혼백관) · 황천해(바다 낚시·생선가게) · 저승 숲(채집·목공방) · 미혹의 숲(깊은 숲·옥자 집) · 업화 갱도(채광+끝 던전·대장간·길드) · 나락(독립 전투). 정체성은 CONTEXT '지리', 배치는 [docs/design/world-map.md](../design/world-map.md).
- **서비스 = 스타듀식 분산.** 카페 출하대(멜)는 유지하되, 생선가게(황천해)·업화 대장간(갱도 입구)·모험가 길드(갱도 입구)·만물상·목공방(숲)·혼백관(나룻터)을 구역에 분산. 점주는 마을 주민 T2.
- **채광·전투 공간 분리 — ADR-0011 개정.** ADR-0011의 "갱도 = 채굴(혼력)+전투(체력) *동시*(이중 시계)"를 바꾼다: **업화 갱도 = 채광 위주 + 갱도 끝에 전투 던전**, 그리고 **나락 = 독립 전투 전용 구역**. 전투 *자원 정책*(별도 체력, 채굴=혼력/전투=체력, HP는 곱셈기 밖)은 ADR-0011 그대로 유지 — 바뀌는 건 두 활동의 *공간 배치*뿐(밤 바·갱도 던전·나락이 같은 전투 엔진 공유).

## Consequences

- **8구역 = 스타듀 코어보다 큼.** 파트타임 솔로엔 큰 표면 → "한 구역씩 끝까지" + 그레이박스 우선(ADR-0001) 원칙으로 통제. 검증된 홈베이스 무대는 보존(카페만 마을로 이주).
- **카페 이주의 부수효과:** 집·농장(안식 농원)과 카페(마을)가 갈리며 "출근 동선"이 생겨 ADR-0009(카페 명소)를 강화. 기존 단일 맵 코드(`main.gd`)는 구역 분리·전환 관리로 리팩터 필요.
- **새 루프 2종이 따라 들어옴:** 바다 낚시(삼도천 강 낚시와 별개 무대)·목축(축사) — 둘 다 맵 확장에선 *건물·무대 자리만* 확보, 메카닉은 활동 루프 Phase에서(서랍).
- ROADMAP에 맵 확장 Phase를 낚시 앞에 신설(재정렬). game-loops.md의 루프별 스케치는 유효.
