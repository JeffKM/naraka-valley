# ADR-0057 — 환경 지형 타일 = PixelLab 저색 crisp 재생성 (노이즈 카펫 폐기, ADR-0043 부분 supersede)

- **상태:** Accepted (owner 결정 2026-07-16 — "이 방향 확정 → 전체 로드맵")
- **개정 대상:**
  - [ADR-0043](./0043-terrain-regen-lush-pro-raggedness.md)(lush full-grass 베이스 + per-cell 변종)를 **부분 supersede** — "타일을 통째로 채운 풀/흙 텍스처" 접근이 노이즈 카펫의 근원. 소스 타일 생성 방식을 교체한다(격자 구조·`_build_ground16` 로직·per-cell 변종·유기경계 시스템은 유지).
  - [ADR-0047](./0047-gemini-full-asset-regen-supersede-adr0001-scope.md)(Gemini 전면 재생성)에 **지형 타일 예외 추가** — 확산 모델(Gemini/nano-banana)은 지형 seamless 타일에서 노이즈를 재생산하므로 배제. 지형 베이스 타일은 PixelLab(픽셀 네이티브)로 생성. (캐릭터 선명도 예외와 같은 결.)

## 맥락 — owner 피드백과 정량 진단

owner가 안식 농원 라이브를 보고 피드백: **"일반 쯔꾸르(RPG Maker)·바람의나라 느낌이 난다. 스타듀밸리의 아기자기한 아트에 근접하려면?"** 지면 채도 A/B/C 튜닝(`_retone_earth` 사후보정)을 먼저 시도했으나, 색 보정만으로는 그 느낌을 벗어나지 못했다.

원인을 픽셀 레벨에서 정량 진단(스타듀 위키 Standard Farm 크롭과 1:1 비교):

| | 128×128 유니크 색 수 |
|---|---|
| 스타듀 잔디 크롭 | 116색 |
| 스타듀 팜 전체(1280×1040) | 256색 (인덱스 팔레트) |
| **기존 `grass_field.png`(1타일)** | **2573색** ⚠️ |
| **기존 `dirt_field.png`** | **1426색** ⚠️ |

같은 128² 잔디인데 스타듀 116색 vs 우리 2573색 — **22배**. "쯔꾸르/바람의나라" 느낌의 정체다.

**결정적 반증 실험:** `grass_field`를 16색으로 강제 양자화(2573→16)해도 노이즈 카펫 구조는 그대로였다. 즉 **색 수는 증상이지 원인이 아니다.** 후처리 양자화는 헛수고.

진짜 원인 3가지(잔디·흙 확대 육안 확인):
1. **균일 노이즈 카펫** — 타일 전면에 똑같은 밀도의 자글 텍스처. 이게 RPG Maker RTP의 시그니처. 스타듀 흙 타일은 오히려 **거의 flat**하고, 디테일은 전부 그 위 오브젝트(그루터기·돌·소나무)에 있다.
2. **좁은 명암 폭** — 세이지그린 한 대역에서만 놀아 뭉갠다. 스타듀는 밝은 라임 하이라이트 → 어두운 밑동까지 명암이 벌어져 또렷하다.
3. **실루엣/아웃라인 부재** — 방향 없는 얼룩. 스타듀 풀은 개별 blade가 솟은 실루엣 + 어두운 경계가 있어 "아기자기"하게 읽힌다.

근원 결정은 [ADR-0043]의 "풀은 타일이 통째로 차서 옆 칸과 연결돼야 한다"였다. 이 방향 자체가 **타일에 텍스처를 다 채워 넣게** 만들었고, 생성형 AI(확산)가 채우니 고색 노이즈가 됐다.

## 결정

**지형 베이스 타일을 PixelLab `create_topdown_tileset`(픽셀 네이티브·저색·crisp)으로 재생성한다.** 스타듀의 아기자기함은 "타일을 예쁘게 그려서"가 아니라 **타일은 단순하게 두고 그 위에 명확한 실루엣의 작은 오브젝트를 흩뿌려서** 나온다 — 이 구조를 채택한다. 3레버:

1. **베이스 타일 = 저색 crisp** — `tile_size=32`, `detail="low detail"`, `outline="selective outline"`, `shading="basic shading"`, `view="high top-down"`. 결과 45색 수준(스타듀 116색보다도 적다). "텍스처 채우기" 금지.
2. **디테일 = 스캐터 오브젝트** — 긴 풀·작은 돌·꽃·나뭇가지는 타일 텍스처가 아니라 flat 베이스 위 산포 오브젝트 레이어로(forage 산포와 동일 구조).
3. **공통 마스터 팔레트** — 전 타일·프롭을 같은 제한 팔레트로 remap. 스타듀의 통일된 아기자기함은 팔레트 일관성에서 온다.

**확산 모델(Gemini/nano-banana)은 지형 seamless 타일에서 배제한다** — 본질적으로 그라데이션·노이즈를 만들어 진단한 문제를 재생산한다(실측: 동일 요청이 2573색). 픽셀 네이티브 생성기(PixelLab)만 ①타일링 보장 ②정확한 픽셀 그리드 ③제한 팔레트를 구조적으로 만족한다. 확산 모델은 타일링이 불필요한 개별 스캐터 오브젝트·컨셉 목업에만 사용.

## 근거 — 인게임 파일럿 검증 (2026-07-16)

PixelLab 잔디↔흙 Wang 타일셋(16타일·32px)을 생성 → 순수 잔디(idx12)·순수 흙(idx6) 타일을 128² self-tile로 `terrain16/grass_field·dirt_field`에 스왑 → **`_build_ground16` 로직 한 줄도 안 바꾸고** 전체 맵 렌더(`tools/home_pilot_dump.gd`).

결과(before/after, 동일 맵·동일 프롭·지면 텍스처만 교체): 흐릿한 세이지 노이즈 카펫 + 파스텔 흙 → **또렷한 초록 blade 잔디 + earthy 골든탄 흙, crisp 경계(2573→16색)**. 스타듀 팜에 확연히 근접. "타일 재생성이 정답"임을 인게임으로 확증.

## 미해결 (로드맵 슬라이스로)

- **잔디 톤 최종 조율** — muted 요청 대비 잔디가 아직 밝은 편(라임). 프롬프트 1회 추가 조율.
- **길 타일 분리** — 파일럿은 마당 흙=길 흙을 같게 둬 십자 길이 소실. 다진 흙길 별도 타일 필요(스타듀도 별개).
- **Wang 오토타일 경계 통합** — 지금은 순수 타일 self-tile. `pixellab_tileset_converter.gd`로 잔디↔흙 전이 타일을 정식 통합.
- **프롭 톤 정합** — 옛 스타일 프롭(왼쪽 덤불줄 등)을 새 팔레트로 순차 재생성.
- **전 구역 확산** — 안식 검증 후 나루·물가 등 타 구역 지형에 동일 파이프라인.

## 대안 (기각)

- **색 양자화 후처리** — 16색 강제해도 노이즈 카펫 구조 불변(실증). 증상만 건드림.
- **`_retone_earth` 채도 A/B/C 튜닝** — 색 보정이라 "쯔꾸르 느낌" 못 벗어남. 새 타일 생성 시 채도·명암을 처음부터 넣으면 사후보정 자체가 불필요.
- **nano-banana(Gemini) 재생성** — 확산 모델이라 노이즈 재생산(실측 2573색). 지형 타일 부적합. (실행도 불가 — API 키 미설정.)
