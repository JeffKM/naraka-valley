# 건물 문·진입 폭 규약 — 기본 2칸 + 신규 짝수폭 footprint

[ADR-0036](0036-building-front-facade-southward-entry-invariant.md)이 "정면 facade + 남향 진입"을 못 박았으나 **문 폭**은 미규정이라, 건물마다 1칸/2칸이 섞여 문·진입로·트리거·아트가 제각각이었다(창고·축사만 2칸, 나머지 1칸). owner 결정(2026-07-02 grill): **전 건물 문을 기본 2칸으로 통일**하고, 이를 깨끗하게 성립시키기 위해 **신규 건물 footprint 폭을 짝수로 기본**한다. 이 결정은 [ADR-0036] §2(진입 방향·정렬)의 확장이다.

## 결정 (규약)

1. **기본 문 폭 = 2칸.** 거의 모든 주거·상점·서비스 건물의 외관 문은 2 논리 타일 폭.
2. **문 = 정수 타일 N칸에 스냅.** 타일은 최소 단위라 문은 항상 정수 좌표 N칸(`x..x+N-1`)을 차지한다 — **반칸(.5) 배치는 구조상 불가능**. "위치 자유"여도 어느 N칸이냐만 자유다.
3. **신규 건물 footprint 폭 = 짝수(기본).** 짝수폭이라야 2칸 문이 중앙 seam을 straddle해 **완벽 중앙**에 앉는다. 홀수폭은 2칸 문이 반드시 반칸 치우친다(못 피함) → 홀수폭은 "문 1칸(예외)" 또는 "반칸 쏠림 감수"할 때만 쓴다.
4. **실내문 ≡ 외관문.** 같은 문이므로 실내 퇴장 문과 외관 진입 문은 **(a) 폭이 같고 (b) 중심 대비 쏠림 방향·정도가 같다**. 좌표 일치가 아니라(실내 방이 외관보다 큼) 상대 위치 일치 — 외관문이 오른쪽으로 쏠렸으면 실내문도 방 중심에서 오른쪽으로 쏠린다.
5. **불변식: 문 폭 = 진입로 폭 = 트리거 폭.** 2칸 문 → 진입로도 2칸, 진입 트리거는 문 전 칸 어디서나 발동. (창고·축사가 이미 이 패턴 — [gemini-shed-barn-spec REVISION 4] "입구 2칸인 건물은 길도 2칸".)
6. **예외 칸수는 건물 성격이 정한다**(기계적 임계 아님):
   - **1칸** = footprint가 좁아 2칸 문이 벽을 다 먹는 소형(폭 ≤ 3칸) 또는 좁은 쪽문이 컨셉(미호 한옥 미닫이 실폭~1칸).
   - **3칸+** = 대형 정문 컨셉(카페·길드·혼백관 등 웅장한 대문). 폭이 넉넉한 건물만.
7. **잠긴(비-enterable) 외관 = 아트만 N칸 문**(시각 일관성), 그리드 PATH·트리거·진입로는 깖지 않음(어차피 진입 불가). 축사는 비-enterable이나 돌봄 RMB 트리거가 2칸 문 위에 이미 있어 유지.

## 적용 범위

- **신규·재생성 건물**: 위 규약 전면 적용. 곧 재생성할 **본가·창고·축사**부터 — 본가는 현재 홀수 9폭이라 **재생성 김에 짝수폭(8 or 10)으로 조정**하고 2칸 문 중앙화.
- **기존 홀수폭 유지 건물**(멜집5 등, 지금 재생성 대상 아님): footprint는 그대로 두고 문만 반칸 쏠린 2칸으로(재배치 회귀 회피). 각자 재생성 시 짝수화.
- 아직 전용 facade 아트 없는 placeholder 건물(만물상·혼백관·생선가게·목공방·대장간·길드 등): 아트 생성 시 이 규약 적용.

## 트레이드오프

레이아웃 자유(임의 폭·문 위치)를 포기하는 대신, **문·진입로·트리거·아트가 전 건물에서 기계적으로 정합**한다(제각각 정렬 버그·아트 재작업 제거). 짝수폭 강제는 배치 시 폭 선택을 절반으로 제약하나, 문 중앙 정합이라는 이득이 그 비용을 상회한다(owner 확정).
