# 인게임 배치 모드 + PROP 좌표 데이터 외부화(layout.json)

> **관계:** [ADR-0020 ③](./0020-item-tool-architecture.md)("별도 콘텐츠 스튜디오 안 만듦 — 정말 인게임 배치가 필요하면 *얇은 디버그 스폰 모드 한 겹*만 허용")의 **escape hatch를 발동**하고, 그 한 겹의 형태·범위를 구체화한다. ADR-0020을 *대체하지 않고 보완*한다(스튜디오 금지 원칙은 유효 — 이건 "한 겹"의 실체화).

## 맥락 — Godot 에디터가 이 장식들의 스튜디오가 아니었다

ADR-0020 ③은 "맵 배치 = Godot의 TileMap·씬 에디터, *Godot 에디터가 이미 그 스튜디오다*"를 전제로 별도 스튜디오를 금지했다. 그런데 Phase 2.8 T3(안식 농원) 데모를 테두리 프레이밍으로 채우려다 그 전제가 이 장식들엔 **안 통함**이 드러났다:

- 현재 외부 장식·실내 가구는 **Godot 씬에 노드로 존재하지 않는다.** `main.gd`의 하드코딩 `PROP_LAYOUT_*`(`Vector2i` 좌표 배열) + `_draw`의 바닥정렬 수동 렌더다(손님·잡귀와 같은 결 — 노드 없는 그리기). 그래서 "에디터에서 드래그"가 *불가능*하다(드래그할 노드가 없음).
- 테두리 프레이밍은 나무·풀·바위·그루터기를 **수십~수백 인스턴스** 흩뿌린다. 좌표를 코드에 손으로 찍는 건 고통이고, 무엇보다 **아트 오너(옥자님)가 "내가 맘대로 수정 배치"하고 싶다**는 요구(grill 2026-06-25)를 코드 편집으론 못 준다.
- 에셋 생성이 "아무렇게나" 된다는 불만도 같은 뿌리다 — *놓아보고 고치는 루프*가 없어서.

즉 ADR-0020이 가정한 "에디터=스튜디오"가 이 데이터엔 성립 안 하므로, ADR-0020이 *명시적으로 남겨둔* "얇은 디버그 한 겹"을 지금 발동한다.

## 결정

**① 인게임 배치 모드(얇은 디버그 한 겹)를 만든다.** 게임 실행 중 토글 키(F9)로 진입 — 마우스로 장식을 집어 드래그·회전·삭제하고, 새 에셋을 클릭으로 놓는다. 저장(Ctrl+S) 시 좌표가 `layout.json`에 기록된다. **개발 전용 디버그 레이어**(릴리스 빌드 비활성), 게임 메카닉·이동·세이브 무변경(art/저작 패스).

**② PROP 좌표를 전 구역 `layout.json`으로 외부화한다.** `PROP_LAYOUT_HOME`·`PROP_LAYOUT_CAFE`·`PROP_LAYOUT_VILLAGE_HOUSE`(+ 신규 테두리 장식)를 main.gd 코드에서 데이터로 빼 **런타임 로드**한다. 배치 모드의 저장 백엔드이자 ADR-0020 ①(데이터 주도) 정신의 연장. **텍스처·SOLID 여부·등불/빛 공유 좌표는 여전히 코드가 정의**(데이터는 *위치*만) — 좌표만 데이터로 나가 회귀 면적을 위치로 한정한다.

**③ "스튜디오 금지"는 유효하다.** 별도 앱·무거운 에디터 UI는 안 만든다. 이건 게임 프로세스 안의 디버그 모드 한 겹 + 데이터 파일 하나다. 더 커지려는(타임라인·다중 레이어 편집기 등) 압력이 오면 그 구체 필요를 ADR로 재검토한다.

## 폐기한 대안

- **(a) Godot 씬/TileMap 노드로 전면 이주** — ADR-0020 정신엔 가장 정석이나, `_draw` 바닥정렬·등불/빛 공유 좌표(`LANTERN_TILES_*`)·SOLID 충돌 생성(`SOLID_PROPS`·`_prop_body`)을 전부 노드 기반으로 재작성 = 검증된 15채 실내·카페·집의 **대규모 회귀 위험**. 아트 오너가 Godot 에디터에 익숙해야 한다는 전제도 인게임 드래그보다 약하다. 인게임 모드가 같은 "직접 배치"를 회귀 없이 준다.
- **(b) 데이터 파일만(드래그 없이 Claude가 편집)** — 가장 싸지만 "*내가* 맘대로"가 아니라 *말로 시키는* 경험. 수십~수백 인스턴스를 대화로 미세조정하는 건 비현실적. 데이터 외부화(②)는 채택하되 그 위에 드래그(①)를 얹는다.
- **(c) 외부화를 HOME만 점진** — 회귀 최소화엔 유리하나, 배치 모드를 T4~T9·카페·마을에 바로 못 써 일관성이 깨진다. 전 구역 한 번에 외부화하고 회귀를 **전 스위트 + 봇 21일 + 부팅 클린**으로 빡세게 잠근다(grill 2026-06-25 선택).

## Consequences

- **장식 위치의 진실의 원천이 코드 → `layout.json`으로 이동한다.** 미래 독자가 "왜 장식이 코드가 아니라 json?"을 물으면 이 ADR이 답이다(배치 모드의 저장 백엔드).
- **회귀 게이트:** 외부화 전/후 `map_dump`·`home_dump`·`interior_dump` 육안 동일 + 전 회귀 스위트 PASS + 봇 21일 9/9 + 부팅 클린(에러·경고·orphan 0). 좌표 데이터가 *기존 하드코딩 값과 바이트 동등*하게 출발(멱등 이주)함이 회귀 0의 근거.
- **에셋 생성 워크플로에 스펙 게이트가 선다(별건, 프로세스).** 생성 전 PixelLab 프롬프트·크기·톤·배치를 "스펙 카드"로 사람이 승인(크레딧 0, ADR-0001 글루) — 이 ADR의 배치 모드와 짝을 이뤄 "생성 전 체크 → 생성 → 인게임 배치 수정" 루프를 만든다. 게이트 자체는 매니페스트 노트로 충분해 ADR로 박지 않는다.
- **릴리스 안전:** 배치 모드·`layout.json` 쓰기는 디버그 빌드 전용. 릴리스에선 `layout.json`을 읽기만 한다(드래그·저장 비활성).
- **이 모델을 되돌리거나(코드로 회귀) 별도 스튜디오 앱으로 키우려면** 이 ADR을 새 ADR로 대체해야 한다.
