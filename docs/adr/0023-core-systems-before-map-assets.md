# 코어 시스템(Phase 2.7)을 맵 에셋(Phase 2.8)보다 먼저 — 착수 순서 역전 + 번호 재정렬

> **개정:** 2026-06-23 grill의 시퀀싱(ROADMAP 배너 + 메모리 `okja-demo-visuals-first-milestone`)을 *뒤집는다*. 당시 번호는 맵 에셋=Phase 2.7 / 코어=Phase 2.8이었으나, 코어를 앞으로 당기며 **번호도 오름차순으로 재정렬**했다(코어=2.7, 맵=2.8). ADR 아니던 결정이라 같은 결로 ROADMAP·메모리에 반영했고, 되돌리기 비싼 트레이드오프라 ADR로도 못박는다.

## 맥락 — "비주얼 먼저"의 전제가 깨졌다

2026-06-23 grill은 코지-와이드 피벗(ADR-0018)으로 넓어진 *휑한* 맵을 먼저 채우려 **맵 에셋(+조연 캐스트)을 코어 시스템(아이템·도구·UI)보다 앞**에 뒀다(당시 번호 2.7=맵 / 2.8=코어). 동기는 **옥자님(실재 컨셉카페·원작 IP 주인)에게 보일 비주얼 데모 마일스톤**이었다("진척이 눈에 보여야" 동력).

2026-06-24 재grill에서 이 가지를 따라 내려가다 두 모순을 만났다:

1. **코어의 UI는 단독 스텝이 아니다.** 핫바 12칸·인벤토리·상점은 그 밑 *일반화된 아이템/도구 시스템*(현재 없음 — 인벤토리=작물 2-Dictionary, "도구"=`field.next_action` 단일키에 암묵) 위에서만 작동한다. UI는 그 시스템의 *얼굴*이라 "UI만 먼저"로 못 쪼갠다 — **코어 전체가 한 덩어리**이고, Phase 3 모든 활동 루프(낚시·채광·가공)의 *하드 의존*이다.
2. **"조연 캐스트 전원 제작"은 residents.md·Phase 3 ④와 충돌한다.** residents.md는 "본체를 한 명씩 + 그때 집 재도색"(미리 다 만들면 갈아엎는 낭비)을 명시하고, Phase 3 ④는 "T1 주민 한 명씩 + 스토리 한 개씩"이다. 결혼 서브시스템(Phase 3 ④) 없이 11명 초상화·도트·집을 미리 만드는 건 조기작업이다.

## 결정

**코어 시스템이 미화보다 먼저다(우선순위 사다리 "기능 > 예쁨"). 번호도 빌드 순서대로 재정렬한다(헷갈림 방지).**

```
Phase 2.7(코어: 아이템·도구·UI, 한 덩어리)
  → Phase 2.8(맵 환경 에셋 — 도달 가능 구역·베이스 팔레트 1벌)
  → Phase 3(활동 루프 → 심화 → 시나리오 → 전투)
```

1. **코어 시스템 = Phase 2.7, 착수 1순위.** 아이템 시스템·도구 선택 모델 피벗·HUD·공통 인벤토리 프레임을 그레이박스로 한 덩어리로 세운다(ADR-0020/0019).
2. **맵 에셋 = Phase 2.8, 착수 2순위.** 옥자님 비주얼 데모 마일스톤은 *코어 위에서* 유지된다. 범위 축소: **도달 가능 구역만**(나락·잠긴 전투 스테이지 아트는 Phase 3 ⑤로) · **베이스 팔레트 1벌**(절기 4종 스왑은 Phase 3 절기 시스템과).
3. **조연 캐스트 전원 블록 폐기 → Phase 3 ④로 흡수.** T1 주민은 시나리오에서 본체+집 재도색+초상화+스토리 한 묶음으로 *한 명씩*(residents.md 점진 추가, ADR-0014).

## 폐기한 대안

- **(a) 맵 에셋 먼저 유지(비주얼-우선)** — UI·도구 시스템 없이 맵만 예쁜 데모는 *걸어다니는 그림*이라 "진짜 스타듀형 게임"의 손맛을 못 보인다. 게다가 코어를 뒤로 미루면 *검증된 농사 입력을 갈아엎는 최대 리스크*(도구 피벗)가 데모 뒤로 밀릴 뿐이다. 데모 설득력은 코어가 굴러가는 위에서 비주얼로도 충분하다.
- **(b) 절기 팔레트·조연·잠긴 구역까지 맵 에셋 Phase에서 다 칠하기** — 절기 시스템·결혼 시스템·전투가 없는 상태의 조기 에셋(붙일 때 갈아엎는 낭비). residents.md 점진 추가·"한 시스템 100% 후 다음" 위반.
- **(c) 번호는 그대로 두고 물리적으로만 2.8을 앞에** — "2.8 다음 2.7" 내림차순 표기가 헷갈린다는 피드백 → 번호를 빌드 순서대로 재정렬(코어=2.7, 맵=2.8).

## Consequences

- **메모리 `okja-demo-visuals-first-milestone` 역전 반영**(옛 "맵 먼저" 정정, 데모 마일스톤은 코어 위에서 유지).
- **ROADMAP 갱신** — 두 Phase 물리 재배치+번호 재정렬(코어=2.7·맵=2.8)·조연 폐기·맵 범위 결정·Phase 3 머리말 순서·루프 단위 빌드 순서 블록. 완료 로그(C2~C9)의 "다음 에셋 Phase" forward-pointer도 새 번호(2.8)로 갱신.
- **shrimp 재등록** — `2.7 코어 → 2.8 맵에셋 → Phase 3 루프 체인`(루프 단위, 원자 분해는 각 루프 착수 grill).
- **다시 비주얼-우선(맵 먼저)으로 되돌리려면** 이 ADR을 새 ADR로 대체해야 한다.
