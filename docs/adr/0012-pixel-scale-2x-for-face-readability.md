# 픽셀 스케일 2배 상향 — 인게임 얼굴 가독성 (ADR-0003 규격 개정)

ADR-0003은 16×16 타일 / 16×32 캐릭터 / 320×180 내부해상도를 고정했다. P2.3 캐스트 도색 후
**직접 플레이테스트에서 "인게임 캐릭터 얼굴이 안 보인다 / 뭉개진다 / 이쁘게 안 나온다"가
반복 피드백**으로 나왔다. 16×32(low top-down)에서 얼굴은 ~5–6px라 어떤 생성/후처리로도
또렷·매력적으로 만들 수 없음이 확인됐다(슬림·큰머리·머리확대 후처리 모두 실패 — p2.0-spike §10).

## 결정

**인게임 픽셀 스케일을 전부 2배로 올린다.** 그리드·좌표·게임 로직은 불변(타일 *개수* 그대로).
바뀌는 건 "타일당 픽셀"과 그에 묶인 아트·해상도뿐이다.

- **캐릭터**: 16×32 → **~32×64**(콘텐츠 ~58-70px). 얼굴에 픽셀이 충분해 깔끔·매력적. 시트 프레임
  **80px**, 발치정렬, native(다운스케일 없음). **생성 = PixelLab `mode=standard`, `size=56`, `detail=high`,
  `proportions` custom 고정값**(head_size 1.4·slim) — ★체형 통일을 위해 v3가 아니라 standard. v3는
  `proportions`를 무시해 캐릭터마다 머리:몸 비율이 달라졌다(체형 안 맞던 원인). standard는 5종에 같은
  proportions를 줘 체형을 통일하고, high detail이라 선명함도 유지(상세=p2.0-spike §10.8).
- **타일**: 아트는 **16×16 유지**(`TILE_ART=16`, 작업량 1/4 이점 보존)하되 **Ground/Field
  TileMapLayer를 ×2 스케일**해 화면엔 32px로 렌더. 월드 좌표는 `TILE=32`.
- **가구·소품**: 16px 원본을 `_draw_props`에서 ×2로 그린다.
- **내부 해상도**: 320×180 → **640×360**(정수배 ×2 = 1280×720 창 유지, 크리스프). UI는
  CanvasLayer ×2 스케일로 320×180 레이아웃을 그대로 채워 위치·크기 불변.
- **파생 2배**: 플레이어 콜리전(28×20)·이동속도(160)·WALL 충돌 폴리곤(타일맵 스케일로 자동)·
  소울 등불 빛웅덩이(LAMP_TEXTURE_SCALE 3.2).

이로써 캐릭터:타일:가구 비율이 일관되고(캐릭터 = 약 2타일), 640×360에서 20×11타일이 보여
팔로우 카메라가 복원된다.

## 근거

- 얼굴 매력은 본 게임의 1순위 사용자 요구로 확인됐다(반복 플레이테스트). ADR-0003의 "도트 부담
  최소화"보다 우선한다고 사용자가 명시적으로 결정.
- 타일 아트를 16px로 유지(렌더만 ×2)해 ADR-0003의 작업량 1/4 이점은 보존한다(에셋 재제작 불요).
- 타일 *그리드*를 안 바꿔 좌석·밤 스폿·NPC 출근·밭·동선 좌표와 시스템·회귀 테스트 8종이 불변.

## ADR-0003과의 관계

ADR-0003의 **탑다운 3/4뷰**, **치비 ~2.5등신**, **정수배 스케일**, **표정=별도 일러스트 초상화**는
그대로 유효하다. 이 ADR은 ADR-0003의 *수치 규격*(타일/캐릭터 px·내부해상도)만 2배로 개정한다.
초상화(ADR-0003)는 여전히 대화 얼굴의 주력이고, 이번 변경은 인게임 도트 얼굴도 읽히게 보완한다.

## 결과

- 환경 아트(타일·가구)는 16px 원본 그대로, 렌더 시 ×2. 신규 환경 에셋도 16px로 만들면 됨.
- 캐릭터 신규/수정은 standard size56 + 공통 proportions 고정값으로(5종 동일값 = 체형 통일). p2.0-spike §10.8에 파이프라인·proportions·캐릭터 ID 기록.
- 검증: 헤드리스 부팅 클린, 회귀 8종(cafe·cafe_margin·night_bar·weave·milestone·npc_station·
  lighting·bana) PASS, map_dump 비율 육안 확인.
