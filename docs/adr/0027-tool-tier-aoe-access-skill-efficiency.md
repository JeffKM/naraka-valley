# 도구 업그레이드 티어 = 범위·접근·용량 / 스킬 = 효율·피지컬 — ADR-0019 도구범위 이관 개정

스타듀 시스템 구체화 grill(2026-06-29, [stardew-systems-catalog](../design/stardew-systems-catalog.md) §1-J)에서
**도구 업그레이드 티어**(대장간 구리→이리듐)를 나라카에 채택할지 따졌다. 그 과정에서
[ADR-0019]와의 충돌이 드러났다: 스타듀는 *도구 티어가 AoE(여러 칸 범위)를 키우는데*,
ADR-0019는 **#1·#4·#5에서 AoE("도구 충전 범위 1×1→1×3→3×3")를 스킬 축에 배정**했다.
둘 다 AoE를 키우면 *같은 보상의 중복*(ADR-0008이 피하는 그라인드)이다.

owner 결정: **스타듀 골격을 충실히 따르되 저승 색을 입히는 게 가장 자연스럽다 → AoE를
스타듀처럼 도구 쪽으로 옮기고, 스킬은 효율(에너지·피지컬)에 집중한다.**

## 결정

**도구 업그레이드 티어를 채택하고, 두 축의 역할을 겹치지 않게 가른다.**

1. **도구 티어 = "더 크고 강한 도구" 축 (업화 대장간 업그레이드).** 골드 + 저승 금속 4티어
   (구리→강철→금→이리듐 대응, 자체 명명, [업화 갱도] 채광 산출물과 연결), 순차 업그레이드.
   - **AoE(범위):** 괭이·물뿌리개 1×1 → 1×3 → 3×3 (★ ADR-0019에서 *이관*).
   - **용량:** 물뿌리개 충전 칸수↑.
   - **접근(큰 장애물):** 큰 바위·그루터기 = 도끼/곡괭이 티어 필요. ⚠️ 단 *기본* 채광·벌목은
     0티어에서도 (느려도) 되게 둔다 — 하드 게이트는 *큰 장애물*에 한정("평평≠막힘" 보존,
     아래 §평평≠막힘).
2. **스킬 = "효율·피지컬" 축 (ADR-0019, AoE만 제거).** 행동 반복으로 저절로 오름.
   - **혼력 절감**(행동당 소모↓, 풀 크기 불변 — ADR-0019 #3 유지)
   - **작업 속도**(괭이·곡괭이·도끼 스윙·낚시 캐스팅 등 — 숙련=매끄러움)
   - **낚시 바 크기·거리 / 크리**(광석 2개·피지컬 크리 등)
   - ~~도구 범위(AoE)~~ → **도구 티어로 이관**(이 ADR).
3. **인챈트(Forge·프리즈마틱 샤드) = 🗄 전투 grill 클러스터로 유지**(Phase 3 ⑤ 전투 본체와 함께).

이로써 **도구 = 닿는 범위·뚫는 힘·담는 양 / 스킬 = 싸고·빠르고·날카롭게**로 숫자가 안 겹친다.

## 평평 ≠ 막힘 (ADR-0008 §15 · ADR-0017 정합 재확인)

- **스킬은 여전히 벽이 아니다**(ADR-0019 #5 유지) — 레벨로 *접근*을 게이팅하지 않는다.
- **도구 티어의 "접근"은 *경제* 게이트지 관계/스킬 게이트가 아니다.** 골드·금속을 투자하면
  큰 장애물·깊은 콘텐츠가 열린다(스타듀 동일). ADR-0008/0017이 막는 건 *관계가 base 활동을
  막는 것*이지, 경제적 도구 진행이 아니다. 코지 보존을 위해 *기본* 채광·벌목은 0티어에서도
  돌고, 티어는 *효율·큰 장애물·용량*을 연다(필수 진행 벽 최소화).

## ADR-0019와의 관계 (개정 지점)

- **개정:** ADR-0019 **#1**(농사 보상 "도구 충전 범위 해금")·**#4**(도구 범위 vs 여우불 reach 구분의
  "도구 범위=스킬 축" 전제)·**#5**("레벨이 해금하는 건 편의=도구 범위")에서 **AoE를 스킬 축에서
  뺀다.** 스킬은 혼력 절감·속도·낚시바·크리만.
- **#4의 구분 자체는 보존**(형태만 변경): *능동 범위(이제 **도구 티어**가 키움) vs 여우불 reach(미호가
  *대신* 돌봄=곱셈기)* — "내가 더 넓게 친다 vs 안 쳐도 된다"의 대비는 유지된다. AoE의 *출처*만
  스킬→도구로 옮겨, 3×3을 *도구로* 치고 빠뜨린 칸을 여우불이 메우는 곱은 그대로다.
- **2축(스킬 base + 관계 곱셈기, ADR-0019/0021) 골격·"평평≠막힘"·혼력 풀 불변은 전부 유효.**

## 결과

- **§1-J 도구·§1-I 스킬 행 갱신**(catalog). 도구 티어 = ✅ 결정(AoE·접근·용량·속도 中 접근/용량/AoE),
  스킬 = AoE 제거.
- 구현: 도구 티어(정수, 세이브 상태)를 AoE·용량·장애물 판정에서 읽고, 스킬은 혼력·속도·크리·낚시바에서
  읽는다(숫자 분리 = 구현으로 강제, ADR-0019 #36 결 유지).
- 저승 금속 4티어·각 도구 효과 곡선은 Phase 3 밸런싱 변수. 업화 대장간(클린트 대응)이 업그레이드 무대.
- 이 역할 분리(도구=범위/스킬=효율)를 되돌리려면 새 ADR로 대체.
