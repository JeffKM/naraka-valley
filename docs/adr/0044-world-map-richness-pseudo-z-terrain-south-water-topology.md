# 월드 맵 리치니스 — 의사 단차(pseudo-Z) 다단 지형 + 남향 물 토폴로지

> **개정 대상:** [ADR-0035](./0035-homestead-elevation-cliff-overgrown-redesign.md)의 "높이 = 2D평면 + 절벽 1타일 + 충돌" 전제를 **2행 pseudo-Z 다단**으로 격상한다(z축 도입 아님 — [ADR-0013](./0013-environment-art-32px-native.md) 2D 평면 데이터·세이브·카메라 불변). [ADR-0015](./0015-multi-region-world-map-expansion.md) 월드 토폴로지의 물 구역 배치를 남향 종축으로 재정렬한다.
>
> **성격:** 되돌리기 비쌈(맵 빌드·아트·회귀 좌표·타일종에 박힘) + 놀라움(왜 평면 안 하고 가짜 고저차?) + 실 트레이드오프(평면 유지 vs 다단 리치니스) → ADR.

owner가 인게임 전 구역을 보고 지적(2026-07-01): **"지금 모든 맵이 너무 단조롭잖아."** 완전 평면 타일 바닥 + 절벽 1타일 돌띠 + 마을을 정직하게 동/서로 가르는 수직 강줄기가 장르적 아늑함(cozy)과 시각 깊이를 저해한다. 발단은 [asset-ruleset §4](../design/asset-ruleset.md) "동향 절벽 면 타일 신규 생성"(넝쿨 가림 핵 제거) 과제였으나, grill 결과 **단일 타일 교정이 아니라 월드 전역의 가짜 고저차 시스템 + 물 흐름 재배향**으로 확장됐다.

핵심 제약(불변): **z축 높이가 아니다.** ADR-0035대로 좌표·카메라·세이브는 2D 평면 그대로고, 시각·충돌·동선만 타일 셋업으로 고저차를 *연출*한다(스타듀도 진짜 3D 아님). 본 ADR은 그 "가짜 높이"를 1타일 → **2행 표준**으로 키우고 물가·전 구역에 강제하는 결정이다.

## §1. 다단 절벽 기하 및 통과 규칙 (Cliff Geometry, H=2)

**표준 수직 구조 (남향 기준, H=2행 = 화면 64px):**

| 행 | 타일종 | 통과 | 역할 |
|---|---|---|---|
| `y-1` | `PLATEAU_GRASS` (고지 풀) | 걷기 O, 배치 O | 고지 평면 |
| `y+0` | `CLIFF_LIP` (고지 밑단) | **걷기 O** | 밝은 상단 하이라이트 라인 · **고지 오브젝트 배치 하단 한계선** |
| `y+1` | `CLIFF_FACE` (절벽 상단) | **SOLID(통과 X)** | 암벽 볼륨(충돌체 탑재) |
| `y+2` | `CLIFF_FACE_BASE` (절벽 접지) | **SOLID(통과 X)** | 저지와 만나는 밑단 · **접지 그림자 self-shadow 베이크**([asset-ruleset §1.3(b)]) |
| `y+3` | `LOWER_GRASS` (저지 풀) | 걷기 O | 저지 평면 |

- **H=2 마스터 표준.** 절벽 단면(Face+Base) = 논리 2행 = 64px. "벽"으로 명확히 읽히는 최소 높이. 좁은 고지·축사 인접 등에서 **구역별 상향 가능한 파라미터**(H는 하한 2).
- **동/서향 확장 = 높이의 가로 치환.** 세로 런(East/West-Facing)은 H를 *너비*로 바꿔 `Lip 1열 + Face 2열` 구조를 강제. 고지 동/서 여백이 그만큼 넓어진다.
- **단순 Flip 금지 · NW 광원 재보정 필수.** 동↔서면은 [asset-ruleset §1] NW(좌상단) 광원 때문에 음영이 다르다 → 거울 반전이 아니라 **밝은 엣지가 항상 좌상단**이 되게 픽셀 재보정([ADR-0037] 방향성 변형 정합).
- **계단 절개(Stairs Notch).** 절벽을 종단하는 계단 오브젝트는 `y+1`·`y+2`의 SOLID 충돌을 일시 해제하는 **2폭 세로 내비게이션 트리거 구역**으로 정의(걷기 O). 기존 ADR-0035 계단이 2행을 종단하도록 확장.
- **넝쿨/덤불 = 장식으로 강등.** ADR-0035/phaseB의 "코너 틈 = 넝쿨·덤불 Y-Sort 덮개(가림 핵)"는 폐기 — 코너는 **edge-to-edge로 고지/저지 엣지를 완전히 채운 코너 타일**로 단단히 마감(초록 새어나옴 방지). 넝쿨·덤불은 남기되 *seam 가림용*이 아니라 순수 장식으로만.

## §2. 남향 물 토폴로지 (Water Topology)

- **월드 동선 수직화.** 글로벌 맵 그래프를 **[북: 나루 마을] → [중앙: 삼도천] → [남: 황천해]**의 종형(縱形) 남향 벡터로 재정렬한다([ADR-0015] 물 구역 배치 개정 — 마을에서 *아래로 내려가면* 강→바다).
- **마을 내부 강 통합.** 나루 마을을 동/서로 가르던 **내부 수직 강줄기를 폐기**하고, 마을 **최남단 가장자리를 동서로 횡단하는 배후 강** 구조로 변경(동선 피로 해소 + 마을 통짜화). 다리가 그 배후 강을 건너 낚시터로 잇는다.
- **물가에도 pseudo-Z 강제.** 단조로움을 물리적으로 억제한다 — ① **강둑 단차:** 강물 타일은 상단에 §1의 `CLIFF_FACE`(흙/암석) 단면 ≥1행을 확보해 저지 수면과의 의사 고저차를 시각화. ② **해안선 전이:** 바다 맵 진입부(북단)에 고지 수풀↔백사장을 가르는 **수평 절벽 런** 배치.

## §3. 이행 매핑 (각 구역 Slice가 자기 맵 재배치를 포함 — ADR-0028 인터리브)

owner 확정: **"각 Slice에서 아트·에셋 작업을 진행하면서 맵 재배치도 모두 진행."** 별도 재배치 페이즈를 두지 않고 구역 Slice에 흡수한다.

- **Slice 0 (아트 토대):** 본 ADR + [asset-ruleset §4] 규격 **문서 박제**(= 지금 완료). 기존 Slice 0 항목 "절벽 동면 타일"(단일 타일)은 **폐기·이관** — 다단 세트는 Slice 1 재배치 산출물.
- **Slice 1 (안식 농원):** 농장 내 1타일 돌띠 제거 + **2행 pseudo-Z 서쪽 고지(하늘 목장)** 최종 구현(동향 면 신규·넝쿨핵 제거·계단 2행 종단). ADR-0035 Phase B 기하 **개정 구현**.
- **Slice 2 (나루 마을):** 마을 배후 강 스위칭 + 내부 이분할 지형 전면 리팩토링(+ 주민 집 디자인·로스터·맵 에셋).
- **Slice 3 (삼도천·황천해):** 종형 맵 렌더 + 강둑/해안선 단차 배치.
