# 재미 게이트를 단일 농사 루프 → 핵심 3캐릭터 루프 통합 그레이박스로 확장한다

Dear My Naraka의 **재미 게이트**(ROADMAP T4.4)는 원래 *농사 단일 루프 + 14일 슬라이스*로 "또 하고 싶나?"를 판정하도록 설계됐다(ADR-0001 그레이박스 우선, [game-loops](../design/game-loops.md) 3절 출시순서 "지금 Phase 1: 농사 1층 + 여우불 곡선 = 게이트 통과 깊이까지만"). T4.4에서 헤드리스 플레이테스트 봇이 재미의 **전제조건 5/5**(무막힘·우상향 엔진·페이싱·선택의 가치·대조군)를 통과시키고 기술적 GO를 권고했다([t4.4-fun-gate-verdict](../design/t4.4-fun-gate-verdict.md)).

그러나 **사람이 직접 14일을 플레이한 결과, "단일 농사 루프만으로는 '또 하고 싶나'를 판정할 수 없다 — 여러 루프가 직조될 때 비로소 재미가 드러난다"**는 결론에 도달했다. 이는 게임 자체가 망가졌다는 뜻이 아니다(봇 전제조건 5/5는 유효하고, 농사 루프는 기술적으로 건강하다). **판정의 대상이 너무 좁았다**는 뜻이다.

이 결론은 청사진이 이미 안고 있던 내부 모순을 드러냈다. [game-loops](../design/game-loops.md)는 한편으로 *"재미의 '또 하고 싶다'는 주로 3층(우상향 곡선)에서 나온다 — 1층만 있으면 루프가 평평하다"*(0절)와 *"루프들이 시간을 두고 맞물리나 — 단일 루프 평평함 방지"*(5절 검증)를 말하면서, 다른 한편으로 *게이트를 농사 단일 루프로 통과시키도록*(3절) 설계해 두었다. 농사는 1층만 완성됐고 3층은 여우불 하나뿐이라 — **체감한 평평함은 청사진이 예측한 결과**다. 평평할 걸 알면서 단일 루프로 게이트를 보려 한 것이 모순이었다.

## 결정

**재미 게이트의 판정 대상을, 농사 단일 루프에서 세 속죄 캐릭터의 활동 루프가 어우러지는 통합 그레이박스 슬라이스로 확장한다.**

- 게이트 전에 그레이박스로 만드는 범위: **농사(미호) + 카페 경제·회계(멜) + 밤 경비·손님 응대(바나)** 세 루프의 1층 MVP + **낮↔밤 직조**가 한 번 작동하는 슬라이스. 각 캐릭터 루프는 여우불에 대응하는 **관계 보상축**을 가진다(ADR-0004·0005, [miho-heart-arc](../design/miho-heart-arc.md) 틀 재사용).
- 게이트 재판정은 이 통합 슬라이스 위에서 한다. "세 캐릭터 루프가 낮↔밤으로 맞물리며 우상향하나, 또 하고 싶나"가 새 판정 질문이다.

## 범위 통제 — 게이트의 손절 기능을 지킨다

게이트의 존재 이유는 **에셋·확장에 크게 투자하기 전, 적은 비용으로 재미를 검증해 손절 지점을 만드는 것**이다. "어우러짐을 보려면 다 만들어야 한다"를 무제한 받아들이면 게이트가 게임 완성 시점으로 밀려 손절 기능을 잃고, ROADMAP 1순위 리스크인 **번아웃**(파트타임 장기전)이 최대가 된다. 따라서 범위를 명시적으로 막는다:

- **에셋(도트·초상화·사운드)은 통합 게이트 통과 후.** 통합 슬라이스도 끝까지 그레이박스다(ADR-0001 불변).
- **순수 활동 루프(낚시·채광·전투)와 합류점(가공·서빙)은 게이트 후**(Phase 3 유지). 이들은 관계 보상축이 없는 "게이트 열쇠" 활동이라(ADR-0005) 직조 판정의 핵심이 아니다.
- **멜·바나는 1층 MVP + 관계 보상축 + 직조까지만.** 깊은 2·3층(시세 변동·손님 다양성 등)은 게이트 후 재방문. game-loops 2.3/2.4 스케치가 MVP의 출발점이다.
- **"한 번에 한 루프 끝까지"**(개발 원칙) 유지 — 멜 → 바나 순차로 쌓고 동시에 만들지 않는다.

## 폐기한 대안

- **(a) 농사 단일 루프로 게이트 강행** — 직접 플레이로 평평함이 확인됐다. 게이트가 거짓 음성(재미없음으로 오판)을 낼 위험.
- **(b) 청사진 전부 그레이박스화**(낚시·멜·바나·채광·전투·가공) — 사실상 게임 완성이라 게이트가 손절점 역할을 잃고 번아웃 위험 최대. game-loops 검증 "한 번에 한 루프 끝까지"와 정면충돌.
- **(c) 농사 3층만 깊이 파기**(재생장·품질·숙련) — 단일 루프 우상향은 보강하나 *직조*는 검증 못 한다. 사용자가 체감한 핵심("루프가 어우러지는지")과 어긋남.

## Consequences

- **ROADMAP Phase 재편:** 통합 그레이박스 Phase(멜·바나 루프 + 직조 + 통합 게이트)를 기존 에셋 Phase 앞에 삽입하고, 에셋 작업을 그 뒤로 민다. T4.4는 "통과/실패"가 아니라 **"판정 보류 — 범위 확장"**으로 종결한다(봇 전제조건 5/5는 유효 기록으로 남긴다).
- **게이트 비용 증가를 인정하되 손절점은 보존.** 통합 슬라이스가 그레이박스(에셋 0)라 비용은 통제되고, 에셋·확장이라는 큰 투자 *앞*에 게이트가 여전히 선다.
- **되돌림 경로 명시:** 통합 그레이박스 중 번아웃·일정 위험이 커지면, 게이트 범위를 "최소 직조 증명"(농사 + 한 루프 + 직조 하나)으로 축소할 수 있다. 범위를 줄이는 건 이 ADR을 깨지 않는다.
- **멜·바나의 관계 보상축·직조·서사 조각 내용은 서랍**(game-loops 4절)이며, 각 루프 착수 시 **grill 세션**으로 채운다(문서가 예고한 재방문 시점). ADR-0005의 멜=사실·바나=목격 조각 각도는 유지한다.
- **단일 농사 루프로 게이트를 통과시키는 원안으로 복귀하려면** 이 ADR을 새 ADR로 대체해야 한다.
