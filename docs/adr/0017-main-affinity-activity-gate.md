# 메인 호감도는 활동 성과 게이트(활동→관계), 척추는 세 활동을 모두 요구

메인 3인(미호·멜·바나)의 호감도가 *대화·선물 누적*이 아니라 **각자의 속죄 활동 실적**으로 오르게 한다(활동→관계). 그리고 그 귀결 — 척추 해결이 농사·카페·밤경비 *세 활동을 모두* 일정 수준 요구하게 됨 — 을 버그가 아니라 의도된 정합으로 못박는다. [ADR-0008](./0008-growth-model-relationship-multiplier.md)이 정의한 *관계→활동(곱셈기)* 축에 **역축(활동→관계 게이트)** 을 더하는 결정이다. 상세 설계는 [서사 바이블 §6.1·§6.5](../design/narrative-bible.md), 도달 수치 grill(2026-06-23) 산출.

## 결정

- **호감도 동력 = 메인/서브 이원화.**
  - **메인 3인 = 활동 성과 게이트(활동→관계).** 호감도가 그 캐릭터의 *속죄 활동 실적*으로 큰다 — 미호 = 작물 누적 수확(/특정 작물 성과), 멜 = 누적 출하 자금·카페 매출, 바나 = 밤 영업 방어 횟수·전투 성과. "함께 일해야 깊어진다"가 문자 그대로. 옥자의 "행동으로 갚는 deed 모델"(바이블 §6.1)을 메인 3인으로 확장한 것.
  - **서브(조연 T1 11인) = 스타듀식 '쉼터'.** 게임 내 성과를 요구하지 않는다 — 동선에서 마주쳐 대화 + 선호 선물로 부드럽게(`affinity.gd` 두 채널). 활동 압박의 쉼터.
- **성과 게이트의 *종류*가 캐릭터마다 다르다**(수확량·매출·방어). ADR-0008이 곱셈기 종류를 분화(미호 자동화·멜 마진·바나 보호)해 그라인드를 피한 것과 같은 원리를 호감도 동력에도 적용 — 같은 "+점수 누적"의 3연속이 아니다.
- **방향 불변식(ADR-0008 불침범):** *활동이 관계를 게이팅*(활동→관계)할 뿐, *관계가 활동을 게이팅*(관계→활동)하지 않는다. ♡0에서도 모든 활동은 base rate로 굴러간다("평평 ≠ 막힘"). 관계→활동은 여전히 *곱셈기*다(ADR-0008 그대로).
- **게이팅 축 = 순수 관계 상태 게이트(날짜 무관).** 척추 도달에 절대 일수를 박지 않는다(달력 시스템도 아직 없음, 룬팩토리식). "막판 갑툭튀"는 날짜 바닥이 아니라 **호감도 진행의 끈끈함**으로 막는다 — ♡0~3은 점수가 차도 *특정 장소 대화/컷신(단계 관문 이벤트)*을 봐야 진급하고, ♡4~5는 *의지 시험*(특정 행동/상징 정표)으로 열린다.
- **★ 척추 = 세 활동 모두(의도된 정합).** 척추 해결 게이트가 `AND(세 조각 전부 ♡4)`(OR 기각, 바이블 §6.1)인데 메인 ♡4가 활동 성과 게이트이므로 → **척추(엔드게임 이야기)를 보려면 농사·카페·밤경비 세 활동을 모두 일정 수준 해야 한다.** 세 조각=세 죄=세 활동의 1:1 거울(바이블 §2.1), **"통합 플레이 = 속죄."**

## Consequences

- **ADR-0008 보완(개정 아님).** ADR-0008의 *관계→활동 곱셈기*·*"이야기만 호감도 게이팅"*·*"평평 ≠ 막힘"* 은 전부 그대로 산다. 본 ADR은 그 위에 *활동→관계*라는 직교 축을 더할 뿐 — 두 축이 양방향 강화 루프를 이룬다(활동하면 관계가 크고, 관계가 크면 곱셈기로 활동이 빨라짐).
- **활동파 부담을 눈 뜨고 수용.** 척추(엔드게임 이야기)를 보려면 세 루프를 다 경험해야 하므로, "한 활동만 파는" 플레이어에겐 무겁다. 그러나 ① 활동 *자체*는 ♡0에서도 안 막히고(ADR-0008 보존), ② 게이팅되는 건 *이야기*뿐이라 ADR-0008 "이야기만 게이팅" 문구 내, ③ 척추는 풀 콘텐츠 경험 후의 최종 엔드게임이라 통합 플레이 강제가 테마(속죄)와 정합.
- **척추는 전투(Phase 3 맨 끝) 의존.** 바나 성과 게이트의 "전투 성과"는 전투 구현에 의존 — 1층(얇은 방어)에선 "밤 방어 횟수"로 ♡ 진행하고, 전투 도입 시 "전투 성과"가 상단 단계에 합류. 척추가 전투에 걸리는 건 척추=최종 엔드게임이라 페이싱상 자연스럽다.
- **절대 수치는 미정·후속.** 미호 ♡4 = 수확 N개 같은 *정확한 도달 수치*는 활동 경제(카페 매출 곡선·밤 방어·낚시·채광 base rate)가 Phase 3에 깔린 뒤 역산·튜닝한다. 본 ADR은 *구조*(동력의 방향·이원화·게이팅 축)만 고정한다.
- **구현은 Phase 3 ④(시나리오).** ADR-0005 운영 규칙대로 메인 서사·관계 심화는 그레이박스 슬라이스가 깔린 뒤. 본 결정은 *설계*만 — 슬라이스(14/21일)는 여전히 척추·메인 ♡4 무관.
