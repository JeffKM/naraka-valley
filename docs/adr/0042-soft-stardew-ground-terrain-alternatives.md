# 지면 아트 부드럽게(스타듀 결) + 터레인 변종 타일

owner가 인게임(스크린샷 2026-06-30)에서 바닥이 "과장되게 울퉁불퉁·지저분, 경계 타일이 반복되고 스타듀처럼 자연스럽지 않다"고 지적. 스타듀 실제 구현을 조사([ground-composition.md](../design/ground-composition.md) §1.5)한 결과, 원인은 **합성 방식이 아니라 (1) 청키·고대비 아트 + (2) config당 단일 전환 타일(변종 0)**이었다. 이를 두 결정으로 교정한다.

## 1. 지면/지형 아트 = 스타듀처럼 부드럽게 ([ADR-0035]·[ADR-0041] 부분 개정)

- ADR-0035/§0의 **"거친 청키 16논리×2 + 단색 외곽선 low detail"**와 ADR-0041의 **"미디엄 양자화(5색 램프 nearest 스냅)"**는 **지면/지형 타일**에서 *과장된 울퉁불퉁·포스터화*를 만든다. 스타듀 레퍼런스는 더 **부드럽고 저대비·미세**하다.
- **개정:** 지면/지형 타일은 **÷2 청키화 폐기 + 저대비·다(多)톤(하드 5색 스냅 금지)**으로 부드럽게 생성·후처리한다. warm 팔레트 *방향*(§9, [ADR-0041] 마스터 램프)은 유지하되 **하드 양자화 대신 부드러운 hue/sat 보정**(원 디테일·계조 보존). PixelLab도 `detail: low`→**medium** 허용(지형 한정).
- **범위:** 우선 **지면(풀·흙·길·밭·물)·지면 디테일**. 캐릭터·건물·프롭의 청키 정도는 별도 판단(일관성 위해 추후 같은 방향 검토 가능, 본 ADR은 강제 안 함).
- 근거: 스타듀 grass tuft = 15×20 작은 저대비 스프라이트, 베이스 지형 = 작가 손그림 변종 타일(알고리즘 1-combo-1-tile 아님).

## 2. 자연 변주 = Godot 터레인 alternative tile ([ADR-0041] 양자화와 별개 — 변주 강제)

- "같은 경계라도 풀이 더/덜 나오고 코너가 둥근" 스타듀의 자연스러움 = **config당 변종 타일 여러 개를 작가가 골라 깐 것**. 우리는 이를 **Godot TileSet alternative tile**(같은 peering bit에 변종 N개 → 오토타일이 랜덤 선택)로 자동 재현한다.
- 베이스 잔디·전환 타일 모두 **변종 2~3종**을 둬 체커 반복 제거. 잔디 tuft는 **작고 부드러운 오버레이**(스타듀식, 큰 청키 클럼프 폐기)로 유지하거나 변종 타일에 흡수.

## 결과/이행

- `tools/quantize_to_palette.py`의 청키(÷2)·하드 스냅을 지면 타일에서 끔(부드러운 보정 경로 추가).
- `_GD_TABLES` 큰 클럼프 오버레이 → 작은 부드러운 tuft + 터레인 변종으로 전환.
