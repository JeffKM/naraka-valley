# 건물 정사영(정면 Facade) 및 남향 진입 불변식

탑다운 평면 세계에서 이소메트릭(3/4)·다방향 건물은 직교 그리드·길과 따로 놀며 둥둥 떠 보인다(owner 지적, 2026-06-30 청키 본가 사례). 모든 건물·대형 구조물 에셋을 **정면 facade + 남향 진입**으로 통일해 그리드 정합·길 정렬·일관 통합을 강제한다.

## 1. 비주얼 프로젝션 (Visual Projection)
- 모든 건물·대형 구조물은 **정면(Front-Facing) Facade** 규격을 준수한다.
- 정면 벽면은 평면으로 렌더링하되, **지붕의 Y축 깊이를 1~2 논리 타일(= 화면 2~4칸px×2)만큼만** 노출시켜 2.5D 착시 입체감을 확보한다.
- **측면 벽 렌더링은 전면 금지**(이소메트릭 금지). PixelLab `view=side` + §2 프롬프트로 강제.

## 2. 진입 방향 및 정렬 (Door Alignment)
1. **남향 고정**: 모든 상호작용 가능한 문(Door)은 남쪽(화면 아래)을 향해 열리며, 플레이어 진입 벡터는 언제나 '아래에서 위로(Northward)'.
2. **DOOR 타일 1:1 매핑**: 에셋의 문 중심 픽셀 열 = 엔진 `*_DOOR` 내비게이션 타일 노드와 데이터상 1:1 일치. `_carve_paths`는 이 좌표의 **남쪽 인접 타일**까지 경로를 자동 생성한다.

> 입체감은 이소메트릭이 아니라 **지붕 깊이 + NW 광원 음영([ADR-0035]/asset-ruleset §1) + 앞쪽 풀/덤불 Y-Sort 겹침**으로 낸다.
> 근거 규칙: [asset-ruleset.md](../design/asset-ruleset.md) §2.
