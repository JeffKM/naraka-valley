# 건물 정사영(정면 Facade) 및 남향 진입 불변식

탑다운 평면 세계에서 이소메트릭(3/4)·다방향 건물은 직교 그리드·길과 따로 놀며 둥둥 떠 보인다(owner 지적, 2026-06-30 청키 본가 사례). 모든 건물·대형 구조물 에셋을 **정면 facade + 남향 진입**으로 통일해 그리드 정합·길 정렬·일관 통합을 강제한다.

## 1. 비주얼 프로젝션 (Visual Projection)
- 모든 건물·대형 구조물은 **정면(Front-Facing) Facade** 규격을 준수한다.
- 정면 벽면은 평면으로 렌더링하되, **지붕의 Y축 깊이를 1~2 논리 타일(= 화면 2~4칸px×2)만큼만** 노출시켜 2.5D 착시 입체감을 확보한다.
- **측면 벽 렌더링은 전면 금지**(이소메트릭 금지). PixelLab `view=side` + §2 프롬프트로 강제.

## 2. 진입 방향 및 정렬 (Door Alignment)
1. **남향 고정**: 모든 상호작용 가능한 문(Door)은 남쪽(화면 아래)을 향해 열리며, 플레이어 진입 벡터는 언제나 '아래에서 위로(Northward)'.
2. **DOOR 타일 1:1 매핑**: 에셋의 문 중심 픽셀 열 = 엔진 `*_DOOR` 내비게이션 타일 노드와 데이터상 1:1 일치. `_carve_paths`는 이 좌표의 **남쪽 인접 타일**까지 경로를 자동 생성한다.

> 입체감은 이소메트릭이 아니라 **지붕 깊이 + NW 광원 음영([ADR-0035]/asset-ruleset §1) + 앞쪽 풀/덤불 Y-Sort 겹침**으로 낸다.
> 근거 규칙: [asset-ruleset.md](../design/asset-ruleset.md) §2.

## 3. 건물 실루엣·디테일 규격 (owner md 2026-07-01 반영 — 필수 반영, 누락 금지)

> 근거: `docs/design/gemini-farmhouse-spec.md`(owner 전면 리터칭 지시 원문). Phase C 재작업 건물 생성 시 아래를 **전부** 반영한다(과거 누락 사례 = 프롬프트에 넣어도 PixelLab이 안 찍음 → 검수에서 누락 시 재생성).

1. **횡적 확장 실루엣:** 위로 좁고 뾰족한 형태 금지. **가로 폭을 넓혀** 널찍·안정감 있는 가옥으로. (본가 기준 세로보다 가로가 긴 편.)
2. **하단 데크 + 정면 계단 + 발매트(진입 연출):**
   - 건물 일체형 하단에 **넓은 나무 데크(porch)** + 문 아래 중앙에서 마당으로 내려가는 **정면 나무 계단**.
   - 문 바로 앞에 **색 대비가 뚜렷한 발매트**(진입 유도 비주얼 포인트).
3. **★굴뚝 = 지붕 마루에서 솟는다:** 벽돌 굴뚝은 **지붕 능선(ridge)에서 위로 솟아오르는** 형태. **측면에 별도 기둥으로 세우는 것 금지**(owner 2026-07-01 피드백 — "따로 왼쪽에 있는 건 별로").
4. **생활감(Set Dressing):** 창문 옆 **여닫이 나무 셔터** + 단조로운 벽면을 깨는 엣지 디테일(처마 밑 장작더미 등). 단 3의 굴뚝은 지붕으로.
5. **Chunky 강제:** 얇은 정밀 그라데이션 금지 · **진한 단색 외곽선 + 굵은 픽셀 블록**(asset-ruleset §0·§1).
6. **통일감:** 창고·축사 등 부속 건물도 본가와 **같은 디자인 언어**(적갈색 기와 지붕·꿀빛 목재 벽·셔터·데크·발매트·warm 팔레트)로 통일. (부속 건물은 굴뚝 생략 가능.)
7. **팔레트 = 따뜻(집은 §9 저승 muted 예외):** 본가 등 주거 건물은 **따뜻한 갈색 계열**(적갈색 지붕·꿀빛 벽)로 아늑하게. asset-ruleset §9 "모든 객체 저승 muted"의 **예외**(주거 건물은 따뜻하게 — owner 2026-07-01).

## 4. 건물 입체감(Z축 깊이) 규격 (owner md revision 2026-07-01 grill — 전 건물 공통)

> 근거: `docs/design/gemini-shed-barn-spec.md`(owner 창고·축사 입체 보완 지시) + 2026-07-01 grill. "종이 판넬"처럼 납작한 정면 facade에 고저차·접지·recess를 넣어 입체감을 살린다. **본가 포함 전 건물 공통**(grill 확정 — 통일감). 단 이소메트릭 회귀 금지(§1 유지) — 입체는 *정면 유지한 채* 처마 돌출·자기그림자·recess로만.

1. **처마 오버행(Overhang):** 지붕 처마를 정면 벽 평면보다 **1~2 논리px 앞으로 돌출**시켜 지붕이 벽 위로 얹힌 고저차를 만든다.
2. **벽 자기그림자(Self-shadow):** 처마 바로 아래 정면 벽 상단에 **단방향 하향 자기그림자 ~1 논리px(화면 2px)**. **색 = 재질별**(asset-ruleset §1.3 (b)): **warm 건물 = 어두운 warm brown**(꿀빛 목재 램프 dark end) / **저승 객체 = 차가운 청보라-슬레이트**. ★warm 건물에 보라 그림자 금지.
3. **Recessed Door(문틀 입체):** 문짝이 벽면보다 안으로 들어가 보이게 **문틀 상단·좌측면에 어두운 단색 아웃라인(#401818)**을 감아 recess 음영을 낸다.
4. **주춧돌 base band:** 최하단 **~1.5 논리px(화면 3px)**에 회색 돌 주춧돌 / 나무 데크 단면을 결합해 **지면 flush**(묵직한 밑단). §11 접지 그림자 오버레이와 **병행**(중복 아님 — 아트 내부 시각 보강).
5. **축사 특례:** 대형 슬라이딩(또는 쌍여닫이) 도어 하단에 **흙먼지 묻은 석조 받침대 계단**으로 지면과 강제 시각 결합. 단 문은 **남향 중앙**(§2) 유지.
6. **★건물 base = 투명(구워진 지면·잔디 금지):** 건물 아트에 잔디·흙 지면을 굽지 않는다 → 맵 지형이 비쳐 자연 연결(뜬 초록 패치 방지, owner 2026-07-01). glue = green-strip. (asset-ruleset §1.3 참조.)
