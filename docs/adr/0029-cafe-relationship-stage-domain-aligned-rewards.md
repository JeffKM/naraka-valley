# 카페 = 관계의 무대: 소프트 시간 분할 · 프리미엄 엔진 역할 · 도메인 일치 당근 · 카페 일구기 affinity 게이트

> **갱신:** 2026-06-29 (카페 운영 IP-코어 grill). [ADR-0021](./0021-relationship-reward-light-affinity-narrative-primary.md) #2(멜=카지노 보상)를 *개정*하고, [ADR-0009](./0009-macro-goal-build-the-cafe.md)(카페 일구기=동료 호감도 입력)를 *명시적 게이트로 승격*한다.

카페 운영 grill에서 네 갈래가 한 매듭으로 묶였다 — (1) 하루 시간 분할의 강도, (2) 카페에서 플레이어의 역할, (3) 메인 4인의 관계 *당근*의 위치, (4) 카페 발전(카페 일구기)과 관계의 결합. 각각이 서로의 전제라 따로 풀 수 없었다.

문제의 핵심은 **비대칭**이었다. ADR-0021은 메인 4인의 기계적 당근을 *가벼운 호감도 당근*으로 내려놓으며 캐릭터별로 종류를 줬다(미호=농사 XP·바나=전투 XP·멜=카지노 보너스). 그러나 grill에서 두 결함이 드러났다:
- **멜 분열:** 멜 호감도는 *카페 서빙*에서 자라는데 보상은 *카지노*에 있어, "호감도 자라는 곳 ≠ 보상 받는 곳"의 split이 멜에게만 존재했다. 원인 = 멜의 속죄 활동(카페 서빙)이 *공유 오후 무대*라 솔로 아침 도메인이 없어, 미호(밭)·바나(던전)를 흉내 내려 카지노를 가짜 본거지+보상으로 덧댄 것.
- **바나 잔재:** ADR-0021로 [밤 경비]가 폐기됐는데도 바나의 카페 매크로 당근을 "심야 영업·안전"으로 잡으면 죽은 밤 경비를 되살리게 된다.

## 결정

**카페를 "관계가 자라고 매크로 목표가 사는 무대"로 확정하되, 네 원칙으로 묶는다.**

### 1. 소프트 시간 분할 — 노동은 종일, 카페는 기회 창

하루(06:00~24:00)에서 **노동(농사·낚시·채광)은 종일 가능**하고, **카페/바는 15:00~24:00에 "열리는" 옵트인 오버레이**다(그때만 직원·손님이 모임). 오전=노동/오후=서비스는 *강제 분할이 아니라 자연 리듬*(아침엔 미호=밭·멜=카지노·바나=던전이라 카페가 비어 오후에 열릴 뿐). "3시에 활동이 잘린다"는 하드 분할은 기각 — [ADR-0008] #1 "평평≠막힘"·옵트인 에토스 정합. (코드는 이미 이렇게 구현돼 있다 — `cafe.gd`는 영업창만 가르고 노동엔 시간 제한 없음.)

### 2. 플레이어 역할 = 카페의 "프리미엄·성장 엔진" (운영 필수 인력 아님)

- **베이스라인**(카페가 *안 멈춤*) = NPC 직원: [주방요괴]=기본 메뉴, 메인 4인=[융합 메뉴]/특별 음료. 플레이어가 스킵해도 굴러간다 → **자유**.
- **프리미엄·성장**(플레이어 *전용*) = 응대→체키→그림→편지 단계 사슬 · 밤 [칵테일] · [단골화]. 카페의 *성장*([카페 일구기])이 주로 플레이어 출석으로 흐른다 → **의무감 = 기회비용(당근), 처벌 아님**.
- **만성 스킵 결과 = 당근만(감쇠 없음).** 스킵 = 그날 성장·관계·골드를 *안 챙길 뿐*. 단골 안 떠나고 감쇠 없음(이중 처벌·번아웃 회피, "평평≠막힘"·은둔 활동파 보호). 의무감은 기회비용으로 주고 처벌로는 안 준다.
- **자유 = 손님당 기여**(단계 사슬이 *per-customer*라 2~3명만 받고 빠져도 보상의 한 조각을 챙김 — 서비스 창 전체에 매이지 않음).

### 3. 관계 당근 = *자기 도메인*에 둔다 (멜 분열 제거 · 바나 도메인 일치)

호감도가 자라는 곳과 보상 받는 곳을 **일치**시킨다(멜 split의 일반 해법):

| 캐릭터 | 도메인(=호감도 자라는 곳) | 일일 당근 | 매크로 당근 | 카페 일구기 직접 게이트 |
|---|---|---|---|---|
| **미호** | 농사(밭) | 농사 XP 가속 | 카페 **메뉴·농산물** 확장 | ✅ |
| **멜** | 카페 서빙(경제) | **서빙 마진**(팁·단가, *가볍게*) | 카페 **경제·매출·자금** 확장 | ✅ |
| **옥자** | 카페 상주·서사 | (서사) | 카페 **정점·정체성** + 미결의 죄 수렴 | ✅(정점) |
| **바나** | **던전([나락])** | 전투 XP 가속 | **던전 진척**(깊이·접근·전투력) | ❌(자기 도메인) |

- **멜:** 보상을 카페로 통일 — *서빙 마진*(ADR-0008이 원래 줬던 마진 당근을 *가볍게* 복원, ADR-0021 "가벼운 당근" 원칙 불침범) + 매크로=카페 경제 확장(돈을 돌리는 자→카페를 키울 자금). **[카지노]는 멜 보상 채널에서 *선택적 [명부의 운] 베팅 무대 + 속죄 플레이버*로 격하**(멜 호감도에 안 매임 — 카지노는 운을 끌어쓰는 한 무대일 뿐, CONTEXT 명부의 운).
- **바나:** 매크로 당근 = *던전(나락) 진척*(밤 경비 잔재 폐기, 도메인 일치). 카페를 직접 게이트하지 않는다. 단 **나락 희귀 산출이 카페 최상위 프리미엄 메뉴·장식으로 흐르는 *공급 사슬*** 로 카페와 엮인다(미호 농산물의 *심연판*) — *affinity 게이트가 아니라 supply*.

### 4. 카페 일구기 = 카페-도메인 3인(옥자·미호·멜) affinity 분산 게이트

[카페 일구기](ADR-0009) 마일스톤은 멀티루프 산출물 + **카페-도메인 3인의 관계 비트**를 요구한다. 단계마다 *그 단계와 테마가 맞는 한 캐릭터*가 자기 도메인의 카페 측면을 연다(미호=메뉴·농산물 / 멜=경제·매출 / 옥자=정점·정체성). 바나는 던전 도메인이라 직접 게이트가 아니다(위 supply로만 연결).

⚠️ **이는 "관계는 게이트 아님"([ADR-0008] #1·[ADR-0017]·[ADR-0021])의 *부분 한정*이다:** *일일 활동*(농사·낚시·서빙)은 ♡0에서도 base rate로 굴러간다(불변). 게이트되는 건 **매크로 목표(카페 명소화)뿐** — 스타듀 커뮤니티 센터가 "모든 루프를 건드려야 완성"이듯, 우리는 거기 *관계까지* 포함시킨다(활동 게이트 아님, *매크로* 게이트). 게다가 그 호감도는 *각자의 속죄 활동*(미호=농사·멜=서빙·옥자=상주/서사)으로 자라니 **별도 사교 그라인드가 아니라 활동의 부산물**이다(강제 사교 3배 회피).

## 폐기한 대안

- **(하드 시간 분할)** 오전 노동/오후 카페 강제 — 스타듀의 자유로운 하루를 깨고 긴 던전 다이브를 3시에 자름. 옵트인 에토스 위반.
- **(만성 스킵에 감쇠 패널티)** 카페 방치 시 단골 이탈 — 기회비용 + 감쇠의 *이중 처벌*, 은둔 활동파 처벌(ADR-0008 #1 위반), 반(反)처벌·코지 톤 위반.
- **(멜 split 유지)** 멜만 카페 관계 / 카지노 보상 — 비대칭이 그대로. grill에서 사용자가 짚어 통일로 해소.
- **(바나 카페 직접 게이트)** "밤 바 확장"으로 바나를 카페 게이트에 넣기 — 바나 호감도(던전)와 보상(카페 밤바)이 또 갈리는 *멜식 분열의 재발*. 도메인 일치 원칙 위반.
- **(전원 동시 게이트)** 카페 단계마다 4인 전원 ♡N — 강제 사교 3배(ADR-0008이 경계한 바). 분산 게이트로 회피.

## Consequences

- **[ADR-0021] #2 개정:** 멜의 기계적 보상이 *카지노 보너스* → *서빙 마진 + 카페 경제 매크로*. 카지노는 선택적 베팅 무대로 격하(콘텐츠로 존속, 멜 호감도 비연동).
- **[ADR-0009] 승격:** "동료 호감도 = 마일스톤 입력"이 *카페-도메인 3인 분산 affinity 게이트*로 구체화. "구조+서사 한 몸"의 메카닉 형태.
- **바나에 *던전 매크로 목표*가 생긴다** — 카페 일구기와 병렬인 나락 진척(깊이·접근·전투력). 나락 산출은 카페 프리미엄으로 공급 연결. 정밀 곡선은 Phase 3 채광·전투 grill.
- **CONTEXT 갱신:** 카페 운영(소프트 분할·프리미엄 엔진), 호감도(도메인 일치 당근표), 멜(카페 경제·카지노 격하), 바나(던전 매크로·공급), 카지노(격하), 단골화 당근.
- **밸런스 노브:** 서빙 마진·카페 경제 수치, 분산 게이트의 ♡ 레벨, 나락→카페 공급 비율은 Phase 3 밸런싱(설계-잠금 아님).
- **되돌리려면** 이 ADR을 새 ADR로 대체해야 한다(특히 "매크로 목표만 관계 게이트" 부분 한정).
