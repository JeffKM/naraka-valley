# 빌드 워크플로우: 그레이박스→아트 시스템별 인터리브 + 재미 게이트 폐지 — ADR-0001/0023·ROADMAP 원칙 개정

스타듀 시스템 구체화 grill(2026-06-29)에서 아트 피벗([ADR-0026])과 시스템 스코프 대확장
(작물 아키타입·과수·도구 티어·목축·가구갭…)이 겹쳐 **Phase 2.8 이후 로드맵 전면 재배치**가
불가피해졌다. 그 재배치의 *순서 원칙*을 owner가 다음 전제로 다시 정했다:

- **게임은 무조건 출시한다** → "재미 게이트(go/no-go 검증)"의 목적이 사라졌다(중단 결정을 안 하므로).
- **옥자(IP 주인) 비주얼 데모 일정이 대폭 연기**됐다 → 조기 비주얼 압박 해제([ADR-0023]·okja 마일스톤 보류).
- 그래서 남은 결정은 *재미 검증*도 *데모*도 아닌 **순수 빌드 효율**: 그레이박스로 전부 짜고 나중에 아트냐,
  아트를 먼저/같이 만들며 눈으로 확인하며 짜냐.

## 결정

**메카닉 시스템은 "설계(그릴) → 그레이박스 메카닉 → 에셋 스펙 잠금 → 스타듀 아트 → 다음 시스템"으로
*시스템별 인터리브*한다.** 두 극단(전부 그레이박스 후 전부 아트 / 아트 먼저)은 기각.

1. **시스템별 인터리브 (메카닉 시스템: 농사·낚시·채광·전투·카페·관계·목축…).**
   - *그레이박스 메카닉을 먼저*: 아트 스펙(프레임·상태·크기·피드백)은 메카닉이 정의한다. 메카닉 전에
     그리면 스펙을 추측→재작업. 회색 도형은 고치는 비용 ~0, 슬림-스타듀 스프라이트는 비싸다.
     그레이박스가 스펙을 잠그면 **아트를 한 번에 맞게**([ADR-0025] "생성 전 스펙 카드" 정합).
   - *바로 다음에 아트*: 느낌·주스(수확 팝·가독성·애니 타이밍)는 진짜 아트로만 드러난다 → 시스템이
     머릿속에 생생할 때 검증. "전부 그레이박스 후 전부 아트"는 맥락 재학습 비용 + 느낌 문제 말기 폭발.
2. **순수 비주얼은 아트-주도.** 월드 룩·캐릭터/초상화·**가구/장식 시스템**처럼 *그 자체가 아트*인 것은
   그레이박스 의미가 없어 아트가 앞선다.
3. **재미 게이트(Phase 1 fun-gate·"매 스프린트 재밌나? 판정") 폐지.** 게임을 무조건 만들므로 go/no-go
   검증이 불필요. *과거 게이트 판정서(t4.4·t7.4)는 역사적 기록*으로 보존.
4. **"그레이박스 먼저" 원칙의 위상 변경.** 더는 *전역 검증 단계*가 아니라 **시스템 내부의 빌드 스캐폴드**다
   (아트 막힘 없이 로직을 짜고 에셋 스펙을 잠그는 수단). 그레이박스→아트가 *한 시스템 안에서* 이어진다.

## 진행 순서 (owner 확정)

**① 남은 전 시스템 그릴 완료 → ② ROADMAP 전면 재배치·확장 완료 → ③ 빌드 착수.** 설계가 굳기 전엔
빌드/아트에 안 들어간다(재작업 회피). 빌드 착수 후에는 시스템별로 위 인터리브를 돈다.

## 관계 (ADR-0001/0023/0025/0026 · ROADMAP · CLAUDE.md)

- **ADR-0001 유효(연장):** PixelLab/RD + Aseprite 파이프라인·자체 도트화 툴 금지 그대로. 이 ADR은 *언제*
  아트를 만드냐(시스템별 인터리브)를 정할 뿐.
- **ADR-0023 보류:** "코어 시스템 먼저, 맵 에셋 나중"의 *옥자 데모 동기*는 데모 연기로 약화. 단 "코어 먼저"
  정신은 본 ADR의 "메카닉 그레이박스 먼저"로 흡수.
- **ADR-0025 정합:** 그레이박스가 잠근 스펙 = 스펙 카드의 입력. 강화 관계.
- **ROADMAP 원칙·CLAUDE.md "개발 원칙" 개정:** "그레이박스 먼저, 에셋 나중"(전역 단계) + "매 스프린트
  재미 게이트"를 본 ADR대로 갱신한다(같은 커밋에서).
- 되돌리려면(재미 게이트 부활·아트-먼저 전환 등) 새 ADR로 대체.

## 결과

- 로드맵 Phase 2.8 이후는 **시스템 그릴 완료 후** 본 워크플로우로 전면 재작성.
- 옥자 데모 마일스톤은 보류(폐기 아님 — 일정 미정).
- 진행 중이던 P2.8 에셋 작업 중단([ADR-0026] 무효화와 동일 결).
