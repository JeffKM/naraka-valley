# ADR-0048 — 안식 농원 데모 완성 패스: UI 통합 리스킨 + 전 화면 + Track B 실내 앞당김

- **상태:** Accepted (owner 결정 2026-07-02, `grill-with-docs`)
- **개정 대상:** [ADR-0028](./0028-build-workflow-greybox-art-interleave-no-fun-gate.md)(시스템별 그레이박스→아트 인터리브)에 **범위 한정 예외**를 추가한다 — *이 데모(안식 농원) 하나*를 시각적으로 끝내는 통합 패스를 허용. [farm-buildings-roster.md](../design/farm-buildings-roster.md) §3의 "동물 2건물 구조 빌드아웃 = 후행 슬라이스" 조항을 **supersede**(실내 진입은 이 패스로 앞당긴다).
- **관련:** [ADR-0025](./0025-asset-spec-card-gate.md)(스펙카드 게이트)·[ADR-0047](./0047-gemini-full-asset-regen-supersede-adr0001-scope.md)(Gemini 1차 생성기)·[ADR-0018](./0018-minimal-hud.md)(미니멀 HUD)·[dialog-ui-hanji-redesign](../design/portrait-spec-card.md)(태운 한지 UI).
- **스펙 문서:** [docs/design/required-assets-roster.md](../design/required-assets-roster.md) — 데모에 필요한 전 에셋 목표 로스터(위키 디프 소스).

## 맥락 (Context)

owner가 위키(구현 현황 대시보드)와 인게임을 직접 확인한 결과 세 가지를 결정했다:

1. **UI를 스타듀식으로 먼저 완성.** 기능형 UI(핫바·통합 인벤토리 프레임·대화창·혼력 바 등)는 이미 있으나 시계·골드·상점·마일스톤·카페 정산 패널이 **스타일 없는 raw `Label`/`Panel`**이고, `assets/ui/`의 한지 프레임이 일부에만 붙어 있다. 스타듀식 **탭 메뉴 + 한지 스킨**으로 통합·정돈한다.
2. **안식 농원에서 표현 가능한 화면을 모두 만든다.** 야외·집 내부뿐 아니라 축사·닭장·창고 실내, 전체 화면(타이틀·설정·하루 정산·상자), 그리고 화면에 뜨는 모든 패널까지.
3. **위키가 "만들어야 하는데 아직 없는 에셋"을 못 보여준다.** 매니페스트가 `game/assets/**` 파일 스캔 기반이라 없는 파일(가축 노을닭·안개소 등)은 목록에서 아예 빠진다.

문제: ①②는 [ADR-0028] "시스템별 그레이박스→아트 인터리브, 재미 게이트 없이 하나씩" 원칙과 결이 다르다(개별 시스템의 아트가 아니라 *데모 전체의 UI·화면·에셋*을 한 번에 밀어붙이는 패스). ②의 건물 실내는 [farm-buildings-roster.md] §3에서 "후행 슬라이스"로 명시 미뤄둔 Track B 구조 빌드아웃을 앞당긴다. 따라서 새 ADR로 그 예외와 앞당김을 박제한다.

## 결정 (Decision)

### 1. 안식 농원 = "데모 완성 패스"의 대상 (ADR-0028 범위 한정 예외)

다른 슬라이스로 넓히지 않고 **안식 농원(데모 1) 하나**를, UI·전 화면·전 에셋이 채워진 *시각적 완성본*으로 끝낸다. [ADR-0028]의 "시스템 단위 인터리브"는 나머지 슬라이스에 유효하며, 이 패스는 "완성 우선순위 1위 데모를 먼저 닫는다"는 ROADMAP 원칙의 실행이다.

### 2. UI = 기존 기능형 UI의 통합 리스킨 (신규 화면 최소, 로직 재작성 0)

- **통합 탭 메뉴** — ESC/인벤토리 키로 여는 스타듀식 탭: **인벤토리 · 관계(주민) · 숙련 · 옵션/나가기**. 메카닉 있는 것만(제작·지도·수집 탭은 후속 슬라이스). 기존 `inv_frame.gd`를 흡수·확장.
- **한지 9-slice 스킨 시스템** — 화면에 뜨는 raw Godot 패널을 **하나도 남기지 않는다**(마일스톤·카페 정산·상점 포함). 태운 한지 톤([dialog-ui-hanji-redesign]) 계승.
- **HUD** — 우상단 시계 위젯(요일·날짜·시각·계절)+골드 · 하단 툴바 · 우하단 혼력 바(체력 자리 Phase 3) · 좌하단 컨텍스트 팝업(초상화+글) · 아이템 획득 토스트 · 레벨업/숙련 알림 · 마우스 호버 툴팁. **버프/날씨 아이콘은 백엔드 부재로 보류**(후속).
- **전체 화면** — 타이틀/시작 · 설정(볼륨·전체화면, 언어=한국어 고정) · 하루 정산 리스킨 · 카페 정산 리스킨.
- 우하단 바 = **혼력**(=스타듀 스태미나, CONTEXT). 라벨·용어는 게임 정식어 유지.

### 3. Track B 건물 실내를 이 패스로 앞당김

축사(넋우릿간)·닭장(넋둥우리)·창고(갈무리방)를 **진입 가능 실내**로 만든다(실내 타일 + 진입 워프 + pathing + 가축/보관 placeholder 배치, 동물 2건물 분리). **가축 성장 단계(새끼→성체)도 앞당긴다**(owner 2026-07-02 — `livestock.gd`에 성장 로직 신규 = B1-b 성장티어 pull-forward). [farm-buildings-roster.md] §3의 "후행 슬라이스" 조항을 이만큼 supersede한다(여물광·혼우물·늘봄방·혼 연못 등 나머지 Track B는 여전히 후행).

### 4. 상자(저장) = 신규 소형 메카닉

집/창고에 아이템을 보관하는 **저장 상자**(CONTEXT [저장 상자])를 신설한다 — 출하함(즉시 판매)·매대(구매)와 구분되는 순수 보관 컨테이너(경제 0). `inv_frame`에 CHEST 컨텍스트 + 컨테이너 인벤토리 로직 추가.

### 5. 에셋 파이프라인 = 역할 분담, 전부 Gemini-교체 가능

- **Claude 즉시 제작:** UI/HUD 전체(한지 프레임·시계·아이콘·9-slice)·실내 타일(PixelLab/절차).
- **owner Gemini(스펙카드):** 게임플레이 스프라이트 — 가축(노을닭·안개소) 티어별·나머지 작물·과수 혼백도·facade 4종·실내 가구([ADR-0047]·[ADR-0025] 워크플로).
- **불변식:** Claude 제작물은 전부 **placeholder가 아니라 "임시 확정본"**이되, 크기·9-slice 여백·앵커·팔레트를 **스펙카드로 박제 + 위키 등재**해 owner가 나중에 같은 규격 Gemini 결과로 **코드 무수정 덮어쓰기** 가능하게 한다. 화면은 즉시 완성되고 아트는 Gemini 배치로 치환된다.

### 6. 위키 = 목표 로스터 + 상태 디프

`game/assets/**` 스캔에 더해, **데모에 필요한 전 에셋 목표 로스터**([required-assets-roster.md])를 두고 빌드 시 실제 파일과 디프해 각 항목을 ✅있음 / 🟡placeholder(임시 확정본) / ❌없음으로 배지·"남은 작업" 필터로 보여준다.

## 결과 (Consequences)

- **긍정:** 데모 1이 시각적으로 완결(플레이 가능+보기 좋음). raw 패널 0. owner가 "남은 에셋"을 위키에서 한눈에 추적. Claude 제작물이 Gemini로 무마찰 교체 가능해 아트 리드타임과 병렬됨.
- **부정/리스크:** 범위가 큼(UI 리스킨+전 화면+4 실내+Track B 앞당김+상자 신규+스펙카드 다수). 순차 워크트리 격리 작업으로 쪼개 관리한다(A~G, ROADMAP S1-11 참조). 상자·실내 진입은 로직 신규라 회귀 위험 → `game/run_tests.sh` 게이트.
- **[ADR-0028] 정합:** 이 패스는 *안식 농원 한정 예외*이며 나머지 슬라이스의 인터리브 원칙은 불변.
- **범위 밖:** 제작·지도·수집 탭, 버프/날씨 HUD, 여물광·혼우물·늘봄방·혼 연못(Track B 잔여), 카페 본체(Slice 6).

## 진행

빌드 순서(각 = 워크트리 격리 1작업, ROADMAP S1-11 하위): A 위키 로스터+디프 → B 한지 스킨+탭메뉴 → C HUD → D 전체화면 → E 실내 → F 스프라이트 스펙카드→owner Gemini → G 데모 1 통합 시각확인(map_dump 전 화면)·ROADMAP 동기화.

## 개정 노트 — "아트 정리 패스" 정련 (2026-07-05 grill-with-docs)

owner "도구도 아트 필요·안식농원 플레이스홀더/그레이박스 아트 먼저 정리하고 가자" → 이 패스의 **실행 정련**(새 축 아님, 신규 ADR 안 팜). 세 가지를 이 ADR에 흡수하고 [required-assets-roster.md](../design/required-assets-roster.md)에 반영한다:

1. **도구 5종 = 로스터 누락 항목 추가(§0 신규).** 괭이·물뿌리개·낫·곡괭이·도끼(`item_catalog.gd` TOOLS)는 데모에 매일 쓰이는데 로스터 어디에도 없었고, 인게임은 **임시 색박스**(`tool_color_of`)·인벤=텍스트만이다. **도구 아트 범위 = 아이콘만(핫바·인벤).** Claude(PixelLab) 즉시 구동. 배선=`TOOL_ICONS` 레지스트리 + `_draw_icon` CAT_TOOL 분기 텍스처화(색박스 폴백 유지).
   - **★휘두름/손든 도구 스프라이트는 캐릭터 시트 재생성 트랙으로 defer**(되돌리기 비싼 커플링 결정): 현재 `char_sprite.gd`는 `walk_*` 4행(걷기)만 정의하고 `_use_tool()`은 즉발 로직이라 **도구-사용 애니 시스템·손든 렌더 슬롯이 없다**. 지금 오버레이로 넣으면 나중 캐릭터 재생성(방향별 사용 포즈)과 충돌(에셋 낭비·`main.gd` 병렬 비안전 변경)이므로, 휘두름 애니는 캐릭터 시트가 재생성될 때 방향별 use-row로 통합한다. 아이콘만으로 도구 사용은 완전히 플레이됨(ADR-0028 "끝까지 플레이 > 예쁨").

2. **§5 maker 분리를 UI에 대해 C-혼합으로 정련.** 원래 §5는 "UI/HUD **전체** = Claude 즉시 제작"이었으나, 대화창·한지 프레임이 owner-Gemini 손그림 계승본인 점을 감안해 **정체성 강한 UI는 owner-Gemini로 옮긴다**:
   - **Claude(구조적):** 9-slice 패널·슬롯·툴팁·토스트·팝업·정산 패널·상자 UI·골드 아이콘·혼력바 프레임 — `hanji_frame.png` 팔레트 샘플로 톤 맞춤(구조적이라 톤 티 덜 남).
   - **owner-Gemini(정체성):** 타이틀/시작 화면·탭 아이콘 4종·시계 위젯 — 첫인상·브랜드라 손그림 한지 값어치.

3. **이번 패스 완주 범위 = Claude 구동분만(범위 A).** 순서 = ①도구 아이콘 5종 → ②구조적 UI → ③구조적 화면(정산·상자 리스킨)을 **완주**하고 다음 시스템으로 간다. owner-Gemini 항목(타이틀·탭·시계·home_deco 6종·캐릭터 5종)은 **스펙카드로 잠가 생성 큐**로 넘긴다(owner 페이스·다음 시스템 비게이트·무수정 드롭인). §5 불변식(Claude 제작물=임시 확정본, 스펙카드로 Gemini 무수정 교체) 그대로.
