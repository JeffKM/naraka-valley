# 에셋 배치 통합 — 피벗/앵커 + 방향성 면별 변형

탑다운 통합에서 에셋이 떠 보이거나(피벗 어긋남), 도구가 손에서 떨어지거나(스윙 피벗), 가구가 벽을 뚫거나, 절벽 한 면을 전 모서리에 복붙해 어색해지는 문제를 막기 위해 **배치 기준점(피벗/앵커)** 과 **방향성 변형**을 불변식으로 잠근다. 근거 규칙: [asset-ruleset.md](../design/asset-ruleset.md) §3·§4.

## 1. 피벗/앵커 (배치 컨텍스트별)
- **바닥 배치**(건물·바위·그루터기·바닥가구·작물·기계) = **Bottom-Center**. art 바텀 = 타일 하단 경계 → flush 접지, 키 큰 부분은 위로 오버행(Y-Sort).
- **벽 부착**(창문·달력·벽선반) = **호스트 벽(N/E/S/W)의 안쪽 면**에 앵커 + 방을 향함. **북벽이 실사용 기본**(현 `WALL_PROP_LIFT`가 이 케이스). E/S/W는 데이터 필드만 예약.
- **들고있는 도구/장착물**(괭이·낚싯대) = **Center**(16×16 또는 16×32 프레임 정중앙) — 스윙 회전 시 손 이탈 방지. Phase 3 도구 애니 대비 선제 규격.
- 구현: 프롭 엔트리에 `pivot` 필드(`floor`/`wall:N|E|S|W`/`held`). facade 블릿을 top-left → **bottom-center** 앵커로.

## 2. 방향성 면별 변형 (Directional Variants)
- 모서리·면을 가진 방향성 에셋(절벽·벽·울타리·다리)은 **면 방향마다 올바르게 향하는 변형을 따로 생성**한다. 한 면 복붙·식생 가림 핵 금지.
- 면: 남(정면)·동·서·북(상단 lip) + 외부/내부 코너. **flip은 진짜 거울대칭(좌↔우 코너)만**, 단 NW 광원([ADR-0035]/asset-ruleset §1) 때문에 동↔서 면은 **광원 재보정** 필요.
- 즉시 부채: 안식 고지 **동향 절벽 면 타일 신규 생성**(현 남면 복붙+넝쿨 가림 핵 제거).
