# 메타서버에서 온라인 커머스 제거, 결제 엔지니어링을 세이브 정산 파이프라인으로 이전

메타서버 포폴 설계에서 **웹 굿즈샵(토스 결제)·거래소·선착순 한정판**을 들어낸다. 이들이 증명하려던 엔지니어링(④ 결제 = 멱등성·exactly-once·보상 트랜잭션·상태머신, ⑤ 동시성)은 **버리지 않고**, 싱글플레이 게임에 실재하는 자리로 옮긴다: ④는 **세이브 동기화 시점의 정산/지급 파이프라인**으로, ⑤는 **동일 계정 동시 세이브 충돌(낙관락)**로.

## Context

naraka-valley는 ADR-0002(Steam/PC, 오프라인 싱글)·ADR-0005(창발형, 유저 간 상호작용 없음)·ADR-0008(관계=곱셈기) 기준 **"오프라인 싱글, 유저 0명"이 정체성**인 게임이다. 그런데 옛 메타서버 설계는 포폴 스토리(결제·동시성)를 위해 **게임에 없는 온라인 기능을 발명**해 끼워넣고 있었다: 웹 굿즈샵·VIP 멤버십·거래소·선착순 한정판. 전부 "실유저 다수가 돈/희소자원을 주고받는 온라인 커머스"라, 면접에서 "싱글 게임인데 거래소·선착순이 왜 있죠? 실유저가 없잖아요?"로 역공당하는 가짜 명분이었다.

선생님이 결제를 칭찬한 이유는 "토스 연동"이 아니라 그 밑의 **도메인 무관 엔지니어링**(멱등성·exactly-once·보상·상태머신·위변조 검증)이다. 이 기술들은 게임 안 *"가치가 정확히 한 번만 이동하고 크래시에도 안전해야 하는" 진짜 자리* — 세이브 동기화 정산 — 에 그대로 얹을 수 있다.

## Considered Options

- **(A) 결제(④)만 제거** — 토스/웹샵만 들어내고 거래소·선착순은 유지. 같은 뿌리(가짜 온라인)의 거래소·선착순이 남아 동일 역공이 재발.
- **(B) 가짜 온라인 커머스 계열 전부 제거 + 엔지니어링 이전** (채택) — 웹샵 결제·거래소·선착순을 들어내고, ④는 세이브 정산 파이프라인, ⑤는 동일 계정 동시 세이브 충돌로 재배치. 메타서버 자체(치트 방지 권위)는 진짜 명분이라 유지.
- **(C) 메타서버 전체 재검토** — "싱글에 Spring 서버" 전제부터 폐기. 과함 — 메모리 조작 방지는 싱글에서도 실재하는 문제라 메타서버 명분은 살아있음.

## Decision (구체 결정 5가지)

1. **커머스 제거 (B)** — 웹 굿즈샵·토스·VIP 멤버십·거래소·선착순 한정판 삭제.
2. **결제 → 세이브 정산 파이프라인** — `SyncSession` 상태머신(READY→VALIDATED→APPLIED→CONFIRMED / REJECTED·ROLLED_BACK), `sync_tx_id` 멱등, 서버 룰 검증으로 exactly-once 반영. (`04-save-settlement-pipeline.md`)
3. **보상은 내부 분산 정합성으로** — 외부 PG(제3자)를 끼우지 않고, **DB(권위) + Redis(랭킹 ZSET) + async(공유카드)** 의 트랜잭션 경계를 **outbox 패턴**으로 원자적 묶음 + 비동기 멱등 릴레이 + 실패 보상. (Steam Web API 같은 외부 경계는 도입하지 않음 — 정석은 클라 Steamworks 처리라 "왜 서버가?" 역공 여지.)
4. **⑤ 동시성 → 동일 계정 동시 세이브 충돌** — 멀티기기/재시도 레이스의 lost update를 **낙관락(`@Version`)** 으로 방어, 비관락과 부하테스트 비교. **분산락(Redisson)은 의도적으로 미사용**("왜 안 썼나"가 근거).
5. **② N+1 관계형 정당화 재정초** — 관계형 인벤토리의 근거에서 "거래소"를 빼고 **"수집률/도감 랭킹의 크로스-유저 집계(GROUP BY)"** 를 주력 근거로 승격(서버검증·아이템옵션은 보조). N+1 트리거(세이브 동기화 로드)는 불변. (ADR-0001 보정)
6. **① 패턴 재배치** — State: `Order`→`SyncSession` 이사. Strategy: `상품 타입별 지급`→**`DeltaApplier`(진행 종류별 검증/반영: 재화·수확·카페성장·도감수집)**. Observer(`ApplicationEvent`)는 정산 완료 후처리로 유지. (`05` 보정)

## Consequences

- **포폴 스토리 강화** — 가짜 유저·가짜 커머스가 사라지고 "오프라인 싱글에서 exactly-once 정합성을 증명한다"는 더 정직하고 세련된 서사로. 면접 역공("실유저 없잖아요")이 원천 차단됨.
- **이력서 문장 유지** — "토스" 단어만 빠지고 멱등성·상태머신·outbox 보상·낙관락 엔지니어링 깊이는 100% 유지.
- **②③ 강화** — 정산 파이프라인이 ③ I/O(세이브 동기화 벌크 + 랭킹/상점 Redis 캐싱)와 자연 결합, outbox가 ③에 분산 정합성 층을 더함.
- **문서 영향** — `04`는 정산 파이프라인 문서로 재작성(파일명 변경), `01`·`02`·`05`·`README`에서 웹샵/토스/거래소/선착순 제거, ADR-0001·0002 보정.
- **트레이드오프** — "화려한 토스 결제창/분산락 선착순" 같은 *겉보기 임팩트*는 줄지만, 그건 애초에 가짜 명분이었으므로 손실이 아니라 정직성의 회복.
