# 서버 권위 범위 — "메타 계층만"

치트 방지를 위해 서버가 데이터를 검증하되, 검증 권위를 **메타 계층(세이브·재화·인벤토리·랭킹·세이브 정산)에만** 둔다. 농사·이동·손님·성장 등 게임플레이는 클라이언트(Godot)가 오프라인으로 처리한다.

> 🔁 **보정(ADR-0003):** 최초 범위엔 "거래소·결제(웹 굿즈샵)"가 있었으나, 둘 다 싱글플레이에 없는 가짜 온라인 기능이라 제거됐다. 결제의 엔지니어링은 **세이브 동기화 시 정산/지급**으로 이전했다.

## Considered Options

- **전체 서버 권위 (온라인 강제)** — 모든 행동을 서버가 검증해 anti-cheat 최강. 단 오프라인 플레이 불가 → 스타듀 장르 기대 위반, naraka-valley ADR-0002(Steam/PC, 오프라인)와 충돌, 실시간 게임상태 서버 보유로 구현 부담 큼
- **메타 계층만 권위** (채택) — 오프라인 경험 보존 + "돈 되는 데이터"만 서버 검증. anti-cheat 명분 유지
- **순수 클라 저장(서버 없음)** — 메모리 조작으로 재화 치트·아이템 복사 가능. 포폴/무결성 목적에 부적합

## Consequences

- 오프라인 플레이가 유지돼 장르 경험이 안 깨짐
- 서버가 실시간 게임상태를 보유하지 않음 → ① ③ 예시를 서버용으로 재배치해야 함(`01-architecture.md` 주의 참조)
  - ① 디자인 패턴: 손님 로직(클라) → 정산/검증/랭킹 서버 로직으로(`DeltaApplier`·`SyncSession`)
  - ③ I/O: 농작물 매초 write(가짜 문제) → 세이브 동기화 벌크 + Redis 읽기 캐싱으로
- 결제 엔지니어링은 별도 웹샵이 아니라 **세이브 동기화 정산 파이프라인**에 위치(ADR-0003, `04-save-settlement-pipeline.md`)
- 클라/서버 데이터 책임 경계를 명확히 정의해야 함(동기화 프로토콜)
